# 计算机组成原理各章节简答题重点押题

> 用途：考前背诵“简述、说明、比较、分析过程”类题目。以下为押题方向和答案要点，不替代计算题训练。

## 第 1 章 计算机系统概述

### 1. 简述冯诺依曼计算机的基本思想。

答案要点：

- 采用“存储程序”思想，程序和数据都以二进制形式存放在存储器中。
- 计算机由运算器、控制器、存储器、输入设备、输出设备五大部件组成。
- 指令和数据都从存储器取出，由控制器解释指令并协调各部件工作。
- 程序按指令序列执行，通常经历取指、译码、执行等步骤。
- 核心意义：把“人工控制计算过程”变成“机器自动按程序执行”。

### 2. 简述计算机硬件系统中 CPU、主存和 I/O 设备的关系。

答案要点：

- CPU 负责执行指令，包括运算器和控制器。
- 主存保存当前正在运行的程序和数据。
- I/O 设备负责计算机与外部世界交换信息。
- CPU 通过总线或接口访问主存和 I/O 设备。
- 程序执行时，CPU 不断从主存取指令、取数据、运算，再把结果写回主存或输出设备。

### 3. 为什么评价计算机性能不能只看主频？

答案要点：

- 程序运行时间由指令条数、平均 CPI 和时钟周期共同决定。
- 主频只决定时钟周期长短，不能反映每条指令需要多少周期。
- 不同体系结构的一条指令完成的工作量可能不同。
- 编译器、指令系统、存储系统、流水线都会影响实际执行时间。
- 更可靠的指标是具体程序的 CPU 时间。

### 4. 简述 CPI、MIPS 和 CPU 时间的含义及联系。

答案要点：

- CPI：平均每条指令需要的时钟周期数。
- MIPS：每秒执行多少百万条指令。
- CPU 时间：CPU 真正执行某程序所花的时间。
- 基本公式：`CPU 时间 = 指令条数 × 平均 CPI / 主频`。
- MIPS 可能受指令系统影响，不能单独作为跨机器性能比较标准。

### 5. 简述高级语言程序变成可执行程序的大致过程。

答案要点：

- 高级语言程序先经过预处理、编译、汇编、链接等阶段。
- 编译把高级语言翻译成汇编或中间形式。
- 汇编把汇编语言转成机器指令。
- 链接把多个目标文件和库函数组合成可执行文件。
- 最终机器执行的是二进制机器指令，不是源代码本身。

## 第 2 章 数据信息的表示

### 1. 简述原码、反码、补码的表示思想及区别。

答案要点：

- 原码：最高位表示符号，其余位表示绝对值，直观但加减运算不方便。
- 反码：正数同原码，负数符号位不变、数值位按位取反。
- 补码：正数同原码，负数等于其反码末位加 1。
- 补码把减法转化为加法，便于硬件统一处理。
- 原码和反码有 `+0`、`-0` 两个零，补码只有一个零，因此补码负数范围多一个最小负数。

### 2. 为什么 n 位补码整数的表示范围是 `-2^(n-1)` 到 `2^(n-1)-1`？

答案要点：

- n 位二进制共有 `2^n` 种编码。
- 最高位作为符号位时，最高位为 0 的编码表示非负数，共 `2^(n-1)` 个。
- 非负数包括 0，所以最大正数只能到 `2^(n-1)-1`。
- 最高位为 1 的编码表示负数，共 `2^(n-1)` 个。
- 补码只有一个 0，因此负数可多表示一个数，即 `-2^(n-1)`。

### 3. 简述移码的表示方法和主要用途。

答案要点：

- 移码通过给真值加一个固定偏置值来得到机器表示。
- 常用于浮点数阶码，使阶码大小比较变得方便。
- 对 n 位移码，常见偏置为 `2^(n-1)-1` 或按标准规定取值。
- IEEE754 单精度阶码偏置为 127。
- 移码不是普通有符号整数表示，做题时要先减偏置得到真实阶码。

### 4. 简述 IEEE754 单精度浮点数的组成及规格化数的含义。

答案要点：

- 单精度共 32 位：1 位符号位、8 位阶码、23 位尾数字段。
- 符号位决定正负。
- 阶码采用偏置表示，真实阶码 `E = 阶码字段 - 127`。
- 规格化数尾数默认有隐藏的最高位 1，即 `1.M`。
- 数值形式可写为 `(-1)^S × 1.M × 2^E`。

### 5. 简述浮点数为什么会产生精度误差。

答案要点：

- 浮点数位数有限，尾数只能保存有限个二进制有效位。
- 很多十进制小数不能被二进制有限表示，如 0.1。
- 运算时可能发生对阶、舍入、规格化，低位信息被丢弃。
- 大数和小数相加时，小数可能因对阶右移而损失有效位。
- 因此浮点数适合近似表示，不适合直接判断很多小数运算的精确相等。

### 6. 简述海明校验码的基本思想和纠错过程。

答案要点：

- 在数据位中插入若干校验位，校验位通常放在 1、2、4、8 等位置。
- 校验位个数满足 `2^r >= k + r + 1`。
- 每个校验位负责检查一组特定位置。
- 接收端重新计算校验结果，得到指错字。
- 指错字为 0 表示无错；非 0 时，其二进制值指出出错位置，可翻转该位纠正一位错误。

### 7. 简述 CRC 循环冗余校验的工作过程。

答案要点：

- 发送端选定生成多项式，对应一个二进制除数。
- 在原始数据后补上生成多项式阶数个 0。
- 用模 2 除法求余数，模 2 减法本质是异或。
- 将余数作为 CRC 校验位附在数据后发送。
- 接收端用同一生成多项式除接收码字，余数为 0 通常认为无错。
- CRC 主要用于检错，不直接定位并纠正错误。

## 第 3 章 运算方法与运算器

### 1. 简述补码加减法为什么可以统一用加法器实现。

答案要点：

- 补码系统中，减法 `x - y` 可以转化为 `x + (-y)`。
- 负数的补码可以通过按位取反加 1 得到。
- 加法器只需要处理二进制加法，不必单独设计复杂减法器。
- 符号位也参与运算，结果仍按补码解释。
- 只需额外判断是否发生带符号溢出。

### 2. 简述补码加减运算中溢出的判断方法。

答案要点：

- 两个同号数相加，结果符号与操作数不同，则溢出。
- 正数加正数得到负数，是正溢出。
- 负数加负数得到正数，是负溢出。
- 异号相加不会发生带符号溢出。
- 也可用最高位进位和次高位进位异或判断。

### 3. 简述原码一位乘法的主要过程。

答案要点：

- 符号位单独处理，乘积符号由两个操作数符号异或得到。
- 数值部分按绝对值进行二进制乘法。
- 每次根据乘数最低位判断是否把被乘数加到部分积。
- 每轮之后部分积和乘数整体右移。
- 本质是用“加法 + 移位”模拟二进制竖式乘法。

### 4. 简述补码一位乘法 Booth 算法的思想。

答案要点：

- Booth 算法直接处理带符号补码乘法。
- 增设一位附加位 `Q-1`，初值为 0。
- 每轮观察 `Q0Q-1`：`01` 加被乘数，`10` 减被乘数，`00` 或 `11` 不操作。
- 操作后进行算术右移，保持符号。
- 优点是能统一处理正负数，并能利用连续 1 减少加减次数。

### 5. 简述原码不恢复余数除法的基本过程。

答案要点：

- 符号位单独处理，商的符号由被除数和除数符号决定。
- 数值部分按绝对值参与运算。
- 先试减除数，根据余数正负决定上商。
- 若余数为正，下一步继续减除数；若余数为负，下一步加除数。
- 因为不立即把负余数恢复成正余数，所以称为不恢复余数法。
- 最后若余数为负，通常需要恢复一次。

### 6. 简述浮点加减运算的基本步骤。

答案要点：

- 对阶：小阶向大阶对齐，尾数右移。
- 尾数加减：根据符号和运算类型进行尾数运算。
- 规格化：调整尾数和阶码，使尾数满足规格化要求。
- 舍入：根据保留位数处理多余低位。
- 判溢出：检查阶码是否超出可表示范围。

### 7. 简述算术移位和逻辑移位的区别。

答案要点：

- 逻辑移位把操作数看成无符号位串，空位通常补 0。
- 算术移位把操作数看成有符号数，需要保持符号含义。
- 算术右移时，高位补符号位，保证负数大致相当于除以 2。
- 算术左移低位补 0，但可能导致溢出。
- 做题时要先判断题目把数据当作无符号数还是补码带符号数。

## 第 4 章 存储系统

### 1. 简述计算机存储层次结构及其设计目的。

答案要点：

- 常见层次为寄存器、Cache、主存、辅存。
- 越靠近 CPU，速度越快、容量越小、单位成本越高。
- 越远离 CPU，容量越大、速度越慢、单位成本越低。
- 存储层次利用程序局部性原理提高平均访问速度。
- 目标是在成本可接受的情况下，让系统表现得“既快又大”。

### 2. 简述程序局部性原理及其对 Cache 的意义。

答案要点：

- 时间局部性：刚访问过的数据或指令，短时间内可能再次访问。
- 空间局部性：访问某地址后，附近地址也可能很快被访问。
- Cache 把近期使用的数据块从主存调入高速缓存。
- 由于局部性，CPU 后续访问很可能命中 Cache。
- Cache 以块为单位调入，正是为了利用空间局部性。

### 3. 请详细描述直接相联映射、全相联映射、组相联映射的工作原理及优缺点。

答案要点：

- 直接相联：主存块只能映射到 Cache 中唯一指定行，通常由块号取模决定。
- 直接相联优点是硬件简单、查找速度快、成本低；缺点是冲突多，两个常用块若映射到同一行会反复替换。
- 全相联：主存块可以放入 Cache 任意一行，查找时要和所有行标记比较。
- 全相联优点是空间利用灵活、冲突最少；缺点是比较电路复杂、成本高、查找控制复杂。
- 组相联：Cache 分成若干组，主存块先映射到某一组，再可放入该组任意一行。
- 组相联折中直接相联和全相联，冲突比直接相联少，硬件复杂度比全相联低。
- 答题时可从“能放哪里、怎么查、优缺点”三方面展开。

### 4. 简述 Cache 替换算法中 FIFO、LRU、随机替换的特点。

答案要点：

- FIFO：替换最早进入 Cache 的块，实现简单，但不一定替换最久未使用的块。
- LRU：替换最长时间未被访问的块，符合时间局部性，命中率通常较好。
- LRU 需要记录使用历史，硬件实现比 FIFO 复杂。
- 随机替换：随机选择一块替换，实现简单，性能不稳定。
- 直接相联无需选择替换块，因为每个主存块只有唯一位置。

### 5. 简述 Cache 写穿法和写回法的区别。

答案要点：

- 写穿法：CPU 写 Cache 时，同时写主存。
- 写穿法优点是主存和 Cache 内容一致，控制简单；缺点是写流量大，速度可能受主存限制。
- 写回法：CPU 只改 Cache，替换该块时如果被修改过，再写回主存。
- 写回法优点是减少主存写次数，提高效率；缺点是控制复杂，需要脏位。
- 写回法中 Cache 和主存可能暂时不一致。

### 6. 简述虚拟存储器的基本思想和地址转换过程。

答案要点：

- 虚拟存储器让程序看到比实际主存更大的地址空间。
- 程序使用虚拟地址，运行时由硬件和操作系统转换为物理地址。
- 虚拟地址通常分为虚页号和页内偏移。
- 页表根据虚页号找到物理页号，页内偏移保持不变。
- 若页不在主存中，发生缺页异常，由操作系统从辅存调入。
- TLB 用于缓存近期页表项，加快地址转换。

### 7. 简述主存按字节编址时容量和地址位数的关系。

答案要点：

- 按字节编址表示每个地址对应 1 个字节。
- 若地址线有 n 位，可表示 `2^n` 个不同地址。
- 因此最大可寻址容量为 `2^n` 字节。
- 若容量为 C 字节，需要地址位数 `log2 C`。
- 做题时要注意 KB、MB、GB 通常按 2 的幂换算。

## 第 5 章 指令系统

### 1. 简述指令格式通常由哪些字段组成。

答案要点：

- 指令通常由操作码字段和地址码字段组成。
- 操作码说明“做什么操作”，如加、减、访存、跳转。
- 地址码说明“操作数在哪里”或“结果放在哪里”。
- 地址码可能是寄存器编号、存储器地址、立即数或偏移量。
- 指令格式设计会影响指令长度、译码复杂度和程序效率。

### 2. 简述扩展操作码的设计思想。

答案要点：

- 在固定指令字长下，不同地址数指令共享编码空间。
- 地址字段少的指令可把空出的地址字段扩展为操作码。
- 多地址指令通常使用较短操作码，少地址或零地址指令可使用较长操作码。
- 设计时要避免某类指令操作码成为另一类指令的前缀而产生歧义。
- 目标是在有限指令字长内安排更多指令类型。

### 3. 简述常见寻址方式及有效地址 EA 的含义。

答案要点：

- 有效地址 EA 是操作数在主存中的实际地址。
- 立即寻址：操作数直接在指令中，无需再访问主存取操作数。
- 直接寻址：指令地址字段就是 EA。
- 间接寻址：地址字段指向一个存储单元，该单元内容才是 EA。
- 寄存器寻址：操作数在寄存器中。
- 变址寻址常用于数组，EA 通常为基准地址加变址寄存器内容。
- 相对寻址常用于转移，EA 通常为 PC 加偏移量。

### 4. 简述 CISC 和 RISC 的主要区别。

答案要点：

- CISC 指令复杂，指令数量多，寻址方式多，单条指令可能完成较复杂功能。
- RISC 指令数量少、格式规整、长度较固定，强调简单高效执行。
- RISC 通常采用 load/store 结构，访存主要由专门的装入和存储指令完成。
- RISC 更利于流水线设计和硬布线控制。
- CISC 程序指令条数可能较少，但控制器设计更复杂。

### 5. 简述 MIPS 指令 R 型、I 型、J 型格式的区别。

答案要点：

- R 型主要用于寄存器之间的算术逻辑运算。
- R 型通常包含 `op、rs、rt、rd、shamt、funct` 等字段。
- I 型用于立即数运算、访存、条件分支等。
- I 型通常包含 `op、rs、rt、immediate`。
- J 型用于无条件跳转，包含操作码和目标地址字段。
- MIPS 指令格式规整，便于译码和流水线实现。

### 6. 简述过程调用中堆栈的作用。

答案要点：

- 过程调用需要保存返回地址、参数、局部变量和部分寄存器内容。
- 堆栈按后进先出方式管理函数调用现场。
- 调用函数时建立栈帧，返回时恢复栈帧。
- 递归调用依赖每次调用都有独立的栈帧。
- 若不保存现场，函数返回后可能无法恢复调用前状态。

## 第 6 章 中央处理器

### 1. 简述 CPU 的基本组成及各部分作用。

答案要点：

- 运算器负责算术逻辑运算，核心部件包括 ALU 和相关寄存器。
- 控制器负责取指、译码、发出控制信号。
- 寄存器组保存操作数、中间结果、地址和控制信息。
- 数据通路连接寄存器、ALU、存储器接口等部件。
- CPU 通过控制信号让数据在数据通路中按指令要求流动。

### 2. 简述指令周期、机器周期和时钟周期的区别。

答案要点：

- 指令周期是完成一条指令所需的全部时间。
- 机器周期是完成一个基本操作阶段所需时间，如取指、访存。
- 时钟周期是 CPU 最小时间单位，由时钟频率决定。
- 一个指令周期可包含多个机器周期。
- 一个机器周期又可包含若干时钟周期，具体取决于处理器设计。

### 3. 简述一条指令从取指到执行的大致过程。

答案要点：

- 取指：根据 PC 从存储器取出指令。
- 更新 PC：通常指向下一条指令，转移指令可能改变 PC。
- 译码：控制器分析操作码和字段含义。
- 取操作数：从寄存器或存储器取得操作数。
- 执行：ALU 运算、地址计算、访存或分支判断。
- 写回：把结果写回寄存器或存储器。

### 4. 比较单周期、多周期处理器的特点。

答案要点：

- 单周期处理器每条指令都在一个时钟周期内完成，CPI 为 1。
- 单周期的时钟周期必须按最慢指令设置，简单指令也被迫等待。
- 多周期处理器把指令分成多个阶段，不同指令可用不同周期数。
- 多周期时钟周期可按较短的基本阶段设置，硬件可复用。
- 多周期控制更复杂，但整体资源利用更灵活。

### 5. 简述数据通路的含义及其在指令执行中的作用。

答案要点：

- 数据通路是 CPU 中数据流动和加工的硬件路径。
- 包括寄存器、ALU、多路选择器、总线、存储器接口等。
- 指令执行时，控制器通过控制信号选择数据来源、运算方式和写回位置。
- 不同指令在同一数据通路上走不同路径。
- 数据通路设计决定了指令能否被正确执行，也影响性能。

### 6. 比较硬布线控制器和微程序控制器。

答案要点：

- 硬布线控制器用组合逻辑和时序逻辑直接产生控制信号。
- 硬布线控制速度快，适合 RISC 和高速流水线，但设计修改困难。
- 微程序控制器把控制信号组织成微指令，存放在控制存储器中。
- 微程序控制灵活、易修改、适合复杂指令系统。
- 微程序控制通常速度较慢，因为要取微指令并解释执行。

### 7. 简述中断或异常的处理流程。

答案要点：

- CPU 在适当时机检测中断请求或异常原因。
- 保存现场，包括 PC、状态寄存器等关键信息。
- 根据中断类型找到中断服务程序入口。
- 执行中断服务程序或异常处理程序。
- 处理结束后恢复现场，返回原程序继续执行或按异常性质终止。
- 中断使 CPU 能及时响应外设和异常事件，提高系统管理能力。

## 第 7 章 指令流水线

### 1. 简述五段指令流水线各阶段的功能。

答案要点：

- IF：取指，从指令存储器取出指令并更新 PC。
- ID：译码，读取寄存器并识别指令类型。
- EX：执行，进行 ALU 运算、地址计算或分支判断。
- MEM：访存，对数据存储器读写。
- WB：写回，把运算结果或访存结果写回寄存器。
- 五段流水线让多条指令在不同阶段重叠执行。

### 2. 简述流水线提高性能的原因。

答案要点：

- 流水线把指令执行过程拆成多个阶段。
- 多条指令可在不同阶段并行推进。
- 单条指令延迟不一定减少，但单位时间完成的指令数增加。
- 理想情况下，流水线填满后每个周期完成一条指令。
- 性能提升受阶段均衡性、冲突、分支和停顿影响。

### 3. 简述流水线中的结构冲突、数据冲突和控制冲突。

答案要点：

- 结构冲突：多条指令同一时刻争用同一硬件资源。
- 数据冲突：后续指令需要前面指令尚未产生或写回的数据。
- 控制冲突：分支或跳转导致下一条应取哪条指令暂时不确定。
- 结构冲突可通过增加硬件资源或合理调度解决。
- 数据冲突可通过转发、暂停、编译调度解决。
- 控制冲突可通过分支预测、延迟槽、冲刷流水线等处理。

### 4. 简述采用重定向方式解决数据冲突的主要过程。

答案要点：

- 重定向也常称为数据转发或旁路。
- 当后一条指令需要前一条指令结果时，不一定等结果写回寄存器。
- 硬件检测源寄存器和前面指令目的寄存器是否相关。
- 若相关且结果已在 EX/MEM 或 MEM/WB 等流水段产生，就把该结果直接送到后续指令需要的 ALU 输入端。
- 这样可以减少因等待寄存器写回造成的停顿。
- 但 load-use 冒险中，装入数据通常到 MEM 末期才可用，紧跟使用时可能仍需插入一个气泡。

### 5. 简述气泡在流水线中的作用。

答案要点：

- 气泡是人为插入的空操作或无效周期。
- 当后一条指令暂时不能继续执行时，通过气泡让它等待。
- 气泡可以避免使用错误数据或错误控制流。
- 插入气泡会降低流水线吞吐率。
- 常见场景包括 load-use 数据冒险、结构资源冲突、分支等待等。

### 6. 简述分支预测的作用和基本思想。

答案要点：

- 分支预测用于缓解控制冲突。
- 在分支结果真正确定前，先预测下一条取指方向。
- 若预测正确，流水线继续推进，减少停顿。
- 若预测错误，需要冲刷错误路径上的指令，并从正确地址重新取指。
- 动态分支预测可根据历史行为调整预测结果。
- BTB 可缓存分支目标地址，BHT 可记录分支方向历史。

### 7. 简述流水线时空图和性能计算中容易注意的点。

答案要点：

- k 段流水线执行 n 条独立指令，理想周期数约为 `k + n - 1`。
- 实际题目要额外加入停顿周期和冲刷损失。
- 阶段时间不均衡时，时钟周期由最慢阶段决定。
- 流水线加速比不是简单等于段数 k。
- 画图时要标清每条指令每个阶段所在周期。

## 第 8 章 总线系统

### 1. 简述总线的概念及分类。

答案要点：

- 总线是多个部件共享的一组信息传输线路。
- 按传输信息可分为地址总线、数据总线、控制总线。
- 地址总线指出访问对象的位置。
- 数据总线传送数据本身。
- 控制总线传送读写、中断、仲裁、时序等控制信号。
- 按位置也可分为片内总线、系统总线、外部总线等。

### 2. 简述总线事务的一般过程。

答案要点：

- 请求总线：主设备提出使用总线的请求。
- 仲裁授权：仲裁机构决定哪个设备获得总线控制权。
- 寻址：主设备发出地址和控制信息。
- 数据传输：在主设备和从设备之间传送数据。
- 结束释放：传输完成后释放总线，供其他设备使用。
- 完整事务时间会影响实际有效传输率。

### 3. 简述为什么共享总线需要三态门。

答案要点：

- 共享总线上多个设备连接到同一组线路。
- 同一时刻若多个设备同时输出不同电平，会造成总线冲突。
- 三态门有 0、1、高阻态 Z 三种输出状态。
- 高阻态表示设备暂时退出总线，不驱动线路。
- 通过三态门控制，保证同一时刻通常只有一个设备向总线输出数据。

### 4. 比较链式查询、计数器定时查询和独立请求三种集中式总线仲裁方式。

答案要点：

- 链式查询：授权信号按设备顺序逐个传递，硬件简单，优先级固定，离仲裁器近的设备优先。
- 计数器定时查询：仲裁器用计数器轮询设备，优先级可随计数改变，灵活性较好。
- 独立请求：每个设备有独立请求线，仲裁器并行判断。
- 独立请求速度快、响应及时，但控制线多、硬件复杂。
- 答题可从硬件复杂度、响应速度、公平性、优先级灵活性比较。

### 5. 比较同步总线、异步总线和半同步总线。

答案要点：

- 同步总线使用统一时钟控制传输，控制简单，适合速度相近的部件。
- 同步方式受最慢设备限制，扩展到速度差异大的设备不灵活。
- 异步总线不用统一时钟，通过请求、应答等握手信号协调。
- 异步方式适合速度差异大的设备，但控制复杂，握手开销较大。
- 半同步总线以统一时钟为基础，同时允许慢速设备插入等待周期。

### 6. 简述突发传输为什么能提高总线效率。

答案要点：

- 普通传输每传一个数据块通常都要重新给出地址和控制信息。
- 突发传输只在开始时给出首地址。
- 后续连续传多个数据，地址可自动递增或按约定变化。
- 地址阶段和仲裁开销被多个数据共享。
- 因此有效数据传输比例提高，实际有效带宽更高。

### 7. 简述总线带宽和有效传输率的区别。

答案要点：

- 总线带宽通常表示理想条件下单位时间可传输的数据量。
- 常用公式：`带宽 = 总线宽度 / 8 × 频率 × 每周期传输次数`。
- 有效传输率只统计真正有效数据量除以完整事务耗时。
- 地址周期、等待周期、恢复周期、仲裁时间都会降低有效传输率。
- 做题时要区分题目问“理论最大值”还是“实际有效值”。

### 8. 简述 PCIe 与传统并行总线的主要区别。

答案要点：

- 传统并行总线多设备共享同一组并行线路。
- PCIe 采用高速串行点对点连接。
- PCIe 使用多条 lane 并行扩展带宽，如 x1、x4、x16。
- 点对点连接减少共享总线争用问题。
- PCIe 更适合高速外设和现代计算机系统扩展。
