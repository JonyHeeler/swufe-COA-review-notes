# 计算机组成原理 期末复习题

> 范围为前五章

### 1. CPI、MIPS 与 CPU 时间计算

某机器主频为 $1.2\text{GHz}$，执行某程序时共有三类指令：

| 指令类型 | 指令条数             | CPI |
| ---- | ----------------:| ---:|
| A    | $3.0\times 10^8$ | 1   |
| B    | $2.0\times 10^8$ | 2   |
| C    | $1.0\times 10^8$ | 4   |

1. 求程序总指令条数。
2. 求程序总时钟周期数。
3. 求平均 CPI。
4. 求 CPU 执行时间。
5. 求 MIPS。

### 2. 优化前后性能比较

某程序在优化前执行 $8\times 10^8$ 条指令，平均 CPI 为 2，机器主频为 $2\text{GHz}$。优化后，指令条数减少为 $6\times 10^8$ 条，但平均 CPI 增加为 2.4，主频不变。

1. 分别求优化前后的 CPU 执行时间。
2. 分别求优化前后的 MIPS。
3. 判断程序实际运行速度是否提升。
4. 结合本题说明：为什么比较程序性能时，应优先看 CPU 时间，而不能只看 MIPS 或 CPI？

### 3. 数制转换与原码、反码、补码、移码

设机器字长为 8 位。

1. 将 $-37$ 写成 8 位原码、反码和补码。
2. 将 `11011011B` 分别按无符号数、原码、反码、补码解释为真值。
3. 写出 8 位原码整数、反码整数、补码整数的表示范围。
4. 说明为什么原码和反码有两个 0，而补码只有一个 0。
5. 若阶码使用 8 位移码，偏置值为 127，实际阶码 $E=-6$ 应存成什么二进制编码？

### 4. C 语言 signed / unsigned 转换

在 16 位补码机器上，`short` 和 `unsigned short` 均占 16 位。执行：

```c
short si = -12345;
unsigned short usi = si;
```

1. 写出 `si` 的 16 位补码位模式。
2. 将该位模式写成十六进制。
3. 求 `usi` 的十进制值。
4. 解释为什么 `si` 和 `usi` 的位模式相同，但按不同类型解释会得到不同真值。
5. 若改为 `short si = -1; unsigned short usi = si;`，求 `usi` 的值。

### 5. IEEE 754 单精度浮点数编码

已知 `float` 采用 IEEE 754 单精度格式。若 $x=-13.625$，求其 32 位浮点表示。

1. 将 $13.625$ 转成二进制。
2. 写成规格化形式 $1.xxxxx_2\times 2^E$。
3. 写出符号位 $S$。
4. 求阶码字段 `Exponent`。
5. 写出尾数字段 `Fraction`。
6. 写出最终 32 位二进制串和十六进制表示。

### 6. IEEE 754 单精度浮点数解码与范围

给定 IEEE 754 单精度浮点数 `C2480000H`。

1. 展开成 32 位二进制。
2. 划分出 $S$、`Exponent`、`Fraction`。
3. 求实际阶码 $E$。
4. 求规格化尾数 $M$。
5. 求该浮点数的十进制真值。
6. 写出单精度最大规格化正数和最小规格化正数的表达式。

### 7. 海明校验码

设要发送 4 位数据 `1101`，采用偶校验构造海明码。

1. 根据 $2^r\ge k+r+1$ 求校验位个数。
2. 指出校验位应放在哪些位置。
3. 将数据位填入其余位置。
4. 计算各校验位的值。
5. 写出最终发送的海明码。
6. 若接收方收到 `1110101`，计算指错字，判断错误位置并给出纠正后的码字。

### 8. CRC 循环冗余校验

设原始数据为 `110101`，生成多项式为：

$$

$$

1. 写出生成多项式对应的二进制除数。
2. 发送端进行模 2 除法前，应在原始数据后补几个 0？为什么？

4. 求 CRC 校验位。
5. 写出最终发送码字。

### 9. 补码一位乘法（Booth 算法）

用 Booth 算法求 $x\cdot y$：

```text
x = 0.11010
y = -0.00101
```

要求使用双符号位部分积，乘数设置附加位 $y_{n+1}=0$。

1. 写出 $[x]_{\text{补}}$、$[-x]_{\text{补}}$、$[y]_{\text{补}}$。
2. 写出 Booth 判定规则。
3. 按 5 位数值位进行 5 轮运算，列出每轮的判定位、加减操作和算术右移后的状态。
4. 写出最终乘积的补码形式。
5. 将结果解释为真值，检查符号是否合理。

### 10. 原码一位乘法

用原码一位乘法求 $x\cdot y$：

```text
x = -0.10110
y = +0.01101
```

1. 写出 $x$ 和 $y$ 的符号位与数值部分。
2. 求乘积符号位。
3. 只对两个数的绝对值进行一位乘法，列出逐位判断、加被乘数、右移的过程。
4. 写出乘积数值部分。
5. 合成最终原码结果。
6. 说明原码一位乘法为什么通常要单独处理符号位。

### 11. 原码不恢复余数除法

用原码不恢复余数除法求 $x/y$。已知：

```text
[x]_{\text{补}} = 0.10011
[y]_{\text{补}} = 0.11101
```

其中 $x$、$y$ 均为正小数，要求求 5 位小数商。

1. 写出 $[y]_{\text{补}}$ 和 $[-y]_{\text{补}}$。
2. 写出不恢复余数法的上商规则。
3. 从初始余数 $R=x$ 开始，列出 5 步加减交替过程。
4. 写出每一步得到的商位。
5. 写出最终商 $Q$。
6. 若最后余数为负，应如何修正？真实余数为什么要乘以 $2^{-5}$？

### 12. 乘除法方法比较

围绕原码一位乘法、补码一位乘法和原码不恢复余数除法回答：

1. 原码一位乘法中，符号位如何确定？数值部分如何处理？
2. Booth 补码一位乘法中，为什么要设置附加位 $y_{n+1}$？
3. Booth 算法中，`01`、`10`、`00/11` 分别对应什么操作？
4. 不恢复余数除法中，余数为正和余数为负时，下一步分别如何处理？
5. 三种算法的共同点是什么？都在怎样利用“加法和移位”完成乘除运算？

### 13. 主存按字节编址

某计算机主存容量为 $16\text{MB}$，按字节编址，字长为 32 位。

1. 主存共有多少个可编址单元？
2. MAR 至少需要多少位？

5. 若改为按 32 位字编址，MAR 至少需要多少位？说明与按字节编址的区别。

### 14. Cache 组相联映射

某计算机 Cache 共有 16 行，采用二路组相联映射；主存块大小为 $32\text{B}$，主存按字节编址。问主存地址 `129` 所在主存块应装入哪一组。

3. 块内偏移是多少？

5. 说明为什么不能直接用地址 `129` 对 Cache 组数取模。

### 15. Cache 地址字段划分与行位数

某机器主存地址为 32 位，按字节编址。Cache 数据区容量为 $32\text{KB}$，主存块大小为 $32\text{B}$，采用直接映射，写策略为 Write Back。每个 Cache 行除数据块外，还需保存 Tag、有效位和脏位。

### 16. Cache 替换算法

### 17. 虚拟存储器地址转换

某系统虚拟地址为 24 位，物理地址为 20 位，页面大小为 $4\text{KB}$。虚拟地址 `12ABCH` 需要转换为物理地址。页表中虚拟页号 `12H` 的页表项有效，映射到物理页框号 `3AH`。

1. 求页内偏移位数。

5. 拼接得到物理地址。
6. 说明为什么虚实地址转换只替换页号，页内偏移保持不变。

### 18. 扩展指令码

某机器采用 16 位定长指令字，每个地址字段为 4 位，采用扩展操作码设计三地址、二地址、一地址和零地址指令。

1. 三地址指令的操作码有多少位？最多可设计多少条三地址指令？
2. 若已设计 12 条三地址指令，最多还能扩展出多少条二地址指令？
3. 若再设计 20 条二地址指令，最多还能扩展出多少条一地址指令？

6. 说明为什么已分配给多地址指令的短操作码不能再作为少地址指令的扩展前缀。
