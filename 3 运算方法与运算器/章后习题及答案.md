# 第 3 章 章后习题及答案

> 来源依据：教材第 3 章习题；第 3 章 PPT；`知识点汇总.txt` 第三章。

## 3.1 名词解释要点

本章名词可以按“加法电路、移位与标志、乘除法、浮点运算”四组记忆。

| 名词 | 解释要点 |
| --- | --- |
| 半加器 | 只计算两个输入位相加，不考虑低位进位。 |
| 全加器 | 计算两个输入位和低位进位，是多位加法器的基本单元。 |
| 进位生成函数 | 表示本位是否自己产生进位，常写为 `G_i=A_iB_i`。 |
| 进位传递函数 | 表示本位是否会把低位进位传到高位，常写为 `P_i=A_i⊕B_i`。 |
| 算术移位 | 面向带符号数，右移时高位补符号位。 |
| 逻辑移位 | 面向位串或无符号数，空位补 0。 |
| 阵列乘法器 | 用规则排列的加法单元并行形成部分积，速度比串行乘法快。 |
| 恢复余数除法 | 试减失败后立即把除数加回去，恢复原余数。 |
| 不恢复余数除法 | 试减失败后不立即恢复，而在下一轮用相反操作修正。 |
| 对阶 | 浮点加减前把两个数的阶码调整一致。 |
| 规格化 | 把浮点运算结果整理成规定的尾数形式。 |
| 保留附加位 | 多保留若干低位，用来减少舍入误差。 |

## 3.2 选择题答案

| 题号 | 答案 | 解析要点 |
| --- | --- | --- |
| 1 | D | `short -9` 符号扩展后为 `FFF7H`，`127+(-9)=118=00000076H`。 |
| 2 | B | `F2H` 按 8 位补码解释为 `-14`，`90H` 为 `-112`，乘积 `1568` 超出 8 位补码范围。 |
| 3 | A | 先按补码求真值：`x=-12`，`2x=-24`，`y=-80`，`y/2=-40`，结果为 `-64=11000000B`。 |
| 4 | C | `x=-33`，`y=65`，所以 `x-y=-98`，32 位补码为 `FFFFFF9EH`。 |
| 5 | B | 逻辑右移高位补 0，算术右移高位补符号位。 |
| 6 | C | 浮点加减要先对阶，再做尾数运算，最后规格化。 |
| 7 | D | 对阶、右规、左规和尾数溢出的判断都成立。 |

做选择题时不要急着套答案。凡是出现十六进制机器数，先问它是“无符号数”还是“补码带符号数”；凡是出现浮点数，先拆成符号、阶码、尾数。

## 3.3 简答题答案要点

1. 并行进位提前计算各位进位，减少逐级等待时间；本质是用更复杂的组合逻辑换更短的加法延迟。
2. 浮点溢出主要看规格化和舍入后阶码是否超过可表示范围，尾数本身不是范围的主要决定因素。
3. 规格化是为了让尾数符合统一形式；尾数太大要右规并使阶码加 1，尾数太小要左规并使阶码减小。
4. CAS 单元的加减控制和上商判断都与余数符号有关，因此可以复用部分控制信号。
5. 左移常对应乘 2，右移常对应除 2；但带符号数要区分算术移位和逻辑移位，且左移可能溢出。

## 3.4 与 3.5 变形补码加减法

这类题不要直接心算，按模板写最稳。

通用步骤：

1. 先统一小数位数或整数位数。
2. 写成双符号位补码。
3. 加法直接相加；减法先转成加相反数。
4. 最高位之外的进位仍按机器位数规则处理。
5. 根据双符号位判断溢出。

双符号位判断表：

| 双符号位 | 含义 |
| --- | --- |
| `00` | 正数，未溢出 |
| `11` | 负数，未溢出 |
| `01` | 正溢出，结果比最大正数还大 |
| `10` | 负溢出，结果比最小负数还小 |

理解方法：双符号位正常时应该相同。如果两个符号位不同，就说明结果已经装不进原来的表示范围。

## 3.6 原码一位乘法

原码乘法先把符号和数值拆开。

解题模板：

```text
1. 符号位 = x符号 ⊕ y符号
2. 被乘数和乘数都取绝对值
3. 部分积初值 = 0
4. 从乘数最低位开始看：
   为 1，则部分积加被乘数
   为 0，则部分积不变
5. 每轮部分积和乘数整体右移
6. 重复到乘数数值位处理完
7. 给最终数值结果加上符号位
```

记忆技巧：乘数某位为 `1`，就贡献一份被乘数；乘数某位为 `0`，这一位没有贡献。右移只是为了让硬件每轮都处理“当前最低位”。

## 3.7 补码一位乘法

补码一位乘法常用 Booth 表。它不把符号位单独拿掉，而是让补码直接参与运算。

| `Q0 Q-1` | 操作 | 怎么理解 |
| --- | --- | --- |
| `00` | 不操作 | 仍处在一段 0 中 |
| `01` | 加 `[x]补` | 连续 1 的结束边界 |
| `10` | 加 `[-x]补` | 连续 1 的开始边界 |
| `11` | 不操作 | 仍处在一段 1 中 |

每轮之后，`A、Q、Q-1` 整体算术右移。这里用算术右移，是因为中间部分积可能是负数，必须保持补码符号。

易错提醒：Booth 法看的是两位 `Q0Q-1`，不是只看乘数当前最低位。

## 3.8 原码不恢复余数除法

这类题的核心是“试商”。原码除法仍然先处理符号：

```text
商符号 = 被除数符号 ⊕ 除数符号
```

数值部分再按绝对值做除法。

解题模板：

1. 取被除数、除数绝对值。
2. 每轮先左移余数和商寄存器。
3. 若操作前余数为正，减除数。
4. 若操作前余数为负，加除数。
5. 操作后余数为正，上商 `1`；余数为负，上商 `0`。
6. 重复到规定商位数。
7. 若最终余数为负，通常加除数恢复。

最容易混的地方是：操作看“操作前余数符号”，上商看“操作后余数符号”。

## 3.9 至 3.10 浮点运算

浮点加减必须按固定流程写：

```text
对阶 → 尾数加减 → 规格化 → 舍入 → 判断溢出
```

每一步怎么想：

| 步骤 | 作用 | 易错点 |
| --- | --- | --- |
| 对阶 | 让两个尾数处在同一位权下 | 通常小阶向大阶对齐 |
| 尾数加减 | 真正做有效数字的加减 | 异号相加本质是减法 |
| 规格化 | 把尾数整理成规定格式 | 右规阶码加 1，左规阶码减小 |
| 舍入 | 处理尾数字段装不下的低位 | 舍入可能再次产生进位 |
| 溢出判断 | 检查阶码是否超范围 | 范围主要看阶码，不是只看尾数 |

IEEE754 单精度题还要注意隐藏位、阶码偏置 `127` 和特殊阶码字段。复习时建议先用十进制科学计数法理解数量级对齐，再回到二进制字段计算。
