# 加法器与先行进位

> 来源依据：第 3 章 PPT“一位全加器”“快速加法器”“先行进位电路”。

## 一句话理解

加法器的难点不在“每一位怎么加”，而在“高位要等低位进位传上来”，先行进位就是为了解决等待太久的问题。

## 基本概念

二进制加法和十进制加法一样，也有进位。比如：

```text
1 + 1 = 10B
```

本位写 `0`，向高位进 `1`。加法器就是实现这种逐位相加的电路。

常见部件：

| 部件 | 能处理什么 | 局限 |
| --- | --- | --- |
| 半加器 | 两个输入位相加 | 不能处理低位来的进位 |
| 全加器 | 两个输入位 + 低位进位 | 可以串起来做多位加法 |
| 串行进位加法器 | 多个全加器逐级连接 | 高位必须等待低位进位 |
| 先行进位加法器 | 提前算出各位进位 | 速度快，但电路更复杂 |

一位全加器有三个输入：

```text
A_i, B_i, C_i
```

其中 `C_i` 是低位送来的进位。它输出：

```text
S_i：本位和
C_{i+1}：送往高位的进位
```

## 工作原理或推导过程

### 1. 串行进位为什么慢

4 位加法看起来只有 4 位，但最高位进位 `C4` 要等：

```text
C0 → C1 → C2 → C3 → C4
```

也就是说，`C4` 必须等前面每一级都算完。如果位数变成 32 位或 64 位，这种等待会变得更明显。

### 2. 进位生成和进位传递

先行进位把每一位对进位的影响分成两种情况。

**进位生成**：

```text
G_i = A_iB_i
```

如果 `A_i` 和 `B_i` 都是 1，那么本位一定会产生进位，不管低位有没有进位。

**进位传递**：

```text
P_i = A_i ⊕ B_i
```

如果 `A_i` 和 `B_i` 恰好一个为 1，那么低位来的进位会被传到高位。

因此一位进位可以写成：

```text
C_{i+1} = G_i + P_iC_i
```

这句话可以读成：高位进位要么由本位自己生成，要么由低位进位经过本位传上来。

### 3. 先行进位怎么快

把公式展开，就能提前算出多个进位：

```text
C1 = G0 + P0C0
C2 = G1 + P1C1
   = G1 + P1G0 + P1P0C0
C3 = G2 + P2G1 + P2P1G0 + P2P1P0C0
```

这样 `C1、C2、C3...` 可以由组合逻辑更早算出来，不必一级一级等进位慢慢传递。

代价也很明显：电路门数更多，结构更复杂。先行进位是用硬件复杂度换速度。

## 典型例子

设某一位：

```text
A_i = 1, B_i = 1
```

则：

```text
G_i = 1
P_i = 0
C_{i+1} = 1
```

这说明本位自己已经产生进位，低位进位是什么都不影响这一位向高位进 1。

再设：

```text
A_i = 1, B_i = 0
```

则：

```text
G_i = 0
P_i = 1
C_{i+1} = C_i
```

这说明本位不自己产生进位，但会把低位进位传上去。

## 常见考法

- 区分半加器和全加器。
- 写出进位生成函数 `G_i` 和进位传递函数 `P_i`。
- 根据 `C_{i+1}=G_i+P_iC_i` 展开多位进位。
- 说明串行进位慢在哪里，先行进位快在哪里。
- 比较先行进位加法器的速度优势和电路复杂度代价。

## 易错点

- 半加器没有低位进位输入，全加器才有。
- 先行进位不是“不需要进位”，而是提前算进位。
- 先行进位速度快，但电路更复杂，不是免费提升。
- 教材或 PPT 中 `P_i` 可能采用 `A_i⊕B_i` 或 `A_i+B_i` 的等价变体，做题时按题目定义使用。

## 小结

- 加法器是运算器和 ALU 的基础。
- 串行进位加法器简单，但高位要等低位进位。
- 先行进位用 `G_i` 和 `P_i` 提前计算进位，提高加法速度。
- 本章后面的乘法器、除法器，本质上也大量依赖加法器和移位器。
