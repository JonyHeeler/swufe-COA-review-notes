# 补码加减法与溢出判断

> 来源依据：`知识点汇总.txt` 第三章；第 3 章 PPT“补码加减法的实现”“溢出检测”；教材第 3 章。

## 一句话理解

补码加减法的核心是：把减法变成加法，让计算机用同一个加法器完成整数加减。

## 基本概念

计算机硬件最擅长做加法。为了让减法也能交给加法器处理，补码设计了一套规则：

```text
[x - y]补 = [x]补 + [-y]补
```

也就是说，机器并不需要真的“减去 y”，它只要“加上 y 的相反数”。

为什么最高位进位可以丢弃？因为 n 位补码运算本质上是在固定的 n 位空间里做模 `2^n` 运算。超过 n 位的那一位已经跑出机器能保存的范围，硬件会自然舍掉。

以 8 位为例：

```text
-1 的补码 = 11111111
+1 的补码 = 00000001

  11111111
+ 00000001
=1 00000000
```

最左边多出来的 `1` 被丢弃，结果是 `00000000`，正好表示 0。这说明“丢弃最高进位”不一定是错误，它是补码模运算的一部分。

## 工作原理或推导过程

### 1. 补码加法怎么做

步骤很固定：

1. 先确定位数，例如 8 位、16 位或 32 位。
2. 把两个操作数都写成相同位数的补码。
3. 逐位相加。
4. 丢弃最高位之外的进位。
5. 按补码规则解释结果，并判断是否溢出。

### 2. 补码减法怎么做

减法先变成加法：

```text
x - y = x + (-y)
```

所以步骤是：

1. 写出 `[x]补`。
2. 写出 `[-y]补`，也就是 y 的相反数的补码。
3. 两个补码相加。
4. 判断结果是否溢出。

### 3. 为什么会溢出

n 位补码能表示的范围有限：

```text
-2^(n-1) ~ 2^(n-1)-1
```

只要数学上的正确结果超出这个范围，机器就装不下，发生带符号溢出。

例如 8 位补码范围是：

```text
-128 ~ 127
```

所以 `127 + 1 = 128` 超出范围，会溢出。

### 4. 溢出怎么判断

最常用的符号规律：

```text
正 + 正 = 负，正溢出
负 + 负 = 正，负溢出
正 + 负 或 负 + 正，不会发生带符号加法溢出
```

也可以用进位判断：

```text
符号位的进位输入 与 符号位的进位输出 不同，则溢出
```

如果用变形补码，也就是双符号位，结果可这样看：

| 双符号位 | 含义 |
| --- | --- |
| `00` | 正数，未溢出 |
| `11` | 负数，未溢出 |
| `01` | 正溢出 |
| `10` | 负溢出 |

## 典型例子

### 例 1：8 位补码计算 `127 + 1`

```text
  01111111   (+127)
+ 00000001   (+1)
= 10000000
```

两个正数相加，结果符号位却是 `1`，按 8 位补码解释为 `-128`。

但数学结果应为 `128`，8 位补码表示不了，所以发生正溢出。

### 例 2：8 位补码计算 `-1 + 1`

```text
  11111111   (-1)
+ 00000001   (+1)
=1 00000000
```

丢弃最高进位后：

```text
00000000 = 0
```

这是正确结果，没有溢出。这个例子说明：最高位产生进位不等于带符号溢出。

### 例 3：8 位补码计算 `50 - 20`

先转成加法：

```text
50 - 20 = 50 + (-20)
```

写补码：

```text
  00110010   (+50)
+ 11101100   (-20)
=1 00011110
```

丢弃最高进位：

```text
00011110 = 30
```

一正一负相加，不会发生带符号溢出。

## 常见考法

- 给两个真值，要求写补码、计算结果并判断溢出。
- 给两个机器码，要求按补码解释并计算。
- 给减法表达式，要求转化为补码加法。
- 判断最高位进位、进位标志和溢出标志的区别。
- 用双符号位变形补码判断正溢出或负溢出。

## 易错点

- 最高位进位被丢弃是补码运算规则，不一定表示出错。
- 带符号溢出看同号相加是否得到异号结果。
- 减法要先变成加相反数，再套加法溢出判断。
- 做不同位数的数相加前要符号扩展，不能直接补 0。
- 无符号溢出和带符号溢出不是一回事，分别对应进位标志和溢出标志。

## 小结

- 补码让加法和减法都能交给加法器处理。
- n 位补码运算按模 `2^n` 进行，最高位之外的进位会被丢弃。
- 溢出的本质是数学结果超出了当前补码位数能表示的范围。
- 最实用的判断口诀是：同号相加得异号，发生溢出。
