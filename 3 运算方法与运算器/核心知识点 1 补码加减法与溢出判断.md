# 补码加减法与溢出判断

> 来源依据：`知识点汇总.txt` 第三章；第 3 章 PPT“补码加减法的实现”“溢出检测”；教材第 3 章。

## 一句话理解

补码加减法的核心是：把减法变成加法，让计算机用同一个加法器完成整数加减。

## 基本概念

计算机硬件最擅长做加法。为了让减法也能交给加法器处理，补码设计了一套规则：

```text
[x - y]补 = [x]补 + [-y]补
```

也就是说，机器并不需要真的“减去 y”，它只要“加上 y 的相反数”。

为什么最高位进位可以丢弃？因为 n 位补码运算本质上是在固定的 n 位空间里做模 `2^n` 运算。超过 n 位的那一位已经跑出机器能保存的范围，硬件会自然舍掉。

以 8 位为例：

```text
-1 的补码 = 11111111
+1 的补码 = 00000001

  11111111
+ 00000001
=1 00000000
```

最左边多出来的 `1` 被丢弃，结果是 `00000000`，正好表示 0。这说明“丢弃最高进位”不一定是错误，它是补码模运算的一部分。

## 工作原理或推导过程

### 1. 补码减法怎么做

1. 写出 `[x]补`。
2. 写出 `[-y]补`，也就是 y 的相反数的补码。
3. 两个补码相加。
4. 判断结果是否溢出。

### 2. 为什么会溢出？（本质原因）

溢出的根本原因在于计算机硬件的“容器大小是固定的”。

在补码运算系统中，如果是 $n$ 位的补码，它能表示的数值范围被严格限制在 $[-2^{n-1}, 2^{n-1}-1]$ 之间。

一旦两个合法的数字相加（或同号相减），得到的**数学真实结果超出了这个固定范围**，计算机用现有的位数就“装不下”了，这就是带符号溢出。

以最常见的 **8 位机器**为例，它的合法范围是 -128 到 127：

- **正溢出**：你试图计算 `127 + 1`。在 8 位补码里，`01111111` (+127) 加上 `00000001` (+1)，进位直接冲进了最高位的符号位，变成了 `10000000`。计算机一看最高位是 1，把它解释成了 -128。两个正数加出了负数，发生了正溢出。

- **负溢出**：你试图计算 `-128 + (-1)`。真实结果是 -129，超出了向下的极限，结果符号位反而变成了 0，被错误地解释成了一个正数。

**注意一个核心易错点**：最高位产生了进位，**绝不等于**溢出。比如 `-1 + 1`，最高位肯定会往外进位 1，但在补码的模运算规则下，这个多出来的 1 会被自然丢弃，留下 `00000000`（即 0），这是完全正确的结果。

### 3. 如何判断溢出？（三种核心方法）

计算机系统和我们在纸上做题，通常使用以下三种判断逻辑。

#### 1. 基于运算结果的符号（最直观的人工判断法）

- 正数 + 正数 = 负数 $\rightarrow$ **正溢出**

- 负数 + 负数 = 正数 $\rightarrow$ **负溢出**

- 正数 + 负数 $\rightarrow$ 绝对不可能溢出

#### 2. 基于单符号位的进位判断（底层硬件级判断）

CPU 不懂什么是“正号负号”，它只认极其精确的数字逻辑电路。CPU 判断溢出是通过一个异或门（XOR）来完成的。

判断规则是比较**符号位的进位情况**：

- **规则**：当“最高数值位向符号位产生的进位（Carry-in）” **不等于** “符号位向外产生的进位（Carry-out）”时，发生溢出。

- 在硬件电路上，只需要把这两个进位信号接入一个 XOR 门。如果一端是 1，另一端是 0，XOR 输出 1，硬件立刻捕捉到溢出。

#### 3. 基于双符号位（变形补码法）

这种方法常用于 ALU（算术逻辑单元）的内部设计。给补码分配**两个符号位**（正数用 `00`，负数用 `11`）。运算时双符号位一起参与：

- 结果为 `00`：正数，未溢出。

- 结果为 `11`：负数，未溢出。

- 结果为 `01`：最高位符号位是 0，次高位变成了 1。说明正数相加破坏了次高位，发生了**正溢出**。

- 结果为 `10`：最高位符号位是 1，次高位变成了 0。发生了**负溢出**。

## 典型例子

### 例 1：8 位补码计算 `50 - 20`

先转成加法：

```text
50 - 20 = 50 + (-20)
```

写补码：

```text
  00110010   (+50)
+ 11101100   (-20)
=1 00011110
```

丢弃最高进位：

```text
00011110 = 30
```

一正一负相加，不会发生带符号溢出。

## 常见考法

- 给两个真值，要求写补码、计算结果并判断溢出。
- 给两个机器码，要求按补码解释并计算。
- 给减法表达式，要求转化为补码加法。
- 判断最高位进位、进位标志和溢出标志的区别。
- 用双符号位变形补码判断正溢出或负溢出。

## 易错点

- 最高位进位被丢弃是补码运算规则，不一定表示出错。
- 带符号溢出看同号相加是否得到异号结果。
- 减法要先变成加相反数，再套加法溢出判断。
- 做不同位数的数相加前要符号扩展，不能直接补 0。
- 无符号溢出和带符号溢出不是一回事，分别对应进位标志和溢出标志。

## 小结

- 补码让加法和减法都能交给加法器处理。
- n 位补码运算按模 `2^n` 进行，最高位之外的进位会被丢弃。
- 溢出的本质是数学结果超出了当前补码位数能表示的范围。
- 最实用的判断口诀是：同号相加得异号，发生溢出。
