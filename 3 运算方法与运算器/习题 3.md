# 习题 3 答案

## 3.1 名词解释要点

#### 一、 加法器与进位机制

这组名词描述了计算机中最基础的加法运算及其优化演进路径。

- **半加器**：能实现两个一位二进制数相加的逻辑电路，但不具备接收低位进位的功能。

- **全加器**：在半加器的基础上，不仅能实现两个一位二进制数相加，还能接收低位的进位输入，是构建多位加法器的基础单元。

- **串行进位**：多位加法器的一种进位传递方式。低位的进位输出直接作为高位的进位输入，进位信号必须逐位向上传递，速度较慢。

- **先行进位**（超前进位）：为了打破串行进位的速度瓶颈，各位的进位信号不再等待低位传递，而是由所有相关的操作数直接通过逻辑电路并行计算产生。

- **进位生成函数**：先行进位机制的基石之一。当本位的两个操作数均为 1 时，必然会向高位产生进位，这个逻辑表达式即为进位生成函数。

- **进位传递函数**：先行进位机制的另一基石。当本位的两个操作数中有一个为 1 时，它自身不产生进位，但若低位传来了进位，它就能将该进位继续传递给高位。

#### 二、 移位操作

这组名词是数据处理和乘除法运算的重要基础操作。

- **逻辑移位**：针对无符号数的数据位移操作。不考虑符号位，无论是左移还是右移，空出的数据位一律补 0。

- **算术移位**：针对带符号数（如补码）的数据位移操作。移位时会严格保持符号位不变，以此来完成有符号数据的乘或除以 2 的指数次幂的运算。

#### 三、 乘除法实现

这组名词介绍了高级算术运算在纯硬件和算法策略上的不同实现。

- **阵列乘法器**：为了追求极致的乘法速度，直接使用大量全加器等组合逻辑器件构成的空间并行硬件阵列来完成计算。

- **阵列除法器**：类似于阵列乘法器，使用大量并行逻辑阵列硬件来直接完成除法运算，成本高但速度极快。

- **原码恢复余数除法**：一种经典的硬件除法算法。当被除数减去除数导致余数为负时，必须再把除数加回来以“恢复”原来的正确余数，然后再继续求下一位的商。

- **原码不恢复余数法**（加减交替法）：针对恢复余数法的改良方案。当余数为负时直接上商 0，且不进行恢复，而是直接在下一步改用加法操作继续运算，省去了多余的步骤，提高了效率。

### 四、 浮点数运算

这组名词展示了浮点数（科学计数法）在进行加减运算时的特殊步骤。

- **对阶**：浮点数加减法必不可少的第一步。通过移动小阶码数的尾数，使得两个浮点数的阶码变得完全一致，从而让尾数具有相同的数学权重以便相加减。

- **保留附加位**：在对阶或尾数运算的过程中，为了防止低位数据因移位而直接丢失，硬件会在尾数末尾额外保留的保护位数（如保护位、舍入位），用以保证最终的舍入精度。

- **规格化**：浮点数运算完成后的标准整理步骤。通过左规或右规调整结果的尾数，使其满足特定的标准形式（如 IEEE 754 标准中尾数最高有效位始终隐藏为 1），以此保证数据表示的唯一性和最大精度。

## 3.2 选择题答案

| 题号  | 答案  | 解析要点                                                            |
| --- | --- | --------------------------------------------------------------- |
| 1   | D   | `short -9` 符号扩展后为 `FFF7H`，`127+(-9)=118=00000076H`。             |
| 2   | B   | `F2H` 按 8 位补码解释为 `-14`，`90H` 为 `-112`，乘积 `1568` 超出 8 位补码范围。     |
| 3   | A   | 先按补码求真值：`x=-12`，`2x=-24`，`y=-80`，`y/2=-40`，结果为 `-64=11000000B`。 |
| 4   | C   | `x=-33`，`y=65`，所以 `x-y=-98`，32 位补码为 `FFFFFF9EH`。                |
| 5   | B   | 逻辑右移高位补 0，算术右移高位补符号位。                                           |
| 6   | C   | 浮点加减要先对阶，再做尾数运算，最后规格化。                                          |
| 7   | D   | 对阶、右规、左规和尾数溢出的判断都成立。                                            |

做选择题时不要急着套答案。凡是出现十六进制机器数，先问它是“无符号数”还是“补码带符号数”；凡是出现浮点数，先拆成符号、阶码、尾数。

## 3.3 简答题答案要点

1. 并行进位提前计算各位进位，减少逐级等待时间；本质是用更复杂的组合逻辑换更短的加法延迟。
2. 浮点溢出主要看规格化和舍入后阶码是否超过可表示范围，尾数本身不是范围的主要决定因素。
3. 规格化是为了让尾数符合统一形式；尾数太大要右规并使阶码加 1，尾数太小要左规并使阶码减小。
4. CAS 单元的加减控制和上商判断都与余数符号有关，因此可以复用部分控制信号。
5. 左移常对应乘 2，右移常对应除 2；但带符号数要区分算术移位和逻辑移位，且左移可能溢出。

## 3.4 与 3.5 变形补码加减法

这类题不要直接心算，按模板写最稳。

通用步骤：

1. 先统一小数位数或整数位数。
2. 写成双符号位补码。
3. 加法直接相加；减法先转成加相反数。
4. 最高位之外的进位仍按机器位数规则处理。
5. 根据双符号位判断溢出。

双符号位判断表：

| 双符号位 | 含义            |
| ---- | ------------- |
| `00` | 正数，未溢出        |
| `11` | 负数，未溢出        |
| `01` | 正溢出，结果比最大正数还大 |
| `10` | 负溢出，结果比最小负数还小 |

理解方法：双符号位正常时应该相同。如果两个符号位不同，就说明结果已经装不进原来的表示范围。

## 3.6 原码一位乘法

原码乘法先把符号和数值拆开。

解题模板：

```text
1. 符号位 = x符号 ⊕ y符号
2. 被乘数和乘数都取绝对值
3. 部分积初值 = 0
4. 从乘数最低位开始看：
   为 1，则部分积加被乘数
   为 0，则部分积不变
5. 每轮部分积和乘数整体右移
6. 重复到乘数数值位处理完
7. 给最终数值结果加上符号位
```

记忆技巧：乘数某位为 `1`，就贡献一份被乘数；乘数某位为 `0`，这一位没有贡献。右移只是为了让硬件每轮都处理“当前最低位”。

## 3.7 补码一位乘法

补码一位乘法常用 Booth 表。它不把符号位单独拿掉，而是让补码直接参与运算。

| `Q0 Q-1` | 操作        | 怎么理解       |
| -------- | --------- | ---------- |
| `00`     | 不操作       | 仍处在一段 0 中  |
| `01`     | 加 `[x]补`  | 连续 1 的结束边界 |
| `10`     | 加 `[-x]补` | 连续 1 的开始边界 |
| `11`     | 不操作       | 仍处在一段 1 中  |

每轮之后，`A、Q、Q-1` 整体算术右移。这里用算术右移，是因为中间部分积可能是负数，必须保持补码符号。

易错提醒：Booth 法看的是两位 `Q0Q-1`，不是只看乘数当前最低位。

## 3.8 原码不恢复余数除法

这类题的核心是“试商”。原码除法仍然先处理符号：

```text
商符号 = 被除数符号 ⊕ 除数符号
```

数值部分再按绝对值做除法。

解题模板：

1. 取被除数、除数绝对值。
2. 每轮先左移余数和商寄存器。
3. 若操作前余数为正，减除数。
4. 若操作前余数为负，加除数。
5. 操作后余数为正，上商 `1`；余数为负，上商 `0`。
6. 重复到规定商位数。
7. 若最终余数为负，通常加除数恢复。

最容易混的地方是：操作看“操作前余数符号”，上商看“操作后余数符号”。

## 3.9 至 3.10 浮点运算

浮点加减必须按固定流程写：

```text
对阶 → 尾数加减 → 规格化 → 舍入 → 判断溢出
```

每一步怎么想：

| 步骤   | 作用           | 易错点            |
| ---- | ------------ | -------------- |
| 对阶   | 让两个尾数处在同一位权下 | 通常小阶向大阶对齐      |
| 尾数加减 | 真正做有效数字的加减   | 异号相加本质是减法      |
| 规格化  | 把尾数整理成规定格式   | 右规阶码加 1，左规阶码减小 |
| 舍入   | 处理尾数字段装不下的低位 | 舍入可能再次产生进位     |
| 溢出判断 | 检查阶码是否超范围    | 范围主要看阶码，不是只看尾数 |

IEEE754 单精度题还要注意隐藏位、阶码偏置 `127` 和特殊阶码字段。复习时建议先用十进制科学计数法理解数量级对齐，再回到二进制字段计算。
