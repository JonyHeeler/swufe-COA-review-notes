# 原码不恢复余数除法

> 来源依据：`知识点汇总.txt` 第三章；第 3 章 PPT“恢复余数法”“不恢复余数法”；教材第 3 章。

## 一句话理解

不恢复余数除法是在做“试商”：余数为正就试着减除数，余数为负就用下一步加除数来修正。

## 基本概念

除法比乘法更像“试出来”的。手算十进制除法时，我们会判断够不够减：够减就商上某个数，不够就换小一点。二进制除法更简单，商的每一位只可能是 `0` 或 `1`：

```text
够减：上商 1
不够减：上商 0
```

原码除法通常把符号和数值分开：

- 商的符号 = 被除数符号位 `⊕` 除数符号位。
- 商的数值部分 = 被除数绝对值 ÷ 除数绝对值。
- 余数符号通常与被除数保持一致，数值部分按算法得到。

**恢复余数法**：如果试减后余数变负，说明不够减，要把除数加回去恢复。

**不恢复余数法**：如果试减后余数变负，不马上加回去，而是在下一轮改做加除数。这样省掉了“立即恢复”这一步。

## 工作原理或推导过程

### 1. 为什么“余数正减除数，余数负加除数”

假设某一步试减后余数为负，说明刚才减多了。恢复余数法会立刻加回除数：

```text
负余数 + 除数 = 恢复后的余数
```

不恢复余数法把这一步推迟到下一轮。下一轮本来要先左移余数，再继续试减。把“先恢复再左移再减除数”合并变形后，就可以得到“余数为负时，下一轮左移后加除数”的规则。

因此操作口诀是：

```text
余数为正：下一步减除数
余数为负：下一步加除数
```

### 2. 基本步骤

原码不恢复余数除法常见流程：

1. 先判断符号，商符号为两操作数符号异或。
2. 取被除数和除数的绝对值，数值部分参加除法。
3. 每轮先让余数和商寄存器整体左移一位。
4. 若上一步余数为正，则减除数；若上一步余数为负，则加除数。
5. 本轮操作后，余数为正则上商 `1`，余数为负则上商 `0`。
6. 重复到规定商位数。
7. 若最后余数为负，通常要加除数恢复成非负余数。

### 3. 怎么记表格

表格题最重要的三列是：

| 当前余数符号 | 本轮操作 | 本轮上商 |
| --- | --- | --- |
| 正 | 减除数 | 结果正则 1，结果负则 0 |
| 负 | 加除数 | 结果正则 1，结果负则 0 |

更简单地说：

- 操作由“操作前余数符号”决定。
- 商位由“操作后余数符号”决定。

## 典型例子

### 例 1：为什么试减失败时上商 0

设当前要判断余数 `R` 够不够减除数 `D`。

如果：

```text
R - D >= 0
```

说明够减，本位商可以写 `1`。

如果：

```text
R - D < 0
```

说明不够减，本位商应写 `0`。恢复余数法会把 `D` 加回去；不恢复余数法把这个修正留给下一轮。

### 例 2：某一轮余数为负怎么办

假设上一轮结束后余数为负，说明上一轮减多了。下一轮不再继续减除数，而是：

```text
左移余数 → 加除数
```

如果加完后余数变正，本轮上商 `1`；如果仍为负，本轮上商 `0`。

## 常见考法

- 给定两个原码小数，用不恢复余数法列表求商。
- 判断每一轮应加除数还是减除数。
- 判断本轮上商 `0` 还是 `1`。
- 比较恢复余数法和不恢复余数法的差异。
- 说明最后余数为负时为什么还要恢复。

## 易错点

- 商的符号和商的数值部分要分开处理。
- “不恢复余数”不是最后也不恢复，最终余数为负时通常要恢复。
- 操作由操作前余数符号决定，商位由操作后余数符号决定。
- 每轮通常包含左移、加/减除数、上商，顺序不要乱。
- 除数不能为 0；定点小数除法还要注意题目规定的数值范围。

## 小结

- 原码除法通常先处理符号，再处理绝对值。
- 不恢复余数法用下一轮相反操作代替立即恢复。
- 余数为正下一步减除数，余数为负下一步加除数。
- 表格题要盯住余数、操作、商位三件事。
