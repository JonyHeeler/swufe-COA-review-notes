# 算术移位、逻辑移位与标志位

> 来源依据：第 3 章 PPT“移位运算”“标志寄存器”“整数乘法溢出判断”。

## 一句话理解

移位看起来只是移动二进制位，但如果这个位串表示带符号数，就必须小心符号、溢出和标志位。

## 基本概念

移位是 CPU 中很常见的操作。它可以用来快速乘以 2、除以 2，也可以用来处理纯粹的位串。

常见移位：

| 类型 | 空出来的位补什么 | 常用于 |
| --- | --- | --- |
| 逻辑左移 | 低位补 0 | 位串操作、无符号乘 2 |
| 逻辑右移 | 高位补 0 | 无符号除 2、位串操作 |
| 算术左移 | 低位补 0 | 带符号数乘 2，但要检查溢出 |
| 算术右移 | 高位补符号位 | 带符号数除 2 的近似实现 |

标志位是 CPU 用来记录运算结果特征的小标记，常见有：

| 标志位 | 含义 | 常见用途 |
| --- | --- | --- |
| `ZF` 零标志 | 结果是否为 0 | 判断相等 |
| `SF` 符号标志 | 结果最高位是否为 1 | 判断结果正负 |
| `CF` 进位标志 | 无符号运算是否产生进位/借位 | 判断无符号溢出 |
| `OF` 溢出标志 | 带符号运算是否溢出 | 判断补码有符号溢出 |

## 工作原理或推导过程

### 1. 为什么左移常相当于乘 2

二进制每一位的权值是 2 的幂。整体左移一位，原来每一位的权值都变成 2 倍，所以通常相当于乘 2。

例如 8 位无符号数：

```text
00001100B = 12
左移一位 -> 00011000B = 24
```

但如果最高位被移出去，或者带符号数符号改变，就可能溢出。

### 2. 逻辑右移和算术右移为什么不同

逻辑右移只把位串当作普通二进制串，高位补 0。

算术右移要尽量保持补码负数的符号，所以高位补符号位。对于负数，符号位是 1，因此右移时高位补 1。

例如 8 位补码：

```text
11011000
```

如果按补码解释，它是 `-40`。

逻辑右移：

```text
01101100
```

最高位变成 0，按补码看变成正数 `108`，符号含义完全变了。

算术右移：

```text
11101100
```

仍然是负数，按补码解释为 `-20`，更接近“除以 2”的效果。

### 3. CF 和 OF 怎么区分

`CF` 主要服务于无符号数。无符号加法超过最大值，或者减法不够减，会影响进位/借位。

`OF` 主要服务于带符号补码数。判断重点是数学结果是否超出带符号范围。

同一个位运算结果，可能出现：

- `CF=1` 但 `OF=0`。
- `CF=0` 但 `OF=1`。

所以不能把进位标志和溢出标志混为一谈。

## 典型例子

### 例 1：8 位补码负数右移

```text
原位串：11011000
逻辑右移 1 位：01101100
算术右移 1 位：11101100
```

如果题目把它当作补码带符号数，应该用算术右移保持符号。

### 例 2：左移导致溢出

8 位补码中：

```text
01000000 = +64
左移 1 位 -> 10000000
```

`10000000` 按 8 位补码表示 `-128`，不是数学结果 `128`。因为 8 位补码最大正数是 `127`，所以发生溢出。

## 常见考法

- 给定位串，分别求逻辑左移、逻辑右移、算术右移结果。
- 判断移位是否等价于乘 2 或除以 2。
- 根据运算结果判断 `ZF、SF、CF、OF`。
- 区分无符号溢出和带符号溢出。
- 判断移位后符号位变化是否说明溢出。

## 易错点

- 逻辑右移高位补 0，算术右移高位补符号位。
- 负数算术右移补 1，不是补 0。
- 左移虽然低位补 0，但仍然可能溢出。
- `CF` 不等于 `OF`，一个偏无符号，一个偏带符号。
- 算术右移负数常接近除以 2，但与高级语言中负数整数除法规则不一定完全相同，做题按课程给定规则。

## 小结

- 逻辑移位把数据当位串，算术移位把数据当带符号数。
- 左移常对应乘 2，右移常对应除 2，但必须检查溢出、符号和舍入。
- 标志位帮助 CPU 记录结果特征，并服务于比较、分支和异常判断。
