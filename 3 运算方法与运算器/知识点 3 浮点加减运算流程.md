# 浮点加减运算流程

> 来源依据：第 3 章 PPT“浮点运算”“对阶”“规格化”“浮点运算器”。

## 一句话理解

浮点加减法要先把两个数的小数点对齐，再对尾数做加减，最后把结果重新整理成规格化形式。

## 基本概念

浮点数可以写成：

```text
N = (-1)^S × M × 2^E
```

其中 `M` 是尾数，`E` 是阶码。两个浮点数相加时，不能只把尾数直接相加，因为它们的阶码可能不同，尾数所在的“位权位置”不同。

就像十进制中：

```text
1.23×10^4 + 4.56×10^2
```

不能直接算 `1.23+4.56`，必须先把数量级对齐：

```text
1.23×10^4 + 0.0456×10^4
```

二进制浮点加减也是同样道理，这一步叫**对阶**。

## 工作原理或推导过程

浮点加减通常按五步：

```text
对阶 → 尾数加减 → 规格化 → 舍入 → 溢出判断
```

### 1. 对阶

对阶就是让两个数的阶码相同。通常采用：

```text
小阶向大阶对齐
```

也就是把阶码较小的数的尾数右移，同时阶码增大。

为什么不把大阶向小阶对齐？因为那样要把大数尾数左移，可能需要更多整数位，更容易溢出。让小数右移虽然会损失低位精度，但更适合硬件处理。

### 2. 尾数加减

阶码相同后，两个尾数才处在同一位权下。此时根据符号做尾数加法或减法：

- 同号相加，尾数相加。
- 异号相加，本质上是尾数相减，结果符号取绝对值较大者的符号。

### 3. 规格化

尾数运算后可能不再满足规格化要求。

如果尾数相加后变成：

```text
10....
```

说明尾数太大，需要右规：

```text
尾数右移 1 位，阶码加 1
```

如果尾数相减后前面出现很多 0，例如：

```text
0.001...
```

说明尾数太小，需要左规：

```text
尾数左移，直到符合规格化；阶码相应减小
```

### 4. 舍入

对阶和规格化时，尾数右移可能会移出一些低位。机器尾数字段有限，不能无限保存所有位，所以要根据舍入规则决定是否进位。

保留附加位的目的就是让舍入更准确。常见思想是：不要刚移出去就完全忘掉，而是多留几位帮助判断。

### 5. 溢出判断

浮点数范围主要由阶码决定。规格化和舍入之后，要检查阶码是否超出范围：

- 阶码太大，可能上溢。
- 阶码太小，可能下溢或变成非规格化数。

## 典型例子

设：

```text
X = 1.101 × 2^5
Y = 1.010 × 2^3
```

先对阶。`Y` 的阶码小，要向 `X` 的阶码 `5` 对齐：

```text
Y = 1.010 × 2^3
  = 0.01010 × 2^5
```

现在阶码相同，可以加尾数：

```text
  1.10100
+ 0.01010
= 1.11110
```

所以：

```text
X + Y = 1.11110 × 2^5
```

这个例子没有发生右规或左规。如果尾数相加得到 `10....`，就要右规；如果相减后得到 `0.00...`，就要左规。

## 常见考法

- 写出浮点加减完整流程。
- 判断为什么要对阶，以及对阶方向。
- 给两个浮点数，手工完成对阶和尾数运算。
- 判断规格化时阶码如何变化。
- 分析舍入、附加位和精度损失。
- 判断阶码是否上溢或下溢。

## 易错点

- 浮点尾数不能直接相加，必须先看阶码是否相同。
- 对阶一般是小阶向大阶对齐。
- 对阶右移会丢低位，所以可能产生精度损失。
- 右规时阶码加 1，左规时阶码减小。
- 舍入可能让尾数再次进位，从而需要再次右规。
- 没有溢出不代表没有误差，浮点运算常常会损失精度。

## 小结

- 浮点加减的主线是对阶、尾数运算、规格化、舍入、溢出判断。
- 对阶是为了让两个尾数处在同一位权下。
- 范围主要由阶码决定，精度主要由尾数决定。
- 浮点运算不是简单的实数精确运算，误差多来自对阶和舍入。
