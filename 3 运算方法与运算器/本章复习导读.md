# 第 3 章 运算方法与运算器复习导读

## 本章定位

第 2 章讲“数在机器里怎样表示”，第 3 章接着讲“这些机器数怎样真正参与加、减、乘、除和浮点运算”。

可以把本章理解成：从数学运算走进硬件电路。人看到的是 `x+y`、`x×y`、`x÷y`，机器真正做的是补码相加、移位、试商、规格化和舍入。

## 核心优先级

| 优先级 | 内容 | 复习要求 |
| --- | --- | --- |
| 核心 | 补码加减法与溢出判断 | 会统一位数、写补码、做加减，并判断带符号溢出 |
| 核心 | 原码/补码一位乘法 | 能看懂“加被乘数、右移”的来源，会列乘法表 |
| 核心 | 原码不恢复余数除法 | 能根据余数正负决定加/减除数和上商 |
| PPT 补充 | 加法器、先行进位、ALU | 理解加法器为什么是运算器的基础，先行进位为什么快 |
| PPT 补充 | 移位、标志位、浮点加减 | 会区分逻辑移位/算术移位，掌握浮点加减五步 |

## 学习顺序

1. 先复习补码。第 3 章的整数加减几乎都建立在补码上。
2. 再学补码加减和溢出判断。这是乘法、除法、ALU 的基础。
3. 然后学一位乘法。它的本质是用“加法 + 移位”模拟二进制竖式乘法。
4. 再学不恢复余数除法。它的本质是不断试商，只是试错后不立即恢复。
5. 最后看加法器、移位和浮点加减流程，把算法和硬件联系起来。

## 本章最重要的思维方式

第 3 章不要只背表格动作，要先问：

- 这个运算最终能不能转化成加法？
- 每一次移位是在模拟乘 2、除 2，还是对齐小数点？
- 当前符号位、进位或余数符号说明了什么？

比如补码减法：

```text
x - y = x + (-y)
```

所以硬件不必单独造一个“减法器”，只要能求 `[-y]补`，再送进加法器即可。

再比如乘法：

```text
1011 × 0101
```

乘数某一位是 `1`，就把被乘数加进结果；乘数某一位是 `0`，这一位不贡献部分积。机器只是在用二进制方式重复这件事。

## 易错提醒

- 带符号溢出看数学结果是否超出补码范围，不能只看最高位有没有进位。
- 做补码加减必须先统一位数，负数扩展时要符号扩展。
- 原码乘除法常把符号位单独处理，数值部分按绝对值运算。
- 补码 Booth 乘法看的是相邻两位 `Q0Q-1`，不是只看当前最低位。
- 不恢复余数除法不是“永远不恢复”，最后余数为负时通常还要恢复。
- 浮点加减必须先对阶，小阶向大阶对齐。
