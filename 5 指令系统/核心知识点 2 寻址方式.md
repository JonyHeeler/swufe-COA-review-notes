# 寻址方式

> 来源依据：`知识点汇总.txt` 第五章第 2 点；第 5 章 PPT“寻址方式”；教材第 5 章。

## 一句话理解

寻址方式就是 CPU 按什么规则找到操作数：指令里给出的那串二进制，有时就是数据本身，有时是地址，有时还要和寄存器或 PC 相加后才是真正地址。

## 基本概念

学习寻址方式，最重要的是分清三件事：

```text
形式地址 D：指令地址字段里直接写着的内容
有效地址 EA：经过寻址方式计算后，真正要访问的主存地址
操作数 S：最终参与运算的数据
```

三者不一定相同。

举个生活化例子：

- 立即寻址：纸条上直接写着“买 3 个苹果”，3 就是数据。
- 直接寻址：纸条上写着“去 5 号柜子拿东西”，5 是地址。
- 间接寻址：纸条上写着“去 5 号柜子找另一张纸条”，另一张纸条才写着真正地址。
- 寄存器寻址：东西就在手边的小抽屉里，小抽屉就是寄存器。
- 寄存器间接寻址：小抽屉里不是东西本身，而是柜子编号。

下面是一些常见的符号约定:

#### 1. 指令自带的数字：`D`

- **含义：** 形式地址 (Formal Address) 或立即数 (Immediate Data)。

- **大白话：** 这是**硬编码在指令那串 0101 二进制代码里**的数字。

- **理解关键：** 此时的 `D` 就是一个纯粹的数字，它具体是个什么东西，完全取决于 CPU 怎么解释它。
  
  - 如果 CPU 把它当成**操作数**（立即寻址），那 `D=1000` 就是真正的数值 $1000$。
  
  - 如果 CPU 把它当成**地址**（直接寻址），那 `D=1000` 指的就是“1000号储物柜”。

#### 2. 最终的目标地址：`EA`

- **含义：** 有效地址 (Effective Address)。

- **大白话：** 经过一系列加减运算、或者翻箱倒柜之后，**最终真正藏着你要的数据的那个储物柜编号**。

- **理解关键：** 无论是基址寻址算出来的 `(BR) + D`，还是相对寻址算出来的 `(PC) + D`，它们最终算出来的那个结果，统称为 `EA`。拿到 `EA`，就等于拿到了取数据的最终钥匙。

#### 3. 柜子编号 vs 柜子里的东西

这是最核心、也最容易出错的一组对比：

##### 主存层面：`x` vs `M[x]`（或 `(x)`）

- `x`：代表**储物柜的编号**（主存地址）。比如 $x = 1000$，就是指 1000 号柜子。

- `M[x]` 或 `(x)`：代表**储物柜里面装的东西**（主存地址 x 中的内容）。只要加了方括号 `[]` 或圆括号 `()`，就意味着“打开这个柜子，拿出里面的数据”。

- **举个例子：** 如果 1000 号柜子里放着数字 35。那么 $x = 1000$，而 $M[1000] = 35$。

##### 寄存器层面：`R` vs `(R)`

- `R`：代表**小抽屉的编号/名字**（寄存器本身）。比如 $R1$，$R2$。

- `(R)`：代表**小抽屉里面装的东西**（寄存器 R 中的内容）。

- **举个例子：** 如果你要把 $R1$ 寄存器里的数字当做地址去访问内存（寄存器间接寻址），公式必须写成 `EA = (R1)`，意思是“打开 $R1$ 抽屉，把里面的那张纸条拿出来，纸条上写的数字才是有效地址”。

#### 4. 特殊的指针：`PC`

- **含义：** 程序计数器 (Program Counter)。

- **大白话：** 这是一个专门用来指路的特殊寄存器，它永远保存着**当前或下一条要执行的指令的地址**。

- **理解关键：** 在“相对寻址”中，我们用 `EA = (PC) + D` 来计算跳转地址。这里的 `(PC)` 就是把 PC 这个寄存器里的当前地址读出来，加上一个偏移量 `D`，算出我们接下来该往哪儿跳。

我们用例 1 的数据来验证一下你是否看懂了括号的魔力：

假设指令给出的 `D = 1000`，且内存中 `M[1000] = 2000`，`M[2000] = 35`。

在**间接寻址**下，CPU 寻找数据的过程如下：

1. 先算有效地址：`EA = M[D]`
   
   - CPU 看到 `M[1000]`，表示去打开 1000 号柜子。
   
   - 柜子里装的是 2000，所以算出 `EA = 2000`。

2. 再取操作数：`S = M[EA]`
   
   - CPU 看到 `M[2000]`，表示再去打开真正的目标柜子（2000 号柜子）。
   
   - 柜子里装的是 35，所以最终取出的数据是 $35$。

只要严格区分开“数字本身（没括号）”和“打开柜子拿里面的东西（有括号）”，寻址方式的计算公式就不会再让你迷糊了。

## 工作原理或推导过程

### 1. 常见寻址方式公式

| 寻址方式    | 指令中给出的内容       | 有效地址或操作数                |
| ------- | -------------- | ----------------------- |
| 立即寻址    | 操作数本身          | `S = D`                 |
| 直接寻址    | 主存地址           | `EA = D`，`S = M[EA]`    |
| 间接寻址    | 存放地址的主存地址      | `EA = M[D]`，`S = M[EA]` |
| 寄存器寻址   | 寄存器编号          | `S = R` 中的内容            |
| 寄存器间接寻址 | 保存地址的寄存器编号     | `EA = (R)`，`S = M[EA]`  |
| 基址寻址    | 位移量 D + 基址寄存器  | `EA = (BR) + D`         |
| 变址寻址    | 形式地址 D + 变址寄存器 | `EA = D + (IX)`         |
| 相对寻址    | 位移量 D + PC     | `EA = (PC) + D`         |

### 2. 立即、直接、间接怎么区分

这三个最容易混。

#### 立即寻址

指令字段里就是操作数。

```text
MOV R1, #5
```

这里 `5` 就是要送入 R1 的数据，不需要把 5 当地址去访问主存。

#### 直接寻址

指令字段里是主存地址。

```text
LOAD R1, 1000
```

意思是到主存地址 1000 取内容：

```text
R1 = M[1000]
```

#### 间接寻址

指令字段里给的是“地址的地址”。

```text
LOAD R1, @1000
```

如果：

```text
M[1000] = 2000
M[2000] = 35
```

那么：

```text
EA = M[1000] = 2000
R1 = M[2000] = 35
```

间接寻址比直接寻址多访问一次主存。

### 3. 基址、变址、相对怎么理解

这三种都常写成“寄存器内容 + 位移量”，但用途不同。

#### 基址寻址

```text
EA = (基址寄存器) + 位移量
```

基址寄存器通常保存一段程序或数据区的起始地址。位移量表示离起点多远。

它适合程序重定位：程序整体搬到主存另一个位置时，只要改基址寄存器，指令中的位移量可以不变。

#### 变址寻址

```text
EA = 形式地址 + (变址寄存器)
```

变址寄存器常用来保存数组下标偏移。

例如数组首地址是 `1000H`，每个元素 4B，要访问 `a[i]`：

```text
EA = 1000H + 4i
```

这里 `4i` 就很适合放在变址寄存器中。

#### 相对寻址

```text
EA = (PC) + 位移量
```

相对寻址常用于转移指令。它不直接写目标地址，而是写“相对当前位置跳多远”。

这样程序整体搬家时，前后指令的相对距离不变，转移仍然成立。

注意：有些题目中 PC 已经指向下一条指令，有些题目中 PC 仍指向当前指令，要按题目说明计算。

## 典型例子

### 例 1：同一个 D 在不同寻址方式下含义不同

设：

```text
D = 1000
M[1000] = 2000
M[2000] = 35
R1 = 3000
M[3000] = 88
```

不同寻址方式结果如下：

| 寻址方式  | 计算                                  | 得到的操作数 |
| ----- | ----------------------------------- | ------ |
| 立即寻址  | `S = D`                             | `1000` |
| 直接寻址  | `EA = 1000`，`S = M[1000]`           | `2000` |
| 间接寻址  | `EA = M[1000] = 2000`，`S = M[2000]` | `35`   |
| 寄存器寻址 | `S = R1`                            | `3000` |
| 寄存器间接 | `EA = R1 = 3000`，`S = M[3000]`      | `88`   |

这个表能看出：不要一看到 `1000` 就默认它是数据或地址，要看寻址方式。

### 例 2：变址寻址访问数组

某数组 `A` 首地址为 `2000H`，每个元素 4B，要访问 `A[5]`。

解读一下已知条件：

- **首地址（2000H）：** 数组 `A` 在内存中连续存放的起点。在机器指令中，这个起点通常直接写在**形式地址 $D$** 里。

- **元素大小（4B）：** 数组里每一个数据（比如一个 `int` 整数）占 4 个字节的空间。

- **目标下标（5）：** 我们要找 `A[5]`，也就是数组的第 6 个元素（因为下标是从 0 开始的：`A[0]` 到 `A[4]` 都在它前面）。

**第一步：算距离（计算字节偏移）**

要从起点走到 `A[5]`，我们需要跨过前面的 5 个元素。

- 既然每个元素占 4 字节，一共要跨过的真实物理距离就是：$5 \times 4 = 20$ 个字节。

- 由于计算机的内存地址通常习惯用十六进制（H）表示，我们需要把十进制的 20 转成十六进制：$20 = 16 \times 1 + 4$，也就是 **`14H`**。

- **关键点：** 在变址寻址的工作机制里，这个每次都会随着下标改变而改变的偏移量 `14H`，会被保存在**变址寄存器 (IX)** 中。

**第二步：算终点（计算有效地址 EA）**

根据变址寻址的标准公式：**$EA = D + (IX)$**

- 有效地址 = 数组起点位置 + 跨过的物理距离

- $EA = 2000H + 14H = \mathbf{2014H}$

如果变址寄存器中保存 `14H`，形式地址保存 `2000H`，就能用变址寻址得到该地址。

### 例 3：相对寻址计算转移目标

**当 CPU 正在“执行”某条跳转指令并计算目的地时，它的程序计数器（PC）其实已经“悄悄”往前走了一步，指向了下一条指令的地址。**

为了让你彻底听懂，我们把这句话拆成三步来看：

#### 1. 为什么 PC 会自己悄悄往前走？

在计算机里，CPU 执行任何一条指令，都必须严格遵守以下三步流水线：

1. **取指令（Fetch）：** CPU 根据 PC 寄存器里的地址，去内存里把指令拿出来。

2. **自动加 1（Increment）：** **【关键点】指令只要一拿出来，CPU 硬件就会立刻自动把 PC 加上这条指令的长度，让它马上指向“下一条指令”的地址。**

3.  **执行指令（Execute）：** 这个时候 CPU 才开始真正研究这条指令要干嘛。

这就意味着，如果当前正在执行的是一条“相对寻址的跳转指令”，当 CPU 走到第 3 步去计算跳转目标（有效地址 EA）时，它手里的 PC 已经**不是**当前这条指令的地址了，而是**第 2 步自动更新后的“下一条指令的地址”**。

所以相对寻址的物理公式本质上是：

有效地址 (EA) =下一条指令的地址 (更新后的 PC) +位移量 D

### 2. 题目里的两种“文字游戏”

考试的题目非常喜欢在这个硬件细节上挖坑，通常会有两种出题方式：

#### 场景 A：题目直接给你“下一条指令的地址”

- **题目原话：** “若当前 PC 已经指向下一条指令地址 `1004H`，转移位移量为 `20H`……”

- **你的应对：** 既然题目给的就是“下一条”的地址，说明 CPU 已经帮你把第 2 步的“自动加 1”算好了。你直接拿来加位移量就行：EA=1004H+20H=1024H

#### 场景 B：题目故意给你“当前指令的地址”

- **题目原话：** “某相对跳转指令存放在 `1000H` 处，指令长度为 4 个字节，PC 指向当前指令，转移位移量为 `20H`……”

- **你的应对：** 此时题目给的 `1000H` 是**取指令之前**的旧 PC。你不能直接用 $1000\text{H} + 20\text{H}$！你必须模拟 CPU 的硬件动作，先手动把指令长度加进去，算出下一条指令的地址：
  
  下一条地址 =1000H+4=1004H
  
  然后再加位移量：
  
  EA=1004H+20H=1024H

## 常见考法

- 给出寻址方式、寄存器值和主存内容，求有效地址 EA 或操作数。
- 判断某条汇编指令使用了哪种寻址方式。
- 结合数组访问，判断变址寻址的有效地址。
- 结合转移指令，计算相对寻址目标地址。
- 区分基址寻址和变址寻址的用途。
- 判断直接寻址和间接寻址需要访问主存几次。
- 结合 C 语言指针、数组和 MIPS 汇编考操作数来源。

## 易错点

- **`(R)` 表示寄存器 R 的内容，不是寄存器编号本身**。
- **`M[D]` 表示主存地址 D 中的内容，不是 D 这个数**。
- **立即寻址没有有效地址 EA，因为操作数就在指令中**。
- **间接寻址比直接寻址多一次“取地址”的过程**。
- **基址和变址公式形式相似，但基址偏重程序搬迁，变址偏重数组访问**。
- **相对寻址中的 PC 是否已经自增，必须看题目说明**。
- **大端/小端影响多字节数据在连续字节地址中的排列，不改变 EA 公式本身**。

## 小结

- 寻址方式题的核心是先判断：指令字段给的是数据、地址，还是地址的地址。
- 做题时先写公式，再代数值，不要凭直觉跳步。
- 直接寻址：`EA = D`；间接寻址：`EA = M[D]`。
- 变址适合数组，基址适合重定位，相对适合转移。
- 一定区分“地址”和“地址中的内容”。
