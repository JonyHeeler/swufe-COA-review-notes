# CISC、RISC 与指令系统发展

> 来源依据：第 5 章 PPT“指令系统发展方向”“精减指令系统”“MIPS 32 & RISC-V”。

## 一句话理解

CISC 倾向于让一条指令做更多事，RISC 倾向于把复杂任务拆成多条简单、规整、容易高速执行的指令。

## 基本概念

**CISC** 是复杂指令系统计算机。它的典型思想是：提供很多功能强、形式多样的指令，让汇编程序可以用较少指令完成复杂任务。

**RISC** 是精简指令系统计算机。它的典型思想是：把指令做得简单、规整，让硬件更容易快速执行，尤其利于流水线。

用生活例子理解：

- CISC 像一个多功能机器，一键完成“切菜、炒菜、装盘”，单个按钮功能复杂。
- RISC 像一组标准动作，“切”“炒”“装盘”分开做，每个动作简单，但组合起来也能完成复杂任务。

RISC 的“精简”不是功能弱，而是指令设计更规整，把复杂功能交给多条简单指令和编译器组合完成。

## 工作原理或推导过程

### 1. CISC 为什么会出现

早期计算机中，存储器贵、容量小，编译器也不如现在强。让一条机器指令完成更多工作，可以减少程序指令条数，节省存储空间。

CISC 常见特点：

- 指令数量多。
- 指令长度可能变化。
- 指令格式多样。
- 寻址方式多。
- 单条指令可能直接访问内存并完成复杂操作。
- 控制器常适合用微程序控制实现复杂指令。

### 2. RISC 为什么强调简单规整

随着芯片技术发展，CPU 速度越来越快，流水线和编译优化变得重要。复杂指令虽然单条功能强，但不同指令执行时间差别大、译码复杂，不利于流水线稳定推进。

RISC 常见特点：

- 指令种类相对少，功能简单。
- 指令长度固定或比较规整。
- 寻址方式少。
- 通用寄存器较多。
- 采用 load/store 结构，只有装入和存储指令直接访问内存。
- 算术逻辑运算主要在寄存器之间进行。
- 控制器常适合硬布线控制。
- 利于流水线执行。

### 3. CISC 和 RISC 的核心对比

| 方面 | CISC | RISC |
| --- | --- | --- |
| 指令复杂度 | 单条指令可较复杂 | 单条指令简单 |
| 指令数量 | 多 | 相对少 |
| 指令格式 | 多样、可能变长 | 规整、常定长 |
| 寻址方式 | 多 | 少 |
| 访存方式 | 许多指令可访存 | 通常只有 load/store 访存 |
| 寄存器数量 | 相对较少或不强调 | 通常较多 |
| 控制方式 | 常见微程序控制 | 常见硬布线控制 |
| 流水线 | 较难规整 | 更容易规整 |

### 4. 为什么 RISC 利于流水线

流水线希望每条指令像工厂流水线上的产品一样，按相似步骤通过：

```text
取指 -> 译码 -> 执行 -> 访存 -> 写回
```

如果指令格式规整、长度固定、执行步骤相近，流水线控制就更简单。

RISC 的定长指令、少寻址方式、load/store 结构，正好减少了取指、译码和执行阶段的不确定性。

## 典型例子

### 例 1：内存加法的两种思路

假设要完成：

```text
M[A] = M[A] + M[B]
```

CISC 可能提供一条较复杂指令，直接从内存取两个数、相加、写回。

RISC 通常拆成多条简单指令：

```mips
lw  $t0, 0($s0)
lw  $t1, 0($s1)
add $t0, $t0, $t1
sw  $t0, 0($s0)
```

虽然指令条数多，但每条指令简单、格式规整，硬件更容易高速流水执行。

### 例 2：MIPS 和 RISC-V

MIPS 和 RISC-V 都体现 RISC 思想：

- 指令格式规整。
- 算术逻辑运算主要使用寄存器。
- 访存通过专门的 load/store 指令完成。
- 适合作为学习数据通路和流水线的例子。

### 例 3：现代处理器不是简单二分

现代高性能处理器常有融合设计。例如一些 CISC 处理器内部会把复杂指令翻译成更简单的微操作再执行。因此考试中通常考“典型特征”，不要把现实处理器机械分成绝对两类。

## 常见考法

- 判断某个描述属于 CISC 还是 RISC。
- 比较 CISC 和 RISC 的指令数量、格式、寻址方式、控制方式。
- 解释 RISC 为什么适合流水线。
- 判断 “RISC 功能弱” 或 “RISC 一定采用微程序控制” 这类说法是否正确。
- 结合 MIPS/RISC-V 说明 load/store 结构。

## 易错点

- **RISC 不是功能少，而是单条指令简单规整**。
- **RISC 程序的指令条数可能更多，但每条指令更容易快速执行**。
- **CISC 和 RISC 的边界在现代处理器中不绝对，考试看典型思想和特征**。
- **RISC 常与硬布线控制、流水线友好联系在一起，不是普遍采用微程序控制**。
- **load/store 结构表示只有取数/存数类指令访问内存，算术逻辑指令主要操作寄存器**。

## 小结

- CISC 强调单条指令表达能力，RISC 强调指令规整和执行效率。
- CISC 常见多指令格式、多寻址方式、复杂控制。
- RISC 常见定长指令、少寻址方式、多寄存器、load/store。
- RISC 适合流水线，是后续学习 MIPS 数据通路和流水线的重要基础。
