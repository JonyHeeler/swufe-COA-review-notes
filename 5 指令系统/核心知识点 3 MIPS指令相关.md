# MIPS 指令相关

> 来源依据：`知识点汇总.txt` 第五章第 3 点；第 5 章 PPT“MIPS 32”“MARS”；教材第 5 章。

## 一句话理解

MIPS 是一种典型 RISC 指令系统，它把指令格式设计得很规整，方便我们看清“汇编语句如何变成机器指令，机器指令又如何控制 CPU 执行”。

## 基本概念

MIPS 的特点可以先抓住三句话：

- **32 位定长指令**：每一条 MIPS 指令，不管它是做加法 (add) 还是读内存 (lw)，翻译成机器码后，永远不多不少刚好占据 32 个比特位 (0101)。就像所有标准集装箱大小一样，CPU 取指令的时候不用费劲猜“这条指令有多长？”。它每次闭着眼睛抓 32 个比特，就一定是一条完整的指令，这让硬件设计变得非常简单且高效。

- **load/store 结构**：如果你要做加减乘除算术运算（比如 `add`），参与运算的数据**必须**已经老老实实呆在 CPU 的寄存器里了。如果数据在主存（内存）里，你不能直接让它和寄存器里的数相加。你必须先用专门的 `Load` 指令（比如 `lw`）把它从内存“搬”到寄存器里。算完之后，如果要把结果保存回内存，必须用专门的 `Store` 指令（比如 `sw`）“搬”回去。

- **格式规整**：常见指令分为 R 型、I 型、J 型。比如，告诉你这是一条 R 型指令，那它的 32 位一定被切成了 `6-5-5-5-5-6` 这样的六个段。告诉你这是一条 I 型指令，那它一定是被切成了 `6-5-5-16` 的四个段。

在 MIPS 架构中，**寄存器（Register）**就是 CPU 内部速度最快、距离核心最近的**“超级工作口袋”**。

因为访问主存（内存）的速度太慢，CPU 无法直接在内存里做复杂的算术。它必须先把数据从内存“搬”到这些随手可得的口袋里，算完之后再搬回去。

MIPS 硬件上其实非常死板，它一共只有 **32 个通用寄存器**，编号是从 `0` 到 `31`。计算机科学家们规定了一套“江湖规矩”，给这些数字口袋起了听得懂的“外号（别名）”。下面是常见寄存器名字：

| 寄存器         | 常见用途  | 解释                                                                                                                      |
| ----------- | ----- | ----------------------------------------------------------------------------------------------------------------------- |
| `$zero`     | 恒为 0  | 计算机里经常需要用到 0（比如把某个变量清零，或者和 0 做比较）。有了它，CPU 随时随地都能掏出一个 0，不需要浪费时间去内存里找。                                                    |
| `$t0`~`$t9` | 临时寄存器 | t 代表 Temporary，就像做数学题时的草稿纸。算一个大算式（比如 (a+b)*(c-d)）时，先把 a+b 的中间结果随手扔进 \$ t 口袋里。但是调用别的函数时，别人可以随意弄脏你的 $t 口袋，回来之后里面的东西就不保准了。 |
| `$s0`~`$s7` | 保存寄存器 | s 代表 Saved，用来存放 C 语言里定义的那些长久有效的变量（比如 int f, g, h）。它的安全性极高。就算中途调用了别人的函数，别人也必须保证“原封不动”地还给你（如果对方要用，得先备份，用完给你恢复）。           |
| `$a0`~`$a3` | 函数参数  | a 代表 Argument，当你想调用另一个函数（比如叫另一个小弟帮你算个账）时，总得把“账单和现金”交给他。给他的参数，就依次放进 \$a0 到 $a3 这四个口袋里递过去。                                |
| `$v0`~`$v1` | 返回值   | v 代表 Value，小弟帮你算完账了，他要把“最终结果”交还给你。这个返回的结果，他就必须放在 \$v0 或 $v1 口袋里，你直接去这两个口袋里拿就行了。                                         |
| `$sp`       | 栈指针   | 这个一看就知道是啥吧？不然龚凯大哥是不是白教了？                                                                                                |
| `$ra`       | 返回地址  | 你的程序正在 1000H 处愉快地跑着，突然要跳转到 2000H 去执行一个函数。去之前，CPU 必须把“原路返回的地址（比如 1004H）”悄悄记在 \$ra 里。函数执行完了，看一眼 $ra，就知道该怎么走回家了。           |

三类指令格式：

| 类型  | 字段                        | 常见指令                    | 适合做什么       |
| --- | ------------------------- | ----------------------- | ----------- |
| R 型 | `op rs rt rd shamt funct` | `add`、`sub`、`and`、`sll` | 寄存器之间运算     |
| I 型 | `op rs rt immediate`      | `lw`、`sw`、`addi`、`beq`  | 访存、立即数、条件分支 |
| J 型 | `op target`               | `j`、`jal`               | 无条件跳转       |

为了方便 CPU 快速看懂，这张单子被严格划分成了几个固定的小格子（也就是**字段**）。不同的指令类型（R、I、J），就是使用了不同排版的单子。

下面我们用大白话逐一拆解这些字母缩写到底代表什么：

- **`op` (Opcode - 操作码)**： 这是 CPU 拿到指令后看的第一眼（占 6 位）。它告诉 CPU 这张单子是“算术题（R型）”、“带数字的题/访存题（I型）”还是“传送门题（J型）”。

- **`rs` (Register Source - 源寄存器 1)**：第一个“原材料”所在的口袋编号。告诉 CPU 去哪个寄存器里拿第一个参与运算的数。

- **`rt` (Register Target / Source - 目标/源寄存器 2)**：在做加减法（R型）时，它是第二个“原材料”口袋。在做读取内存（I型如 `lw`）时，它摇身一变，成了装最终战利品的“目标口袋”。

- **`rd` (Register Destination - 结果寄存器)**：只有 R 型指令才有这个字段。做完复杂的算术后，最终得出的结果就塞进这个口袋里。

#### 🛠️ R 型指令的专属微调

- **`funct` (Function - 功能码)**： R 型指令的 `op` 通常都是填 `0`，CPU 看到 0 只知道“哦，要做算术了”，但具体是加法（`add`）还是减法（`sub`）？就得靠最后的 `funct` 字段来拍板定音。

- **`shamt` (Shift Amount - 移位量)**：只有在遇到类似 `sll`（逻辑左移）这种极其特殊的指令时才有用，告诉 CPU 把数字往左/右挪几位。平时做加减法时，这个格子直接填 0。

#### 🔢 I 型和 J 型的“自带数字”

- **`immediate` (立即数)**：I 型专属（占 16 位）。有时候我们就是要加上一个常数（比如加 4），或者去内存某个固定偏移量找东西。与其去寄存器里翻，不如直接把这个数字写在指令单上，CPU 拿来就用，非常干脆。

- **`target` (目标地址)**： J 型专属。因为它不需要做算术，也不需要找三个寄存器，所以它把单子后面剩下的 26 位全部打通，用来写一个巨大的内存地址，告诉 CPU “什么都别管了，直接跳到这里去执行”。

## 工作原理或推导过程

### 1. R 型指令：寄存器和寄存器运算

所以在底层机器码层面，字段的排列顺序是：**`op` -> `rs` -> `rt` -> `rd` -> `shamt` -> `funct`**。也就是下面给出的 R 型常见格式：

```text
op(6) | rs(5) | rt(5) | rd(5) | shamt(5) | funct(6)
```

在汇编代码层面，你的书写顺序是：**操作符 -> `rd` -> `rs` -> `rt`**。例如：

```mips
add $s0, $s1, $s2
```

含义是：

```text
$s0 = $s1 + $s2
```

这里目的寄存器写在汇编的第一个位置，但机器码字段中结果寄存器是 `rd`很多同学在做题时，看到 `add $s0, $s1, $s2`，就会顺理成章地以为二进制机器码里也是按顺序排的（以为 rs=\$s0, rt=\$s1, rd=$s2），这就大错特错了。

### 2. I 型指令：立即数、访存和分支

I 型格式：

```text
op(6) | rs(5) | rt(5) | immediate(16)
```

##### `addi`

```mips
addi $t0, $t1, 4
```

含义：

```text
$t0 = $t1 + 4
```

##### `lw`**(Load Word - 取字)**：相当于 **“下载”**。

```mips
lw $t0, 8($s1)
```

等效于：

```text
$t0 = M[$s1 + 8]
```

这个动作的意思是：CPU 去内存里找一个包裹（地址是 `$s1` 里的数字加上 8），拿到包裹后，**装进** CPU 自己的口袋 `$t0` 里。

#### `sw`**(Store Word - 存字)**：相当于 **“上传/保存”**。

```mips
sw $t0, 8($s1)
```

等效于：

```text
M[$s1 + 8] = $t0
```

这个动作是 CPU 把自己口袋 `$t0` 里的东西拿出来，**塞到**内存的一个格子里（地址是 `$s1` 里的数字加上 8）。

#### `beq`(Branch on Equal - 相等则跳转)

```mips
beq $s0, $s1, label
```

含义：

```text
如果 $s0 == $s1，则跳转到 label
否则顺序执行下一条指令
```

MIPS 分支位移通常以“字”为单位存储，实际形成字节地址时要左移 2 位，并且和 PC 相关。

这是一个硬件省空间的聪明设计。在 MIPS 里，每一条指令都是死死的 32 位（也就是 4 个字节长）。如果你想往前跳过 5 条指令，真实跳过的内存物理距离其实是 $5 \times 4 = 20$ 个字节。为了省地方，CPU 在指令卡片上只写个“5”（以字为单位）。当它真要跳的时候，硬件会自动把这个数字乘以 4（也就是左移 2 位），还原出真实的物理距离 20，再去加到 PC 上。

课程题如果给出具体规则，要按题目规则算。

### 3. J 型指令：跳转

J 型格式：

```text
op(6) | target(26)
```

常见：

```mips
j target
jal function
```

`jal` 不只是跳转，还会把返回地址保存到 `$ra`，用于函数返回。

### 4. 为什么 MIPS 强调 load/store

MIPS 算术指令一般不能直接拿内存中的数相加。

如果 C 语言中有：

```c
f = g + h;
```

且 `f`、`g`、`h` 已经在寄存器 `$s0`、`$s1`、`$s2` 中，可以直接：

```mips
add $s0, $s1, $s2
```

但如果 `g` 在内存中，就要先用 `lw` 取到寄存器，再运算：

```mips
lw  $t0, 0($s1)
add $s0, $t0, $s2
```

这就是 load/store 思想：内存访问和算术运算分开。

## 典型例子

### 例 1：C 表达式翻译

C 语句：

```c
f = g + h;
```

若：

```text
f -> $s0
g -> $s1
h -> $s2
```

MIPS：

```mips
add $s0, $s1, $s2
```

意思是：

```text
R[$s0] = R[$s1] + R[$s2]
```

### 例 2：数组访问

假设数组 `A` 的基地址在 `$s1`，要取 `A[3]` 到 `$t0`，每个整数 4B。

偏移量：

```text
3 × 4 = 12
```

MIPS：

```mips
lw $t0, 12($s1)
```

含义：

```text
$t0 = M[$s1 + 12]
```

如果下标 `i` 是变量，通常要先把 `i × 4` 算出来，例如用左移 2 位实现乘 4。

### 例 3：区分 `lw` 和 `sw`

```mips
lw $t0, 0($s1)
```

是从内存读：

```text
$t0 = M[$s1 + 0]
```

```mips
sw $t0, 0($s1)
```

是写入内存：

```text
M[$s1 + 0] = $t0
```

两条指令形式很像，但数据流方向相反。

## 常见考法

- 判断一条 MIPS 指令属于 R 型、I 型还是 J 型。
- 给汇编指令，写出寄存器或内存传送含义。
- 给简单 C 表达式，翻译成 MIPS 汇编。
- 根据 `lw`、`sw`、`beq` 判断使用哪些数据通路部件。
- 判断 R 型字段 `rs`、`rt`、`rd` 的作用。
- 结合分支偏移计算跳转目标。
- 为后续 CPU 数据通路和流水线分析做准备。

## 易错点

- **`lw` 的第一个寄存器是目的寄存器，`sw` 的第一个寄存器是数据来源**。
- **MIPS 算术运算通常在寄存器之间进行，不能随便写成内存和内存直接相加**。
- **R 型指令 `op` 常为 0，具体操作看 `funct`**。
- **I 型中的 `rt` 有时是目的寄存器，有时是源寄存器，要看指令类型**。
- **数组下标要乘以元素字节数，`int` 数组常见是乘 4**。
- **分支立即数字段常不是直接字节地址，要注意是否需要左移 2 位和 PC 基准**。

## 小结

- MIPS 是典型 RISC：定长、规整、load/store。
- R 型主要做寄存器运算，I 型常做立即数、访存和分支，J 型做跳转。
- 看 `lw`/`sw` 时，一定先判断数据流方向。
- 学 MIPS 的价值在于连接汇编、机器码、数据通路和流水线。
