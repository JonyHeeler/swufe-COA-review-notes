# MIPS 指令相关

> 来源依据：`知识点汇总.txt` 第五章第 3 点；第 5 章 PPT“MIPS 32”“MARS”；教材第 5 章。

## 一句话理解

MIPS 是一种典型 RISC 指令系统，它把指令格式设计得很规整，方便我们看清“汇编语句如何变成机器指令，机器指令又如何控制 CPU 执行”。

## 基本概念

MIPS 的特点可以先抓住三句话：

- **32 位定长指令**：常见 MIPS 指令都是 32 位，便于取指和译码。
- **load/store 结构**：算术逻辑运算主要在寄存器之间进行，访问内存主要靠 `lw` 和 `sw`。
- **格式规整**：常见指令分为 R 型、I 型、J 型。

常见寄存器名字：

| 寄存器 | 常见用途 |
| --- | --- |
| `$zero` | 恒为 0 |
| `$t0`~`$t9` | 临时寄存器 |
| `$s0`~`$s7` | 保存寄存器 |
| `$a0`~`$a3` | 函数参数 |
| `$v0`~`$v1` | 返回值 |
| `$sp` | 栈指针 |
| `$ra` | 返回地址 |

三类指令格式：

| 类型 | 字段 | 常见指令 | 适合做什么 |
| --- | --- | --- | --- |
| R 型 | `op rs rt rd shamt funct` | `add`、`sub`、`and`、`sll` | 寄存器之间运算 |
| I 型 | `op rs rt immediate` | `lw`、`sw`、`addi`、`beq` | 访存、立即数、条件分支 |
| J 型 | `op target` | `j`、`jal` | 无条件跳转 |

字段含义：

- `op`：操作码，先说明大类。
- `rs`：第一个源寄存器。
- `rt`：第二个源寄存器，或 I 型中的目标/数据寄存器。
- `rd`：R 型结果寄存器。
- `shamt`：移位位数。
- `funct`：R 型中进一步说明具体操作。
- `immediate`：立即数或位移量。
- `target`：跳转目标字段。

R 型指令的 `op` 常为 0，真正区分 `add`、`sub` 等操作的是 `funct` 字段。

## 工作原理或推导过程

### 1. R 型指令：寄存器和寄存器运算

R 型常见格式：

```text
op(6) | rs(5) | rt(5) | rd(5) | shamt(5) | funct(6)
```

例如：

```mips
add $s0, $s1, $s2
```

含义是：

```text
$s0 = $s1 + $s2
```

这里目的寄存器写在汇编的第一个位置，但机器码字段中结果寄存器是 `rd`。

### 2. I 型指令：立即数、访存和分支

I 型格式：

```text
op(6) | rs(5) | rt(5) | immediate(16)
```

#### `addi`

```mips
addi $t0, $t1, 4
```

含义：

```text
$t0 = $t1 + 4
```

#### `lw`

```mips
lw $t0, 8($s1)
```

含义：

```text
$t0 = M[$s1 + 8]
```

也就是以 `$s1` 为基址，加上位移量 8，到内存取一个字放入 `$t0`。

#### `sw`

```mips
sw $t0, 8($s1)
```

含义：

```text
M[$s1 + 8] = $t0
```

注意 `sw` 的 `$t0` 不是目的寄存器，而是要写入内存的数据来源。

#### `beq`

```mips
beq $s0, $s1, label
```

含义：

```text
如果 $s0 == $s1，则跳转到 label
否则顺序执行下一条指令
```

MIPS 分支位移通常以“字”为单位存储，实际形成字节地址时要左移 2 位，并且和 PC 相关。课程题如果给出具体规则，要按题目规则算。

### 3. J 型指令：跳转

J 型格式：

```text
op(6) | target(26)
```

常见：

```mips
j target
jal function
```

`jal` 不只是跳转，还会把返回地址保存到 `$ra`，用于函数返回。

### 4. 为什么 MIPS 强调 load/store

MIPS 算术指令一般不能直接拿内存中的数相加。

如果 C 语言中有：

```c
f = g + h;
```

且 `f`、`g`、`h` 已经在寄存器 `$s0`、`$s1`、`$s2` 中，可以直接：

```mips
add $s0, $s1, $s2
```

但如果 `g` 在内存中，就要先用 `lw` 取到寄存器，再运算：

```mips
lw  $t0, 0($s1)
add $s0, $t0, $s2
```

这就是 load/store 思想：内存访问和算术运算分开。

## 典型例子

### 例 1：C 表达式翻译

C 语句：

```c
f = g + h;
```

若：

```text
f -> $s0
g -> $s1
h -> $s2
```

MIPS：

```mips
add $s0, $s1, $s2
```

意思是：

```text
R[$s0] = R[$s1] + R[$s2]
```

### 例 2：数组访问

假设数组 `A` 的基地址在 `$s1`，要取 `A[3]` 到 `$t0`，每个整数 4B。

偏移量：

```text
3 × 4 = 12
```

MIPS：

```mips
lw $t0, 12($s1)
```

含义：

```text
$t0 = M[$s1 + 12]
```

如果下标 `i` 是变量，通常要先把 `i × 4` 算出来，例如用左移 2 位实现乘 4。

### 例 3：区分 `lw` 和 `sw`

```mips
lw $t0, 0($s1)
```

是从内存读：

```text
$t0 = M[$s1 + 0]
```

```mips
sw $t0, 0($s1)
```

是写入内存：

```text
M[$s1 + 0] = $t0
```

两条指令形式很像，但数据流方向相反。

## 常见考法

- 判断一条 MIPS 指令属于 R 型、I 型还是 J 型。
- 给汇编指令，写出寄存器或内存传送含义。
- 给简单 C 表达式，翻译成 MIPS 汇编。
- 根据 `lw`、`sw`、`beq` 判断使用哪些数据通路部件。
- 判断 R 型字段 `rs`、`rt`、`rd` 的作用。
- 结合分支偏移计算跳转目标。
- 为后续 CPU 数据通路和流水线分析做准备。

## 易错点

- **`lw` 的第一个寄存器是目的寄存器，`sw` 的第一个寄存器是数据来源**。
- **MIPS 算术运算通常在寄存器之间进行，不能随便写成内存和内存直接相加**。
- **R 型指令 `op` 常为 0，具体操作看 `funct`**。
- **I 型中的 `rt` 有时是目的寄存器，有时是源寄存器，要看指令类型**。
- **数组下标要乘以元素字节数，`int` 数组常见是乘 4**。
- **分支立即数字段常不是直接字节地址，要注意是否需要左移 2 位和 PC 基准**。

## 小结

- MIPS 是典型 RISC：定长、规整、load/store。
- R 型主要做寄存器运算，I 型常做立即数、访存和分支，J 型做跳转。
- 看 `lw`/`sw` 时，一定先判断数据流方向。
- 学 MIPS 的价值在于连接汇编、机器码、数据通路和流水线。
