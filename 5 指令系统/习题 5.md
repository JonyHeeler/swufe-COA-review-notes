# 习题 5 答案

# 5.1 名词解释

### 一、 指令与架构体系基础

- **指令**：计算机执行某种基本操作的命令，是软硬件交换信息的最小单位。

- **指令系统**：一台计算机所能执行的全部指令的集合，也称为指令集体系结构（ISA），它直接决定了计算机的硬件功能。

- **CISC**：复杂指令集计算机（Complex Instruction Set Computer）。其特点是指令数量多、功能复杂、指令长度可变，试图用硬件直接实现复杂的软件功能。

- **RISC**：精简指令集计算机（Reduced Instruction Set Computer）。其特点是指令数量少、功能简单、指令长度固定，并且大量依赖通用寄存器进行操作，便于实现流水线技术。

### 二、 指令格式与组成部分

- **操作码**：指令字中的一个字段，用来指出该指令需要执行什么操作（例如加法、减法、移位等）。

- **扩展操作码**：一种在指令字长固定的情况下，通过改变操作码字段的位数来增加指令条数的设计技术（通常是利用地址码的冗余位来扩展操作码）。

- **地址码**：指令字中除操作码以外的字段，用来指出参与操作的数据所在的位置（或操作数本身），以及运算结果的存放位置。

### 三、 寻址方式与地址计算

- **寻址方式**：确定本条指令的操作数地址以及下一条将要执行的指令地址的方法。分为指令寻址和数据寻址两大类。

- **有效地址**：操作数在存储器中的真实物理地址（或逻辑地址）。它通常由指令中给出的形式地址结合具体的寻址方式计算得出。

### 四、 关键寄存器支持

- **程序计数器 PC**：用来存放下一条将要执行的指令在存储器中地址的寄存器。它具有自动加1（或加上指令字节长度）的功能。

- **基址寄存器**：用于基址寻址的寄存器。它存放一个固定的基准地址，主要面向操作系统，用于解决程序在主存中的定位和扩大寻址空间。

- **变址寄存器**：用于变址寻址的寄存器。它存放一个地址偏移量，主要面向系统用户，非常适合处理数组、字符串、循环等规律性结构的问题。

### 五、 堆栈结构与程序跳转

- **存储器堆栈**：在主存中按“先进后出”原则划出的一块专门区域来实现的堆栈。它的容量大，但读写速度相对较慢。

- **寄存器堆栈**：利用CPU内部的一组串联寄存器来实现的硬件堆栈。它的读写速度极快，但受限于硬件成本，容量通常很小。

- **转子指令**：也叫调用子程序指令。用于打断当前主程序的执行，无条件跳转到子程序去执行，并且在跳转前会自动保存主程序的断点地址（通常压入堆栈），以便子程序执行完毕后能正确返回主程序继续执行。

## 5.2 选择题答案

**（1）某计算机按字节编址，指令字长固定且只有两种指令格式，其中三地址指令29条，二地址指令107条，每个地址字段为6位，则指令字长至少应该是？**

- **答案：** A. 24位

- **解析：** 这是一道经典的**扩展操作码**题目。
  
  1. 假设指令字长为 $L$ 位。由于每个地址码占 6 位，三地址指令的地址码共占用 $3 \times 6 = 18$ 位。那么三地址指令的操作码位数为 $L - 18$ 。
  
  2. 题目说三地址指令有 29 条，所以必须满足 $2^{L-18} \ge 29$ 。如果 $L = 23$，则 $2^5 = 32 \ge 29$，刚好够用。此时，三地址操作码还剩余 $32 - 29 = 3$ 种状态。
  
  3. 将这 3 种剩余状态扩展给二地址指令（让出一个 6 位的地址码），二地址指令最多可以有 $3 \times 2^6 = 192$ 条，这完全满足题目要求的 107 条 。
  
  4. 按理说 23 位就够了，但计算机通常按字节（8位）或半字编址，指令字长一般是 8 的整数倍 。在给出的选项（24, 26, 28, 32）中，满足 $L \ge 23$ 且最符合常规设计的最小值就是 24 位 。当 $L = 24$ 时，三地址操作码 6 位（64种状态，用 29 剩 35），二地址可扩展出 $35 \times 2^6 = 2240$ 条，绰绰有余。

**（2）某计算机有16个通用寄存器，采用32位定长指令字，操作码字段(含寻址方式位)为8位，Store指令的源操作数和目的操作数分别采用寄存器直接寻址和基址寻址方式。若基址寄存器可使用任一通用寄存器，且偏移量用补码表示，则Store指令中偏移量的取值范围是？**

- **答案：** A. -32768 ~ +32767

- **解析：** 我们来算一下 32 位指令字是怎么瓜分的：
  
  1. 操作码字段占用 8 位 。
  
  2. 源操作数（寄存器直接寻址）：需要指出具体的寄存器。一共有 16 个通用寄存器，需要 $4$ 位二进制来编码（$2^4=16$） 。
  
  3. 目的操作数（基址寻址）：需要指出基址寄存器，同样因为是任一通用寄存器，需要 $4$ 位 。
  
  4. 那么留给“偏移量”的位数只剩下：$32 - 8 - 4 - 4 = 16$ 位 。
  
  5. 题目明确指出偏移量用**补码**表示 。16 位补码的表示范围是 $-2^{15}$ 到 $2^{15}-1$，也就是 $-32768 \sim +32767$ 。

**（3）某计算机采用16位定长指令字格式，操作码位数和寻址方式位数固定，指令系统中有48条指令，支持直接、间接、立即、相对4种寻址方式，单地址指令中直接寻址方式可寻址范围是？**

- **答案：** A. 0 ~ 255

- **解析：** 同样是“分蛋糕”的题目：
  
  1. 有 48 条指令，操作码至少需要 6 位，因为 $2^5=32$ 不够，$2^6=64$ 够用 。
  
  2. 有 4 种寻址方式，需要 2 位来编码表示（$2^2=4$） 。
  
  3. 由于是固定位数的格式，操作码加寻址方式共占用了 $6 + 2 = 8$ 位 。
  
  4. 指令总长 16 位，减去这 8 位，留给操作数地址码的长度为 $16 - 8 = 8$ 位 。
  
  5. 在直接寻址下，这 8 位形式地址会被直接当作无符号的主存绝对地址来用。8 位无符号数的范围是 $0 \sim 2^8-1$，即 $0 \sim 255$ 。

**（4）某指令格式如图5.33所示。其中M为寻址方式，I为变址寄存器编号，D为形式地址。若采用先变址后间址的寻址方式，则操作数的有效地址是？**

- **答案：** C. ((I)+D)

- **解析：** 重点在于理解“先变址，后间址”的顺序：
  
  1. **先变址**：将变址寄存器 I 的内容，加上指令中的形式地址 D，得到变址后的地址结果 $(I) + D$ 。
  
  2. **后间址**：以上一步计算出来的 $(I) + D$ 作为一个指针地址去访问主存，主存这个位置里存的数据，才是真正的有效地址。在符号表示法中，表示“取地址里的内容”就在外面套一层括号，因此是 $((I) + D)$ 。

**（5）某计算机字长为16位，主存按字节编址，转移指令采用相对寻址，由两个字节组成，第一字节为操作码字段，第二字节为相对位移量字段。假定取指令时，每取一个字节PC自动加1。若某转移指令所在主存地址为2000H，相对位移量字段的内容为06H，则该转移指令成功转移后的目标地址是？**

- **答案：** C. 2008H

- **解析：** 相对寻址的核心公式是：**目标地址 = PC当前值 + 偏移量** 。
  
  1. CPU 执行指令时，PC 总是指向“下一条将要执行的指令（或字节）”。
  
  2. 这条转移指令占据了主存的 `2000H`（操作码）和 `2001H`（偏移量）两个字节 。
  
  3. 题目交代“每取一个字节PC自动加1” 。当 CPU 完整地取完这条 2 个字节的指令后，PC 的值已经变成了 $2000H + 2 = 2002H$ 。
  
  4. 此时进行相对寻址计算，目标地址 = $2002H + 06H = 2008H$ 。这题特别容易误拿 2000H 直接去加，一定要记住 PC 是指取完该指令后的下一地址。

**（6）偏移寻址通过将某个寄存器内容与一个形式地址相加来生成有效地址。下列寻址方式中，不属于偏移寻址方式的是？**

- **答案：** A. 间接寻址

- **解析：** 偏移寻址是一类寻址方式的总称，它的共性是 `有效地址 = 基准地址(某个寄存器) + 偏移量(形式地址)` 。
  
  - 基址寻址：`基址寄存器 + 形式地址`，属于偏移寻址 。
  
  - 变址寻址：`变址寄存器 + 形式地址`，属于偏移寻址 。
  
  - 相对寻址：`PC（程序计数器） + 形式地址`，属于偏移寻址 。
  
  - 间接寻址：指令里的形式地址直接指向一个主存单元，该单元里存放的才是真正的有效地址，**不涉及任何寄存器相加的算术运算**，因此不属于偏移寻址 。

## 5.3 简答题答案要点

1. 指令是机器可执行命令，指令系统是全部指令集合。
2. 多种寻址方式能提高访存和程序设计灵活性。
3. 寻址方式可由专门字段表示，也可隐含在操作码中。
4. 基址寻址便于程序重定位，变址寻址便于数组访问；二者都用寄存器加位移。
5. RISC 指令少、格式规整、寄存器多、load/store、利于流水线。
6. 定长指令译码简单但编码效率低；变长指令编码紧凑但译码复杂。
7. 操作码说明操作类型，地址码说明操作数或结果位置。

## 5.4 寻址方式填空

1. 操作数在指令中：立即寻址。
2. 操作数地址在指令中：直接寻址。
3. 操作数在寄存器中：寄存器寻址。
4. 操作数地址在寄存器中：寄存器间接寻址。

## 5.5 至 5.9 计算题模板

扩展操作码题：

```text
先写出指令总位数
再扣除地址字段位数
剩余位数为当前格式可用操作码
保留未用短操作码作为下一类指令的扩展前缀
```

相对寻址题：

```text
目标地址 = 取指完成后的 PC + 位移量
```

若位移量用补码表示，先按带符号数解释，再加到 PC。

## 5.10 MIPS 机器码识别

给定机器码先按 32 位拆字段。若 `op=000000`，通常为 R 型指令，再看 `rs、rt、rd、shamt、funct`。`funct=100000B` 常对应 `add`。

## 5.11 MIPS 汇编

表达式 `f=(g+h)*i/j` 可按以下思路写：

```mips
add  $t0, $12, $13
mul  $t1, $t0, $14
div  $t1, $15
mflo $11
```

若课程限定基础 MIPS 指令，应按教材约定使用乘除相关指令和结果寄存器。

## 5.12 至 5.13 综合题要点

- MAR 位数由主存地址空间和编址单位决定。
- MDR 位数由一次存取的数据宽度决定。
- 条件转移的检测位为 1 才参与判断。
- 无符号“小于等于”通常可由借位标志 CF 或零标志 ZF 判断。
