# 汇编语言过程调用与堆栈

> 来源依据：第 5 章 PPT“过程调用实现机制”“堆栈操作”“函数调用的机器级表示”。

## 一句话理解

函数调用不是“跳过去执行”这么简单，机器必须记住从哪里回来、参数在哪里、哪些寄存器要恢复；堆栈就是管理这些临时信息的主要工具。

## 基本概念

高级语言中的函数调用：

```c
f(x);
```

在机器层面至少要解决几个问题：

- 参数怎么传给函数。
- 函数执行完后回到哪里。
- 函数返回值放在哪里。
- 调用前寄存器中的重要值如何保存。
- 局部变量放在哪里。
- 如果函数里又调用别的函数，返回地址如何不丢。

**堆栈**是一种后进先出结构：

```text
后放进去的，先取出来
```

它非常适合函数调用，因为函数调用也是嵌套的：

```text
main 调用 f
f 调用 g
g 先返回 f
f 再返回 main
```

最后调用的 `g` 最先返回，正好符合后进先出。

常见术语：

| 名称 | 含义 |
| --- | --- |
| 栈指针 SP | 指向当前栈顶位置 |
| 返回地址 | 函数执行完后要回到的指令地址 |
| 栈帧 | 一次函数调用在栈上占用的一块区域 |
| 现场保护 | 保存将来还要恢复的寄存器或状态 |
| 现场恢复 | 函数返回前把保存的信息恢复 |

## 工作原理或推导过程

### 1. 一次函数调用大致发生什么

可以按时间顺序理解：

```text
调用前：调用者准备参数，必要时保存自己还要用的寄存器
调用时：保存返回地址，跳转到被调用函数入口
函数内：建立栈帧，保存必要寄存器，分配局部变量空间
返回前：把返回值放到约定位置，恢复保存的寄存器和栈指针
返回时：跳回返回地址
```

不同体系结构细节不同，但这些基本任务相同。

### 2. 为什么一定要保存返回地址

函数调用本质上是一种跳转。

普通跳转只需要知道“跳到哪里”，但函数调用还要知道“做完后回到哪里”。如果不保存返回地址，函数执行完就不知道下一条该执行调用者的哪条指令。

在 MIPS 中：

```mips
jal function
```

通常表示：

```text
$ra = 返回地址
PC = function 的入口地址
```

函数返回时：

```mips
jr $ra
```

表示跳回 `$ra` 中保存的返回地址。

### 3. 为什么需要堆栈保存现场

如果一个函数里面又调用另一个函数，新的 `jal` 会覆盖 `$ra`。为了不丢失原来的返回地址，函数需要在调用别人之前把 `$ra` 保存到栈上。

类似地，如果函数要使用某些调用者还需要的寄存器，也应按约定保存。

一个典型栈帧可能保存：

- 返回地址 `$ra`。
- 旧的帧指针。
- 需要保护的寄存器。
- 局部变量。
- 临时空间。
- 传给下一层函数的参数。

### 4. 栈增长方向

很多体系结构中，栈向低地址增长。MIPS 教学中常见写法：

```mips
addi $sp, $sp, -8
sw   $ra, 4($sp)
sw   $s0, 0($sp)
```

含义：

```text
先把栈顶向低地址移动，腾出 8 字节空间
再把 $ra 和 $s0 保存到这块空间
```

返回前恢复：

```mips
lw   $s0, 0($sp)
lw   $ra, 4($sp)
addi $sp, $sp, 8
jr   $ra
```

注意不同体系结构的栈增长方向可能不同，考试按题目或课程约定。

## 典型例子

### 例 1：为什么嵌套调用要保存 `$ra`

假设：

```text
main 调用 f
f 调用 g
```

执行 `main -> f` 时，`$ra` 保存“从 f 返回 main 的地址”。

如果 `f` 内部直接执行：

```mips
jal g
```

那么 `$ra` 会被改成“从 g 返回 f 的地址”。原来“从 f 返回 main 的地址”就丢了。

所以 `f` 在调用 `g` 前通常要把 `$ra` 压栈保存，`g` 返回后再恢复。

### 例 2：一个简化的函数进入和返回

```mips
func:
    addi $sp, $sp, -4
    sw   $ra, 0($sp)

    # 函数主体

    lw   $ra, 0($sp)
    addi $sp, $sp, 4
    jr   $ra
```

含义：

- 进入函数时，先在栈上留空间。
- 把返回地址保存起来。
- 函数结束前恢复返回地址。
- 栈指针回到调用前的位置。
- 跳回调用者。

### 例 3：堆栈为什么适合局部变量

局部变量只在当前函数调用期间有效。函数进入时分配，函数返回时释放。堆栈正好支持这种“进入时压入、退出时弹出”的生命周期。

如果递归调用同一个函数，每次调用也会有自己的栈帧，因此每层递归的局部变量互不冲突。

## 常见考法

- 判断堆栈的后进先出特点。
- 说明函数调用为什么要保存返回地址。
- 结合 MIPS 解释 `jal`、`jr $ra`、`$sp`、`$ra` 的作用。
- 判断函数调用前后栈指针如何变化。
- 根据压栈/出栈顺序判断恢复是否正确。
- 区分过程调用的现场保护和中断处理的现场保护。

## 易错点

- **函数调用不是普通跳转，必须能返回调用点**。
- **`jal` 会改写 `$ra`，嵌套调用时不保存就可能丢返回地址**。
- **栈是后进先出，保存和恢复顺序通常相反**。
- **栈增长方向由体系结构或系统约定决定，不要脱离题目假设**。
- **返回地址和局部变量一样，本质都是需要保存的二进制信息**。
- **过程调用和中断处理都涉及现场保护，但触发原因和保存范围不同**。

## 小结

- 过程调用要解决参数、返回地址、返回值和现场保存问题。
- 堆栈适合管理嵌套调用，因为它后进先出。
- MIPS 中 `jal` 保存返回地址到 `$ra`，`jr $ra` 用于返回。
- 嵌套调用时要特别注意保存 `$ra` 和必要寄存器。
- 理解过程调用有助于后续学习数据通路、中断和异常。
