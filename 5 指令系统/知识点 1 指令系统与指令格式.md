# 指令系统与指令格式

> 来源依据：第 5 章 PPT“指令系统基本概念”“指令格式”。

## 一句话理解

指令系统规定 CPU 会哪些“动作”，指令格式规定这些动作如何写成二进制，让 CPU 能取出、识别并执行。

## 基本概念

**指令**是 CPU 能识别并执行的机器命令。程序最终都要变成一条条机器指令，CPU 才能执行。

**指令系统**是一台计算机全部机器指令的集合，也常叫 ISA 的重要组成部分。它规定：

- 有哪些操作，例如加法、取数、存数、转移。
- 有哪些寄存器可以使用。
- 支持哪些寻址方式。
- 指令如何编码。
- 程序员或编译器能看到怎样的机器级接口。

一条指令通常包含：

```text
操作码 OP：说明做什么
地址码 A：说明操作数在哪里，结果放哪里
```

例子：

```text
ADD R1, R2, R3
```

直观含义：

```text
做什么：加法
对谁做：R2 和 R3
结果放哪：R1
```

对应到机器指令中，“加法”要编码成操作码，寄存器编号要编码成地址字段或寄存器字段。

## 工作原理或推导过程

### 1. 指令执行为什么需要格式

CPU 看到的是一串二进制，例如：

```text
00000010001100101000000000100000
```

如果没有固定解释规则，CPU 不知道哪几位表示操作，哪几位表示寄存器，哪几位表示立即数。

指令格式就是这种解释规则。

例如某种格式可能规定：

```text
前 6 位：操作码
接着 5 位：源寄存器 rs
接着 5 位：源寄存器 rt
接着 5 位：目的寄存器 rd
剩余位：功能码或其他信息
```

CPU 译码时按格式拆字段，再控制数据通路执行。

### 2. 地址码数量有什么区别

按显式给出的地址码数量，指令可分为三地址、二地址、一地址和零地址。

| 类型 | 形式 | 直观含义 |
| --- | --- | --- |
| 三地址指令 | `OP A1, A2, A3` | `A1 = A2 OP A3` |
| 二地址指令 | `OP A1, A2` | `A1 = A1 OP A2` |
| 一地址指令 | `OP A1` | 常和累加器配合 |
| 零地址指令 | `OP` | 常用于堆栈，操作数隐含在栈顶 |

三地址指令表达清楚，但字段多，指令长度压力大。

二地址指令省一个地址字段，但通常会覆盖其中一个操作数。

一地址指令依赖隐含寄存器，例如累加器 AC：

```text
ADD A
```

可理解为：

```text
AC = AC + M[A]
```

零地址指令常用于堆栈机。例如栈顶两个数相加：

```text
ADD
```

操作数默认来自栈顶，不需要在指令中写地址。

### 3. 固定长度和可变长度指令

指令字长有两种常见设计。

| 类型 | 优点 | 缺点 |
| --- | --- | --- |
| 固定长度指令 | 取指、译码简单，利于流水线 | 编码空间可能浪费 |
| 可变长度指令 | 编码紧凑，复杂指令表达能力强 | 取指和译码复杂 |

RISC 指令系统通常倾向固定长度和规整格式；CISC 指令系统常见变长和多格式。

### 4. 地址码不一定是主存地址

“地址码”这个名字容易误导。它不一定总是主存地址，也可能是：

- 寄存器编号。
- 立即数。
- 位移量。
- 寻址方式中的形式地址。
- 隐含操作数的补充信息。

所以看到地址字段时，要结合寻址方式判断它真正代表什么。

## 典型例子

### 例 1：三地址和二地址表达同一计算

计算：

```text
C = A + B
```

三地址指令可以写成：

```text
ADD C, A, B
```

意思是把 A 和 B 相加，结果放 C，A 和 B 原值不变。

二地址指令可能写成：

```text
ADD A, B
```

意思是：

```text
A = A + B
```

如果最终结果必须放到 C，可能还需要额外传送指令：

```text
MOV C, A
```

这说明地址码数量会影响程序需要多少条指令。

### 例 2：一地址指令依赖累加器

若有累加器 AC，执行：

```text
LOAD A
ADD B
STORE C
```

可以理解为：

```text
AC = M[A]
AC = AC + M[B]
M[C] = AC
```

指令里只写一个地址，但另一个操作数隐含在 AC 中。

### 例 3：零地址指令依赖堆栈

堆栈机计算 `A + B` 可能是：

```text
PUSH A
PUSH B
ADD
POP C
```

其中 `ADD` 没有显式地址，因为它默认弹出栈顶两个数相加，再把结果压回栈。

## 常见考法

- 区分操作码和地址码的作用。
- 判断三地址、二地址、一地址、零地址指令的含义。
- 比较不同地址数指令的优缺点。
- 比较固定长度和可变长度指令。
- 判断地址字段是否一定表示主存地址。
- 为扩展操作码计算题打基础。

## 易错点

- **操作码说明“做什么”，地址码说明“操作数相关信息”**，不要混在一起。
- **地址码不一定是主存地址**，也可能是寄存器编号、立即数或位移量。
- **地址码少不代表不能完成运算**，可能依赖累加器、堆栈或隐含寄存器。
- **固定长度指令不等于所有指令格式完全一样**，也可能通过扩展操作码区分格式。
- **三地址指令表达能力强，但会占用更多指令位**。

## 小结

- 指令系统是 CPU 能执行的机器命令集合。
- 指令格式规定二进制字段如何解释。
- 操作码决定操作，地址码提供操作数位置或相关信息。
- 地址数越多，单条指令表达越直接，但编码空间压力越大。
- 指令格式是汇编语言、机器码和硬件数据通路之间的桥。
