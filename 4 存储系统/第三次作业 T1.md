# 第三次作业 第一题

## 原题

> 某计算机的 cache 共有 16 块，采用二路组相联映射方式（即每组 2 块）。每个主存块大小为 32B，按字节编址。主存 129 号单元所在主存块应装入的 cache 组号是多少？

## 涉及知识点与公式

这道题主要涉及两个文件中的知识。

第一，来自 **`核心知识点 1 主存按字节编址与容量计算.md`** 的“按字节编址”：

按字节编址表示 **1 个地址对应 1 个字节**，所以“主存 129 号单元”不是“第 129 个主存块”，而是“地址编号为 129 的那个字节”。如果每个主存块大小为 `B` 字节，那么地址 `A` 所在的主存块号为 `主存块号 = ⌊A / B⌋`，其中 `⌊ ⌋` 表示向下取整；块内偏移为 `块内偏移 = A mod B`。

第二，来自 **`核心知识点 2 Cache与主存映射替换写策略.md`** 的“组相联映射”：`k` 路组相联的意思是 **每组有 k 个 Cache 块**。因此 `Cache 组数 = Cache 总块数 / 相联路数`。在组相联映射中，主存块不能随便放到任意组，而是只能放到它对应的那一组；对应关系用公式 `Cache 组号 = 主存块号 mod Cache 组数` 计算。这里的 `mod` 是取余数，它的作用可以理解为“循环分配”：如果 Cache 有 8 组，那么主存块 0、8、16……都会回到第 0 组，主存块 1、9、17……都会回到第 1 组。

## 解题思路与答案

先明确题目中的几个量：Cache 共有 `16` 块，二路组相联表示每组 `2` 块，所以 Cache 被分成 `16 / 2 = 8` 组。这个地方最容易错：**“二路”说的是一组里面有两个位置，不是说只有两组**。

接着看“主存 129 号单元”，因为题目说明按字节编址，所以 129 是一个字节地址。每个主存块大小是 `32B`，也就是每 `32` 个连续字节组成一个主存块。

主存块的划分是这样的：第 0 块包含地址 `0~31`，第 1 块包含地址 `32~63`，第 2 块包含地址 `64~95`，第 3 块包含地址 `96~127`，第 4 块包含地址 `128~159`。因此地址 129 显然落在第 4 块中。用公式计算就是 `主存块号 = ⌊129 / 32⌋ = 4`，因为 `129 = 32 × 4 + 1`，商 `4` 表示第 4 个完整的 32B 块，余数 `1` 表示它在这个块内的偏移是 1。

最后根据组相联映射公式，`Cache 组号 = 主存块号 mod Cache 组数 = 4 mod 8 = 4`，所以答案是：**主存 129 号单元所在主存块应装入 Cache 第 4 组**。如果题目默认 Cache 组号从 0 开始编号，答案写 `4 号组`；计算机组成原理题目通常默认从 0 开始。如果某些教材或题目特别规定从 1 开始编号，那么对应写成第 5 组，但这不是本题的一般默认答案。

## 为什么不能直接用 129 取模

这道题的关键是要先把“字节地址”转换成“主存块号”。

Cache 和主存之间交换数据时，通常不是一次搬一个字节，而是一次搬一个主存块。题目问“129 号单元所在主存块应装入哪一组”，真正进入 Cache 的是包含地址 129 的整个主存块，也就是第 4 号主存块。

因此计算顺序必须是 `地址 → 主存块号 → Cache 组号`，也就是 `⌊129/32⌋ = 4`，然后 `4 mod 8 = 4`。如果直接算 `129 mod 8 = 1`，相当于把“字节编号”当成“主存块编号”，这就错在单位不一致：129 的单位是“字节地址”，8 的单位是“Cache 组数”，二者不能直接用于映射公式。

## 同类题通用模板

遇到这类题，先判断题目给的是“主存地址”还是“主存块号”。

如果给的是地址，例如本题的 129，就必须先算 `主存块号 = ⌊主存地址 / 主存块大小⌋`；如果题目直接说“第 20 号主存块”，那就不需要再除以块大小。

接着看映射方式：

直接相联用 `Cache 行号 = 主存块号 mod Cache 行数`；

组相联先算 `Cache 组数 = Cache 总块数 / 相联路数`，再算 `Cache 组号 = 主存块号 mod Cache 组数`；

全相联则不需要算组号，因为可以放入任意 Cache 行。
