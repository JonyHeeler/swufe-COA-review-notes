# 第 4 章 存储系统复习导读

## 本章定位

第 4 章研究“数据放在哪里、怎样找到、怎样让 CPU 更快访问”。它把主存、cache、虚拟存储器串成一个层次化系统，是考试中计算题最集中的章节之一。

## 核心优先级

| 优先级 | 内容 | 复习要求 |
| --- | --- | --- |
| 核心 | 主存按字节编址 | 会算地址位数、容量、芯片数量 |
| 核心 | cache 与主存交互 | 会做地址划分、映射、命中率、替换算法 |
| 核心 | 虚拟存储器 | 会做 VPN/VPO/PPN/PPO 划分和虚实地址转换 |
| PPT 补充 | SRAM、DRAM、ROM、Flash | 理解特点、刷新、易失性 |
| PPT 补充 | 存储器层次与扩展 | 会解释层次结构和位/字扩展 |

## 复习顺序

1. 先掌握容量与地址位数计算。
2. 再学 SRAM、DRAM、ROM 的区别。
3. 然后重点练 cache 地址划分。
4. 最后把 TLB、页表、cache 串起来做虚实地址转换题。

## 易错提醒

- 按字节编址时，地址数等于字节数。
- cache 块内偏移看块大小，组索引看组数，剩余才是 tag。
- TLB 命中意味着页在主存中，不能再出现缺页。
- 写回法需要脏位，写穿法会增加主存写流量。
