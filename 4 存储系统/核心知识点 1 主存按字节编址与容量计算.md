# 主存按字节编址与容量计算

> 来源依据：`知识点汇总.txt` 第四章第 1 点；第 4 章 PPT“主存中数据组织”“存储器扩展”；教材第 4 章。

## 一句话理解

主存编址就是给存储空间贴“门牌号”：按字节编址时，一个门牌号对应 1 个字节，所以容量有多少字节，就需要能表示多少个地址。

## 基本概念

学习这一节最容易混的是三个量：**地址、数据、容量**。

- **地址**：告诉 CPU 要访问“哪一个位置”。地址像门牌号，只负责定位。
- **数据**：某个地址中真正存放的内容。数据像房间里的东西。
- **容量**：一共有多少个可存放数据的位置，以及每个位置能放多少位。

主存可以按不同单位编址。

| 编址方式  | 一个地址对应什么       | 做题时怎么想             |
| ----- | -------------- | ------------------ |
| 按字节编址 | 1 个地址对应 1B     | 地址个数 = 字节数         |
| 按字编址  | 1 个地址对应 1 个机器字 | 地址个数 = 字节数 / 每字字节数 |

例如一台机器按字节编址，地址 `1000H` 指向的是某一个字节；如果要读一个 32 位整数，CPU 通常会从这个字节开始连续取 4 个字节。

再看两个硬件寄存器：

- **MAR，存储器地址寄存器**：保存要访问的地址。MAR 有多少位，就最多能指出多少个地址。
- **MDR，存储器数据寄存器**：MDR 的位数决定了 CPU 一次访存能读写多少位的数据。它通常与计算机的**字长**相等。

所以：

- **MAR 位数决定可寻址空间大小**。
- **MDR 位数决定一次传送的数据宽度**。

这两个量不是一回事。很多题会故意把它们放在一起考。

## 工作原理或推导过程

### 1. 为什么 n 位地址能表示 `2^n` 个位置

二进制每一位只有 0 和 1 两种可能。

```text
1 位地址：0, 1                         共 2 个地址
2 位地址：00, 01, 10, 11               共 4 个地址
3 位地址：000 到 111                   共 8 个地址
n 位地址：                              共 2^n 个地址
```

因此，只要知道“需要多少个地址”，就能反推地址位数。

```text
地址个数 = 2^地址位数
地址位数 = log2(地址个数)
```

### 2. 按字节编址怎么求地址位数

按字节编址时，一个地址对应 1B，所以：

```text
地址个数 = 容量字节数
容量 = 2^地址位数 B
```

例：主存容量为 8MB，按字节编址。

```text
8MB = 8 × 2^20B = 2^3 × 2^20B = 2^23B
```

需要能表示 `2^23` 个字节地址，所以地址位数为 23 位。

### 3. 按字编址怎么求地址位数

按字编址时，一个地址不是指向 1B，而是指向 1 个字。此时必须先算“有多少个字”。

```text
地址个数 = 总字节数 / 每字字节数
```

例：32KB 存储器，按 16 位字编址。

```text
32KB = 32 × 2^10B = 2^15B
16 位 = 2B
字数 = 2^15B / 2B = 2^14 个字
```

所以地址寄存器需要 14 位。每个字有 16 位，所以数据寄存器通常需要 16 位。

### 4. 芯片规格 `4K×8` 到底是什么意思

存储芯片常写成：

```text
字数 × 字长
```

例如 `4K×8` 表示：

- 芯片内部有 `4K = 2^12` 个可寻址单元。
- 每个单元有 8 位，也就是 1B。
- 芯片总容量为 `4K × 8bit = 4KB`。

它不是“4K 个字节再乘 8 字节”，而是“4K 个单元，每个单元 8 位”。

## 典型例子

### 例 1：按字节编址求地址线

某主存容量为 64KB，按字节编址，问地址线至少多少根？

```text
64KB = 64 × 2^10B = 2^6 × 2^10B = 2^16B
```

按字节编址时地址个数等于字节数，所以需要 `2^16` 个地址。

```text
地址线数 = 16
```

### 例 2：按字编址求 MAR 和 MDR

某存储器容量为 1MB，字长 32 位，按字编址，问 MAR 和 MDR 至少多少位？

先统一单位：

```text
1MB = 2^20B
32 位 = 4B
字数 = 2^20B / 4B = 2^18
```

所以：

```text
MAR = 18 位
MDR = 32 位
```

这里 MAR 不是 20 位，因为题目不是按字节编址，而是按 32 位字编址。

### 例 3：芯片容量判断

`8K×16` 芯片容量是多少？

```text
8K = 8 × 2^10 = 2^13 个单元
每个单元 16 位 = 2B
总容量 = 2^13 × 2B = 2^14B = 16KB
```

## 常见考法

- 已知主存容量和编址单位，求地址位数。
- 已知 MAR 位数和编址单位，求最大可寻址容量。
- 已知容量和字长，求 MAR、MDR 位数。
- 已知芯片规格，判断单片容量或需要多少芯片。
- 区分地址线数量、数据线数量和容量大小。

## 易错点

- **不要把“地址位数”和“数据位数”混在一起**：地址位数负责找位置，数据位数负责传数据。
- **KB、MB 在计算机组成原理中通常按 2 的幂计算**：`1KB = 2^10B`，`1MB = 2^20B`。
- **按字编址时，地址个数不是字节数**：要先除以每个字占多少字节。
- **芯片规格 `4K×8` 中的 8 是 bit，不是 byte**。
- **DRAM 地址引脚可能采用行列复用**：实际芯片引脚数可能少于逻辑地址位数，不能简单等同。

## 小结

- 做容量题先问自己：一个地址对应 1B，还是对应 1 个字？
- 按字节编址：`地址个数 = 字节数`。
- 按字编址：`地址个数 = 总字节数 / 每字字节数`。
- `n` 位地址最多表示 `2^n` 个地址。
- MAR 看地址位数，MDR 看一次传送的数据位数。
