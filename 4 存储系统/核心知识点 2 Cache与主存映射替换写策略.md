# Cache 与主存映射、替换和写策略

> 来源依据：`知识点汇总.txt` 第四章第 2 点；第 4 章 PPT“高速缓冲存储器”“cache 与主存交互”；教材第 4 章。

## 一句话理解

Cache 是放在 CPU 和主存之间的“小而快”的临时仓库，考试最核心的问题是：主存中的某一块数据应该放到 Cache 的哪里，找它时又怎样判断它在不在。

## 基本概念

CPU 访问主存很慢，但程序经常反复访问最近用过的数据，或者访问相邻的数据。Cache 就利用这种局部性，把近期可能用到的数据块提前放到离 CPU 更近的地方。

先区分几个基本词：

- **主存块**：主存按固定大小切成的块，例如每块 64B。
- **Cache 行**：Cache 中能存放一个主存块的位置，也常叫 Cache 块。
- **块内偏移 offset**：一个块内部第几个字节。
- **标记 tag**：用来判断 Cache 行里装的是主存中的哪一块。
- **有效位 valid bit**：表示这一行当前内容是否有效。
- **脏位 dirty bit**：写回法中使用，表示这一行被改过但还没有写回主存。

可以把主存想象成一个大图书馆，Cache 是书桌。CPU 不会一次只拿书上的一个字，而是把一整页或一整段拿到书桌上。下次如果还要看这一段，就不用再跑去图书馆。

Cache 访问有两种结果：

- **命中**：要访问的数据已经在 Cache 中。
- **缺失**：要访问的数据不在 Cache 中，需要从主存取来。

## 工作原理或推导过程

### 1. 为什么地址要拆成 tag、index、offset

一条主存地址本质上回答三个问题：

```text
1. 要访问块内哪个字节？         offset
2. 应该去 Cache 的哪一行/哪一组找？ index
3. 找到的位置里是不是我要的主存块？ tag
```

所以组相联 Cache 中，地址通常划分为：

```text
tag | 组索引 index | 块内偏移 offset
```

三段位数的计算顺序固定：

```text
offset 位数 = log2(块大小字节数)
Cache 行数 = Cache 数据容量 / 块大小
组数 = Cache 行数 / 相联路数
index 位数 = log2(组数)
tag 位数 = 主存地址位数 - index 位数 - offset 位数
```

其中“相联路数”就是每组有几行。二路组相联表示每组 2 行，不是有 2 组。

### 2. 三种映射方式怎么理解

Cache 映射方式规定“主存块能放到 Cache 的哪些位置”。

| 映射方式 | 主存块能放哪里 | 优点 | 缺点 |
| --- | --- | --- | --- |
| 直接相联 | 只能放到唯一一行 | 硬件简单、查找快 | 容易冲突 |
| 全相联 | 可以放到任意一行 | 冲突少 | 比较器多、硬件复杂 |
| 组相联 | 只能放到某一组，组内任意行 | 折中 | 比直接相联复杂 |

直接相联的行号计算：

```text
Cache 行号 = 主存块号 mod Cache 行数
```

组相联的组号计算：

```text
Cache 组号 = 主存块号 mod Cache 组数
```

全相联没有 index 字段，因为主存块可以放到任何 Cache 行，查找时要比较所有行的 tag。

### 3. 替换算法什么时候才需要

替换算法不是每次访问都用。

只有在以下条件同时满足时才需要替换：

- 发生 Cache 缺失。
- 新主存块要放入的目标位置已经满了。
- 目标位置中有多个候选行可以替换。

例如直接相联中，每个主存块只有唯一位置，没得选，严格来说不需要在多行中“选择”。组相联和全相联才更常考替换算法。

常见算法：

| 算法 | 直观含义 |
| --- | --- |
| FIFO | 最早进入 Cache 的先被换出 |
| LRU | 最久没有被访问的先被换出 |
| LFU | 访问次数最少的先被换出 |
| 随机替换 | 随机选一行换出 |

考试中 LRU 最常见。做 LRU 题不要凭感觉，要维护每一组内“最近使用顺序”。

### 4. 写策略解决什么问题

读操作比较简单：Cache 没有就去主存取。

写操作复杂，因为 CPU 改了 Cache 中的数据后，主存中的旧数据要不要马上改？

| 写策略 | 做法 | 优点 | 缺点 |
| --- | --- | --- | --- |
| 写穿法 write-through | 写 Cache 的同时写主存 | 主存总是最新 | 写主存次数多 |
| 写回法 write-back | 先只写 Cache，替换时再写回主存 | 减少主存写流量 | 需要脏位，控制更复杂 |

写缺失时还可能配合：

- **写分配**：写缺失时把块调入 Cache 再写。
- **非写分配**：写缺失时直接写主存，不调入 Cache。

课程考试通常重点比较写穿和写回，尤其要记住：**写回法需要脏位**。

## 典型例子

### 例 1：地址字段划分

某机器主存地址 32 位，Cache 数据容量 16KB，块大小 64B，采用 4 路组相联。求 tag、index、offset 位数。

先求块内偏移：

```text
块大小 = 64B = 2^6B
offset = 6 位
```

再求 Cache 行数：

```text
Cache 行数 = 16KB / 64B
          = 2^14B / 2^6B
          = 2^8 行 = 256 行
```

4 路组相联表示每组 4 行：

```text
组数 = 256 / 4 = 64 = 2^6
index = 6 位
```

最后剩下的是 tag：

```text
tag = 32 - 6 - 6 = 20 位
```

所以地址划分为：

```text
tag 20 位 | index 6 位 | offset 6 位
```

### 例 2：组号计算

Cache 有 16 行，二路组相联，主存块号为 20，问映射到哪一组。

```text
组数 = 16 / 2 = 8 组
组号 = 20 mod 8 = 4
```

所以主存块 20 只能放在第 4 组，但在第 4 组内部可以放任意一路。

### 例 3：为什么会发生冲突

直接相联 Cache 有 8 行，主存块 0 和主存块 8 的行号都是：

```text
0 mod 8 = 0
8 mod 8 = 0
```

它们都只能放第 0 行。如果程序反复访问块 0、块 8，即使 Cache 还有其他空行，也可能不断互相替换。这就是直接相联的冲突问题。

## 常见考法

- 给 Cache 参数，求 tag、index、offset 位数。
- 给主存块号，判断映射到哪一行或哪一组。
- 给一串访问序列，模拟直接相联、组相联或全相联命中情况。
- 用 LRU/FIFO 判断替换哪一块。
- 计算命中率、缺失率或平均访问时间。
- 比较写穿法和写回法，判断是否需要脏位。
- 计算 Cache 总位数，注意 tag、有效位、脏位等附加位。

平均访问时间常见表达为：

```text
平均访问时间 = 命中时间 + 缺失率 × 缺失代价
```

它说明 Cache 的意义不只是“命中时快”，还要看缺失时要付出多大代价。

## 易错点

- **offset 看块大小，不看 Cache 总容量**。
- **index 看组数，不看主存块数**。
- **二路组相联表示每组 2 行，不是 2 组**。
- **tag 不是随便取高位，而是地址去掉 index 和 offset 后剩下的高位**。
- **替换算法只在目标组满时才需要**。
- **写回法需要脏位，写穿法通常不需要脏位**。
- **Cache 对程序员透明**：程序正常写地址，不需要知道数据是否命中 Cache。

## 小结

- Cache 题先把地址拆成 `tag | index | offset`。
- 直接相联看行号，组相联看组号，全相联只比较 tag。
- `offset = log2(块大小)`，`index = log2(组数)`，剩下才是 tag。
- LRU 题要按组维护最近使用顺序。
- 写策略重点记住：写穿立即写主存，写回替换时才写主存。
