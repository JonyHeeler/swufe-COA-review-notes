# 存储器扩展与多体交叉存储器

> 来源依据：第 4 章 PPT“存储器扩展”“字长扩展”“字数扩展”“多模块存储器”。

## 一句话理解

存储器扩展就是用多片小芯片拼出 CPU 需要的“容量”和“字长”；多体交叉存储器则是把连续地址分散到多个存储体，让它们轮流工作，提高连续访问速度。

## 基本概念

先看芯片规格：

```text
4K×8
```

含义是：

- 有 `4K` 个可寻址单元。
- 每个单元宽度是 `8` 位。
- 总容量是 `4K × 8bit = 4KB`。

如果目标存储器比单片芯片更宽或更大，就需要扩展。

| 扩展类型 | 解决什么问题 | 怎么连接 |
| --- | --- | --- |
| 位扩展/字长扩展 | 每个地址要读写更多位 | 多片芯片并联数据线 |
| 字扩展/字数扩展 | 需要更多地址单元 | 用高位地址译码产生片选 |
| 字位同时扩展 | 字长和字数都不够 | 先并联成一组，再扩展多组 |

多体交叉存储器中的“体”可以理解为多个可并行工作的存储模块。它不是让单个存储体变快，而是让多个存储体错开工作。

## 工作原理或推导过程

### 1. 存储器扩展先看两个方向

做扩展题时不要直接猜芯片数，按两个方向算：

```text
字长方向：目标字长 / 单片字长 = 每组并联芯片数
字数方向：目标字数 / 单片字数 = 需要多少组
总芯片数 = 每组并联芯片数 × 组数
```

为什么这样算？

- 字长不够，说明一次读出的位数不够，要让多片芯片同时输出不同的数据位。
- 字数不够，说明地址范围不够，要让不同芯片组负责不同地址段。

### 2. 位扩展怎么连接

例：用两片 `4K×4` 组成 `4K×8`。

思路：

- 两片芯片的地址线接同一组地址，因为它们响应同一个地址。
- 两片芯片同时被选中。
- 一片提供低 4 位，另一片提供高 4 位。

直观图示：

```text
同一个地址 -> 芯片 1 输出 D0~D3
同一个地址 -> 芯片 2 输出 D4~D7
合起来就是 8 位数据
```

这叫字长扩展，因为地址个数没变，单个地址的数据位数变宽了。

### 3. 字扩展怎么连接

例：用两片 `4K×8` 组成 `8K×8`。

思路：

- 每片芯片内部仍然只能表示 `4K = 2^12` 个地址，所以低 12 位地址接到两片芯片的地址输入。
- 还需要 1 位高位地址决定选哪一片。

```text
高位地址 = 0 -> 选第 1 片，地址范围 0 ~ 4K-1
高位地址 = 1 -> 选第 2 片，地址范围 4K ~ 8K-1
```

这叫字数扩展，因为每个地址的数据宽度没变，但地址范围变大了。

### 4. 多体交叉存储器为什么能加速

如果只有一个存储体，连续读多个字时，只能等一个访问完成再开始下一个。

多体交叉把地址分散到多个存储体，常见低位交叉方式：

```text
体号 = 地址低位
体内地址 = 地址剩余高位
```

例如 4 体低位交叉：

```text
地址 0 -> 体 0
地址 1 -> 体 1
地址 2 -> 体 2
地址 3 -> 体 3
地址 4 -> 体 0
地址 5 -> 体 1
```

这样连续地址会落到不同存储体中。访问地址 0 后，不必等体 0 完全空闲才能启动地址 1，因为地址 1 在体 1 中。

这提高的是连续访问的吞吐率，不是单个存储体的访问周期。

## 典型例子

### 例 1：用 `4K×4` 组成 `16K×8`

目标：`16K×8`

芯片：`4K×4`

先算字长方向：

```text
目标字长 / 单片字长 = 8 / 4 = 2
```

每组需要 2 片并联，才能一次输出 8 位。

再算字数方向：

```text
目标字数 / 单片字数 = 16K / 4K = 4
```

需要 4 组，负责 4 个地址段。

总芯片数：

```text
2 × 4 = 8 片
```

连接理解：

- 每组 2 片并联，形成 `4K×8`。
- 4 组再通过片选扩展成 `16K×8`。

### 例 2：地址线怎么分配

仍然是 `4K×4` 组成 `16K×8`。

单片 `4K = 2^12`，所以每片需要 12 根片内地址线。

目标 `16K = 2^14`，系统需要 14 根地址线。

因此：

```text
A0~A11：送到每片芯片内部，选择片内单元
A12~A13：送入译码器，产生 4 个片选信号
```

这就是“低位寻址，高位片选”的典型做法。

### 例 3：4 体低位交叉判断体号

若采用 4 体低位交叉，问地址 13 在哪个体。

4 体说明体号可以用地址除以 4 的余数判断：

```text
13 mod 4 = 1
```

所以地址 13 在 1 号体。

连续地址 12、13、14、15 分别在 0、1、2、3 号体，适合连续读写。

## 常见考法

- 给芯片规格和目标存储器规格，求芯片数量。
- 判断需要几片并联、几组扩展。
- 分配地址线、数据线和片选信号。
- 根据地址范围判断选中哪一组芯片。
- 判断多体交叉存储器中某地址属于哪个存储体。
- 解释低位交叉为什么适合连续访问。
- 结合 DRAM 计算刷新间隔或刷新周期。

## 易错点

- **字长扩展靠数据线并联，字数扩展靠地址译码和片选**。
- **芯片规格中的 `×8` 是每个单元 8 位，不是 8 个地址**。
- **总芯片数要同时考虑字长方向和字数方向**。
- **高位地址通常用于片选，低位地址通常用于片内寻址**。
- **多体交叉提高连续访问吞吐率，不会缩短单个存储体的访问周期**。
- **低位交叉适合连续地址，高位交叉更像把地址空间分成连续大段**。

## 小结

- 存储器扩展题先拆成“字长够不够”和“字数够不够”。
- `每组芯片数 = 目标字长 / 单片字长`。
- `组数 = 目标字数 / 单片字数`。
- `总芯片数 = 每组芯片数 × 组数`。
- 多体交叉通过多个存储体并行错开工作，提高连续访问效率。
