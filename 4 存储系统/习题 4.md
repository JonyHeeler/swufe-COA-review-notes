## 4.1 名词解释

#### 一、 存储器基础性能与数据存放

这组名词是衡量存储器好坏以及数据在物理介质中如何摆放的基础概念 。

- **存取时间**：从启动一次存储器操作（读或写）到完成该操作所经历的时间。

- **存取周期**：存储器进行连续两次独立的读（或写）操作所需的最小时间间隔。通常存取周期大于存取时间。

- **存储器带宽**：单位时间内存储器能够提供或接收的最大数据量（通常以字节/秒或位/秒衡量）。

- **边界对齐的数据存放**：指多字节数据在存储器中存放时，其起始地址必须是该数据长度（如双字、字、半字）的整数倍，以此来减少 CPU 访问主存的次数。

- **大端存储**：多字节数据的高位字节存放在存储器的低地址处，低位字节存放在高地址处。

- **小端存储**：多字节数据的低位字节存放在存储器的低地址处，高位字节存放在高地址处。

#### 二、 主存的构成与扩展提速

这组名词讲述了主存（内存）的物理元件差异，以及当单块芯片容量或速度不够时，如何进行系统级别的扩展与优化 。

- **静态存储器 (SRAM)**：利用双稳态触发器保存信息，速度极快但集成度低、功耗大，只要不断电信息就不会丢失，常用于 Cache。

- **动态存储器 (DRAM)**：利用栅极电容上的电荷来保存信息，集成度高、成本低但速度较慢，常用于主存。

- **刷新 / 刷新周期**：由于 DRAM 的电容会随时间漏电，必须在电荷流失前重新读出并写入原数据，这个过程称为刷新。两次刷新之间的最大时间间隔即为刷新周期。

- **字扩展 / 位扩展**：当单片存储芯片的容量不满足需求时，通过增加芯片数量来扩充存储字的数量（字扩展）或每个存储字的位数（位扩展）。

- **多体交叉存储器**：将主存分成多个相互独立、容量相同的存储体，并通过交叉编址的方式让 CPU 在不同存储体间进行流水线式的高速并发访问，以提高主存带宽。

- **双端口存储器**：具有两组相互独立的读写控制电路、地址线和数据线的存储器，允许两个独立的控制器同时异步地访问存储单元。

#### 三、 Cache 高速缓存机制

主存速度跟不上 CPU 时，需要引入 Cache。这组名词构成了 Cache 运行的核心逻辑 。

- **时间局部性**：如果一个数据项正在被访问，那么在近期它很可能还会被再次访问。这是 Cache 能够起作用的理论基石。

- **高速缓冲存储器 (Cache)**：位于 CPU 和主存之间的小容量、超高速 SRAM 存储器，用于存放主存中活跃的数据块。

- **命中率**：CPU 要访问的数据刚好能在 Cache 中直接找到的概率，是衡量 Cache 性能的最核心指标。

- **相联存储器**：一种不按照地址而是“按内容寻址”的存储器，在 Cache 的标记查找和 TLB 中广泛使用。

- **地址映射 / 直接相联映射 / 全相联映射 / 组相联映射**：主存块调入 Cache 时的位置分配规则。**直接映射**最死板（多对一），**全相联**最灵活（任意放），**组相联**是两者的折中（主存块只能放入特定的组，但在组内可以随便放）。

- **LRU算法 / LFU算法**：Cache 空间满了之后的替换策略。LRU 淘汰“近期最少使用”的块，LFU 淘汰“访问频次最低”的块。

- **cache一致性 / 写回法 / 写穿法**：CPU 修改 Cache 数据后，如何保证与主存数据一致。**写回法**只修改 Cache，等该块被替换时才写回主存；**写穿法**则是每次修改 Cache 时都同步写入主存。

## 4.2 选择题

**第(1)题** **正确答案：A**

- **解析**：
  
  - **I 正确**：RAM（随机存取存储器）掉电会丢失数据，属于易失性存储器；ROM（只读存储器）掉电不丢失数据，属于非易失性存储器 。
  
  - **II 正确**：RAM 和 ROM 的访问时间都与存储单元的物理位置无关，因此它们均采用随机存取方式进行信息访问 。
  
  - **III 错误**：Cache 对读写速度的要求极高且需要频繁擦写，通常由速度极快的 SRAM 构成；ROM 无法进行快速、频繁的写入操作，所以不能用作 Cache 。
  
  - **IV 错误**：只有 DRAM（动态随机存取存储器）需要通过定期刷新来防止电容电荷泄漏，SRAM 和 ROM 都不需要进行刷新操作 。

**第(2)题** **正确答案：A**

- **解析**：
  
  - 该 DRAM 芯片的容量为 4M × 8 位。由于数据线宽度为 8 位，因此需要 **8** 个数据引脚 。
  
  - 芯片的地址空间为 4M（即 2²²），逻辑上需要 22 位地址 。由于 DRAM 普遍采用行、列地址复用技术（分两次传送地址），地址引脚数量可减半，因此需要 22 / 2 = **11** 个地址引脚 。
  
  - 总引脚数 = 地址引脚 + 数据引脚 = 11 + 8 = **19** 。

**第(3)题** **正确答案：D**

- **解析**：
  
  - 主存总容量为 64KB，且按字节编址（即 64K × 8 位） 。
  
  - **计算 ROM 芯片数**：ROM 区的容量为 4KB（4K × 8 位）。选用 2KB × 8 位的 ROM 芯片，需要的芯片数量为：(4K × 8) / (2K × 8) = **2** 片 。
  
  - **计算 RAM 芯片数**：RAM 区的剩余容量为 64KB - 4KB = 60KB（即 60K × 8 位）。选用 4KB × 4 位的 RAM 芯片，这里需要进行字和位的双重扩展。所需的 RAM 芯片数量为：(60K × 8) / (4K × 4) = 15 × 2 = **30** 片 。
  
  - 综上，需要的 ROM 和 RAM 芯片数分别是 2 和 30 。

**第(4)题** **正确答案：D**

- **解析**：
  
  - 目标存储器规格为 8K × 8 位，而使用的基础芯片为 2K × 4 位 。构建时，首先需要两片芯片并联组成 2K × 8 位的模块（位扩展），然后再用 4 组这样的模块进行串联（8K / 2K = 4），以覆盖完整的 8K 地址空间 。
  
  - 这 4 组模块负责的地址范围（每组 2K 大小，即 0800H 长度）依次为：
    
    - 第 0 组：0000H ~ 07FFH
    
    - 第 1 组：0800H ~ 0FFFH
    
    - 第 2 组：1000H ~ 17FFH
    
    - 第 3 组：1800H ~ 1FFFH
  
  - 已知目标地址 0B1FH 恰好落在第 1 组的范围内，而该组芯片负责的最小（起始）地址就是 **0800H** 。

**第(5)题** **正确答案：C**

- **解析**：
  
  - 题目中提到的 2K × 1 位（实为 2048 个存储单元）的 DRAM 芯片，其存储阵列单元总数为 $r \times c = 2048 = 2^{11}$ 。
  
  - **保证地址引脚数最少**：DRAM 采用行列地址复用机制，复用引脚数取决于行地址和列地址位数中的较大值。为了让该最大值尽可能小，行、列地址的位数应尽可能接近（即 5 位和 6 位，复用引脚数为 6），对应的是 32 × 64 或 64 × 32 的排列组合 。
  
  - **尽量减少刷新开销**：DRAM 的刷新是**按行进行**的，刷新一遍所耗费的周期数等同于行数 $r$。在容量确定的前提下，行数 $r$ 越小，需要刷新的行数就越少，相应的刷新开销也就越低 。因此，应当选择行数更小的方案，即 $r = 32$，$c = 64$ 。

**第(6)题** **正确答案：B**

- **解析**：
  
  - **总线带宽计算公式**：$\text{带宽} = \text{工作频率} \times (\text{总线宽度} / 8) \times \text{通道数}$。
  
  - 题目中给出的 DDR3-1333 内存，其等效工作频率即为数据传输率，为 $1333\text{ MHz}$（即每秒进行 $1333 \times 10^6$ 次数据传输） 。
  
  - 单通道带宽 = $1333\text{ MHz} \times (64\text{ 位} / 8\text{ 字节/位}) = 1333 \times 8 = 10664\text{ MB/s} \approx 10.66\text{ GB/s}$。
  
  - 该计算机采用 3 通道结构，因此总线总带宽 = $10.66\text{ GB/s} \times 3 \approx \textbf{32 GB/s}$ 。

**第(7)题** **正确答案：D**

在序列末尾，**8004 和 8000 是紧挨着的两次访问，且它们都映射到了 0 号存储体**，因此必然会发生访存冲突。选项 D 符合这一情况 。

**第(8)题** **正确答案：B**

- **解析**：
  
  - **SDRAM (Synchronous Dynamic RAM)** 是同步动态随机存取存储器，本质上属于 DRAM 。DRAM 利用电容存储电荷来保存信息，电容会随时间漏电，因此必须**周期性刷新**以保持数据不丢失 。
  
  - SRAM（静态 RAM）利用触发器保存信息，只要不断电就不需要刷新 。
  
  - ROM（只读存储器）和 FLASH（闪存）都是非易失性存储器，不但不需要刷新，掉电后数据也不会丢失 。

**第(9)题** **正确答案：B**

- **解析**：
  
  - 随机存取（Random Access）指的是存储器的访问时间与其物理位置无关。
  
  - EPROM、DRAM、SRAM 均属于半导体存储器，采用电路进行矩阵寻址，可以实现完全的随机存取 。
  
  - **CDROM（光盘）**属于外存储器，读取数据时需要依靠主轴电机旋转和激光头径向移动来定位，其访问时间与数据所在的物理位置密切相关，因此属于**直接存取 / 顺序存取**，不采用随机存取方式 。

**第(10)题** **正确答案：A**

- **解析**：
  
  - **A 选项错误**：闪存（Flash Memory）在写入数据之前通常需要先进行“擦除”（Erase）操作（将块内的位全部置为 1），然后再进行编程（写入 0）。擦除和写入机制使得**闪存的写速度远低于读速度**，两者并不一样快 。
  
  - B 选项正确：闪存的存储元确实是由浮栅 MOS 管组成的半导体存储器 。
  
  - C 选项正确：闪存是一种非易失性存储器，掉电不丢失信息 。
  
  - D 选项正确：闪存可以随机访问，且广泛用于替代传统的机械硬盘（如目前的 SSD 和 U 盘） 。

**第(11)题** **正确答案：B**

数组元素在内存中是连续存放的。当我们顺序遍历数组时，访问了当前元素后，很快就会访问紧邻的下一个单元，这就体现了极强的**空间局部性**。但在大多数简单的遍历操作中，每个数组元素只被访问一次，之后便不再被访问，因此它**没有时间局部性**（时间局部性是指同一个数据被反复多次访问）。所以正确描述是无时间局部性，有空间局部性 。

**第(12)题** **正确答案：C**

**第(13)题** **正确答案：A**

- **解析**：
  
  - 1. 确定映射关系：按字编址，块大小为 1 个字，所以主存地址直接就是块地址 。Cache有 4 行，二路组相联，则总共有 $4 \div 2 = 2$ 组（第 0 组和第 1 组） 。
  
  - 2. 计算索引：映射的组号 = `主存地址 % 2`。观察题目给出的地址序列：0、4、8、2、0、6、8、6、4、8 。它们全都是偶数！因此所有地址的计算结果都为 0，这意味着**全部数据都映射到了第 0 组**。
  
  - **3. 模拟LRU替换（第0组仅有2个行位）**：
    
    - 问 0：未命中，装入 [0]
    
    - 问 4：未命中，装入 [0, 4]
    
    - 问 8：未命中，踢出最久未用的 0，装入 [4, 8]
    
    - 问 2：未命中，踢出 4，装入 [8, 2]
    
    - 问 0：未命中，踢出 8，装入 [2, 0]
    
    - 问 6：未命中，踢出 2，装入 [0, 6]
    
    - 问 8：未命中，踢出 0，装入 [6, 8]
    
    - **问 6：命中！状态更新，6变为最近使用，仍为 [8, 6]**
    
    - 问 4：未命中，踢出 8，装入 [6, 4]
    
    - 问 8：未命中，踢出 6，装入 [4, 8]
  
  - 整个过程中仅命中 **1** 次 。

**第(14)题** **正确答案：C**

- **解析**：
  
  - 1. 计算容量与块数：Cache 数据区能存放 4K 字，每字 32 位（4B） ，所以数据区总容量 = $4\text{K} \times 4\text{B} = 16\text{KB}$。主存块大小 = 4个字 = $4 \times 4\text{B} = 16\text{B}$ 。则 Cache 共有 $16\text{KB} \div 16\text{B} = 1024$ 行（即 $2^{10}$ 行）。
  
  - 2. 划分地址字段：主存地址为 32 位 。块内地址（块大小 $16\text{B} = 2^4\text{B}$）占 4 位；Index索引（$1024 = 2^{10}$ 行）占 10 位；剩余的 Tag 标记位 = $32 - 10 - 4 = 18$ 位。
  
  - 3. 计算附加位（Overhead）：除了数据区和 Tag，直接映射Cache每行还需要 1 位有效位（Valid bit）。此外，题目明确采用“**写回（Write-back）**”方式 ，因此必须增加 1 位脏位（Dirty bit，用于标记是否被修改过）。
  
  - **4. 计算总容量**：Cache 每行的总位数 = 数据位（$16\text{B} \times 8 = 128$位）+ Tag位（18位）+ 有效位（1位）+ 脏位（1位）= **148 位**。
    
    - 总容量 = $1024 \text{行} \times 148 \text{位/行} = 148\text{K}$ 位 。

**第(15)题** **正确答案：D**

- **解析**：
  
  - 在现代 CPU 的指令流水线中，取指阶段（IF）需要访问存储器获取指令，访存阶段（MEM）需要访问存储器读写数据 。
  
  - 如果指令和数据存放在同一个 Cache（统一 Cache）中，这两个阶段同时进行时就会发生“抢占同一个存储资源”的现象，引发流水线的**结构冲突（也称资源冲突）**。
  
  - 将指令 Cache 和数据 Cache 分离（即哈佛结构），可以完美解决这种冲突，从而允许 CPU 在同一时钟周期内既取指令又读写数据，大大减少流水线停顿 。

**第(16)题** **正确答案：B**

- **解析**：
  
  - 指令 `add xaddr, 3` 的功能是读取主存单元 `xaddr` 的数据，将其加 3，然后再写回主存单元 `xaddr`。
  
  - 题目要求的是完成该指令功能需要访问主存的**至少**次数。为了达到“至少”，我们需要假设系统运行在最理想的状态：
    
    1. **取指令阶段**：假设指令已经存在于指令 Cache 中（命中），且虚实地址转换时 TLB 命中，不需要访问主存。
    
    2. **读操作数阶段**：假设 `xaddr` 所在的页表项已经在 TLB 中（TLB 命中，访存 0 次），且该数据也恰好在数据 Cache 中（Cache 命中，访存 0 次）。
    
    3. **写回结果阶段**：由于题目明确指出 Cache 使用的是**写穿方式（Write-Through）**，这意味着每次向 Cache 写入数据时，必须同时将数据写回主存，以保持数据一致性。
  
  - 综上，最理想情况下，读数据不需要访存，但写数据因为“写穿”机制必定会访问 1 次主存。因此至少访问主存的次数为 1。

**第(17)题** **正确答案：D**

**TLB（快表）** 是页表的高速缓存，里面存放的是近期最常使用的页表项。如果 **TLB 命中**，就意味着该虚拟页的页表项就在 CPU 内部，且该页表项一定指示该页已经存在于物理主存中。如果一页已经存在于物理主存中，那么去主存页表中查找时，**Page（页）必然是命中的**。

- 因此，绝对不可能出现“TLB 命中，但 Page 未命中（缺页）”的情况。选项 D 的逻辑是自相矛盾的。

**第(18)题** **正确答案：A**

- **解析**：
  
  - **1. 划分虚拟地址结构**：
    
    - 虚拟地址空间为 4GB（$2^{32}$ 字节），因此虚拟地址（VA）总共有 32 位。
    
    - 页面大小为 4KB（$2^{12}$ 字节），因此页内偏移地址（VPO）占 12 位。
    
    - 虚拟页号（VPN） = 总位数 - 偏移位数 = $32 - 12 = 20$ 位。
  
  - **2. 拆解给定的虚拟地址**：
    
    - 给定虚拟地址：`03FFF180H`。
    
    - 提取低 12 位（右侧 3 个十六进制数）作为页内偏移：`180H`。
    
    - 提取高 20 位（左侧 5 个十六进制数）作为虚拟页号（VPN）：`03FFFH`。
  
  - **3. 查表与拼接物理地址**：
    
    - 查阅题目给出的 TLB 表（表4.12），找到标记为 `03FFFH` 的表项。
    
    - 该表项的有效位为 `1`（说明 TLB 命中），对应的页框号（物理页号 PPN）为 `0153H`。
    
    - 物理地址 = 物理页号 + 页内偏移 = `0153H` 拼接 `180H`，最终结果为 `0153180H`。

**第(19)题** **正确答案：D**

- **解析**：
  
  - **A 选项正确**：缺页是在虚拟地址转换为物理地址的过程中，MMU（内存管理单元）发现页表项的有效位为 0 时触发的一种内部异常（故障）。
  
  - **B 选项正确**：缺页异常触发后，CPU 会陷入内核态，交由操作系统的缺页处理程序来接管。
  
  - **C 选项正确**：缺页处理程序会根据引发缺页的地址，找到所需页面在外存（磁盘）上的位置，并将其调入物理主存的空闲页框中。
  
  - **D 选项错误**：缺页属于“故障（Fault）”类型的异常。这类异常的一个核心特征是：在操作系统解决掉故障（调入缺失的页）返回后，CPU 必须**重新执行刚才那条引发缺页的指令**。如果像选项 D 所说执行“下一条指令”，那么原本应该进行的存取操作就会被跳过，导致程序执行错误。

## 4.3 简答题答案要点

**(1) 计算机系统中采用层次化存储体系结构的目的是什么？**

- **目的：** 主要是为了解决存储器**容量、速度和价格**之间的矛盾。

- **原理：** 通过将不同速度、容量和成本的存储器（如 CPU 内部寄存器、Cache、主存、辅存）组织成金字塔型的层次结构，利用程序的局部性原理，使整个存储系统在速度上接近最快（且最贵）的存储器，在容量上接近最大（且最便宜）的存储器，从而实现最优的性价比。

**(2) 为什么在存储器芯片中设置片选输入端？**

- **目的：** 为了实现对多芯片存储器系统的精准寻址和控制。

- **原理：** 主存通常由多块存储芯片组合（字扩展或位扩展）而成。片选输入端（如 CS 或 CE）用于控制该芯片是否被激活并参与当前的读/写操作。CPU 给出的高位地址通过译码电路产生片选信号，只有收到有效片选信号的芯片才会与数据总线连通，从而避免了总线冲突。

**(3) 动态MOS存储器为什么要刷新？如何刷新？**

- **为什么要刷新：** 动态存储器（DRAM）是利用极微小的电容来存储电荷表示 0 和 1 的。由于绝缘体不完美，电容上的电荷会随时间逐渐泄漏。如果不及时补充电荷，原有的信息就会丢失。

- **如何刷新：** 硬件必须在规定的时间（刷新周期，通常为 2ms）内，对所有存储单元进行一次“读出并重新写入”的操作。刷新是**按行**进行的，常见的刷新策略有三种：
  
  - **集中刷新：** 在一个刷新周期内，暂停所有正常的读写操作，集中一段时间刷完所有行（存在死区时间）。
  
  - **分散刷新：** 把每个读写周期分为两半，前半部分读写，后半部分刷新（没有死区，但加长了存取周期）。
  
  - **异步刷新：** 前两者的折中，将刷新周期除以行数，每隔固定时间产生一次刷新请求。

**(4) 试述多体交叉存储器的设计思想和实现方法。**

- **设计思想：** 利用多个独立存储体的**时间重叠**操作，来提高主存的整体带宽和数据传输率，缓解 CPU 与主存之间的速度瓶颈。

- **实现方法：** 将主存分成若干个容量相同、各自拥有独立地址寄存器和读写控制电路的存储体。采用**低位交叉编址**（即把连续的主存地址依次分布在不同的存储体中）。这样，CPU 就可以采用流水线的方式，连续、交叉地向各个存储体发出存取请求，实现高速的并发访问。

**(5) 为什么说 cache 对程序员是透明的？**

- **原因：** Cache 的所有操作（包括主存块的调入、替换、地址映射、数据一致性维护等）**全部由纯硬件自动完成**。程序员无论是编写汇编语言还是高级语言，能操作的只有统一的主存物理地址或虚拟地址。程序员不需要（也无法）在代码中指定数据是存在 Cache 还是主存中，因此 Cache 对程序员是不可见的（透明的）。

**(6) 直接相联映射方式下为什么不需要使用替换算法？**

- **原因：** 在直接相联映射中，主存中的每一块只能映射到 Cache 中的**唯一固定行**（公式：Cache 行号 = 主存块号 mod Cache 总行数）。当发生 Cache 缺失需要调入新块时，它的目标位置是确定的。如果该位置已经被占用，只能无条件把旧块替换掉，根本没有其他候选位置可以挑，因此不需要替换算法介入。

**(7) 为什么要考虑 cache 的一致性？**

- **原因：** Cache 中的数据只是主存数据的副本。当 CPU 执行写操作修改了 Cache 中的数据后，Cache 里的值就是最新的，而主存里的值就成了旧的错误数据（此时出现了不一致）。如果此时 DMA 控制器（I/O 设备）直接读取主存，或者多核处理器中的其他核心共享该数据，就会读到脏数据导致系统出错。因此必须通过写回法（Write Back）**或**写穿法（Write Through）来维护两者的一致性。

**(8) 替换算法有哪几种？它们各有何优缺点？** 常用的替换算法主要有四种：

- **随机算法 (RAND)：**
  
  - 优点：实现极其简单，硬件开销极小。
  
  - 缺点：完全没有利用程序的局部性原理，命中率不稳定且较低。

- **先进先出算法 (FIFO)：** 替换最早调入 Cache 的块。
  
  - 优点：实现较为简单，开销小。
  
  - 缺点：没有考虑局部性原理，最早调入的块可能是经常使用的数据，替换掉会导致命中率下降。

- **近期最少使用算法 (LRU)：** 替换近期最长时间未被访问的块。
  
  - 优点：完美契合时间局部性原理，命中率高，是实际中最常用的算法。
  
  - 缺点：硬件实现复杂，每个 Cache 行都需要配置计数器来记录访问历史。

- **最不经常使用算法 (LFU)：** 替换过去一段时间内被访问总次数最少的块。
  
  - 优点：考虑了数据的访问频度。
  
  - 缺点：不能很好地适应程序运行阶段的变化（前期高频访问但后期不用了的数据很难被替换），且硬件实现繁琐。

## **4.4**

**题目：** 对于32KB容量的存储器，若按16位字编址，其地址寄存器应是多少位？数据寄存器是多少位？

- **数据寄存器 (MDR) 位数**：
  
  数据寄存器的位数取决于存储器的编址字长。题目指出“按16位字编址”，因此每次访存读写的数据量是一个16位的字，数据寄存器应为 **16位**。

- **地址寄存器 (MAR) 位数**：
  
  地址寄存器的位数取决于存储器中可独立寻址的单元总数。
  
  总容量为 32KB = $32 \times 1024$ 字节 = $32768$ 字节。
  
  每个字为 16位 = 2字节。
  
  可寻址的字单元总数 = $32768 \div 2 = 16384$ 个（即 $16\text{K}$ 个，或 $2^{14}$ 个）。
  
  要寻址 $2^{14}$ 个单元，需要 **14位** 的地址线，因此地址寄存器应为 **14位**。

### **4.12**

**题目：** Cache 由 64个块构成，四路组相联；主存含 4096个块，每块 128个字，访问地址为字地址。

**(1) 主存地址和 Cache 地址各有多少位？**

- **主存地址位数**：
  
  主存块数 = $4096 = 2^{12}$，即块号需要 12位。
  
  每块字数 = $128 = 2^7$，且按字编址，块内字偏移需要 7位。
  
  主存总字数 = $2^{12} \times 2^7 = 2^{19}$。所以主存地址为 **19位**。

- **Cache 地址位数**：
  
  Cache 块数 = $64 = 2^6$。
  
  每块字数 = $2^7$。
  
  Cache 总字数 = $2^6 \times 2^7 = 2^{13}$。所以 Cache 逻辑地址为 **13位**。

**(2) 主存地址的划分情况** 采用四路组相联映射，主存地址（19位）需要划分为：标记 (Tag)、组号 (Index)、块内偏移 (Offset) 三个部分。

- **块内偏移 (Offset)**：每块 128个字，因此字偏移占 **7位**。

- **组号 (Index)**：Cache 共有 64块，四路组相联（每组 4块），则 Cache 共有 $64 \div 4 = 16$ 组。由于 $16 = 2^4$，因此组索引占 **4位**。

- **标记 (Tag)**：主存地址总位宽 19位，减去组号和偏移，剩余 $19 - 4 - 7 =$ **8位**。

**主存地址划分图示如下：**

| **标记 (Tag)** | **组号 (Index)** | **块内偏移 (Offset)** |
| ------------ | -------------- | ----------------- |
| 8位           | 4位             | 7位                |

### **4.13**

**已知条件**：

- 主存容量：4MB

- Cache容量：16KB

- 每块包含：8个字，每个字32位（即4字节，4B），块大小 = $8 \times 4\text{B} = 32\text{B}$

- 映射方式：四路组相联

- 访问模式：读取主存第0, 1, 2, ..., 99号单元（共100个字），重复10次。采用LRU替换算法。

**（1）求 cache 的命中率**

1. **块数计算**：100个字占据了多少个数据块？
   
   因为每块包含8个字，所以100个字需要占用 $\lceil 100 / 8 \rceil = 13$ 个数据块（即第0块到第12块）。

2. **Cache 容量检验**：Cache共有 $16\text{KB} / 32\text{B} = 512$ 块。13块远远小于Cache的总容量，因此整个过程中**不会发生Cache替换**。

3. **命中情况分析**：
   
   - **第1次循环（读100个字）**：读每个块的第1个字时会发生未命中（共13次未命中，将13个块调入Cache），读该块其余字时全部命中。所以第1次循环有 13 次未命中，87 次命中。
   
   - **第2到10次循环**：由于这13个块已经全部驻留在Cache中且未被替换，后续9次循环的 $9 \times 100 = 900$ 次访问全部命中。

4. **命中率计算**：
   
   - 总访问次数：$10 \times 100 = 1000$ 次
   
   - 总未命中次数：13 次
   
   - 总命中次数：$1000 - 13 = 987$ 次
   
   - 命中率 = $987 / 1000 = 98.7\%$

**（2）若 cache 比主存快10倍，分析采用 cache 后存储访问速度提高了多少**

- 设 Cache 的访问时间为 $t$，则主存的访问时间为 $10t$。

- **不采用Cache时的平均访问时间**：$T_1 = 10t$

- **采用Cache后的平均访问时间**：$T_2 = \text{命中率} \times t + \text{缺失率} \times 10t = 0.987t + 0.013 \times 10t = 0.987t + 0.13t = 1.117t$

- **速度提升倍数**：$T_1 / T_2 = 10t / 1.117t \approx 8.95$ 倍。
  
  *(注：也可表述为速度提高了 $(10t - 1.117t)/1.117t \approx 7.95$ 倍，即795%)*

### **4.17 解析**

**已知条件**：

- 虚存 2GB ($2^{31}\text{B}$)，主存 4MB ($2^{22}\text{B}$)，按字节编址。

- 页大小 128KB ($2^{17}\text{B}$)。

- TLB：四路组相联，共16个页表项。

- Cache：16KB，每块8字，每字32位 ($8 \times 4\text{B} = 32\text{B}$，即 $2^5\text{B}$)，四路组相联。

**（1）虚拟地址划分（VPN, VPO, TLB标记, TLB索引）**

- 虚拟地址总位数：31位。

- **页内地址 (VPO)**：页大小128KB对应 17 位。

- **虚拟页号 (VPN)**：$31 - 17 = 14$ 位（虚拟地址的高14位）。

- **TLB索引**：TLB共16项，4路组相联，则组数 = $16 / 4 = 4$ 组。需 $\log_2(4) = 2$ 位（在VPN的低2位）。

- **TLB标记**：VPN位数 - TLB索引位数 = $14 - 2 = 12$ 位（在VPN的高12位）。

**（2）物理地址划分（PPN, PPO）**

- 物理地址总位数：22位。

- **偏移地址 (PPO)**：与VPO相同，为 17 位。

- **物理页号 (PPN)**：$22 - 17 = 5$ 位。

**（3）主存与数据cache的组相联映射地址划分**

- 物理地址（主存地址）总位数：22位。

- **块内地址 (Block Offset)**：块大小32B，需 5 位。

- **组索引 (Set Index)**：Cache共有 $16\text{KB} / 32\text{B} = 512$ 块。四路组相联，组数 = $512 / 4 = 128$ 组。需 $\log_2(128) = 7$ 位。

- **主存标记 (Tag)**：物理地址总位 - 组索引位 - 块内地址位 = $22 - 7 - 5 = 10$ 位。

- **地址划分方式**：最高 10 位为 Cache 标记 (Tag)，中间 7 位为组索引 (Index)，最低 5 位为块内偏移 (Offset)。
