# 存储器分类与层次结构

> 来源依据：第 4 章 PPT“存储器分类”“存储系统分层结构”。

## 一句话理解

存储系统不是只靠一种存储器，而是把“快但贵的小存储器”和“慢但便宜的大存储器”分层组合，让 CPU 大多数时候感觉既快又够用。

## 基本概念

CPU 理想中想要一种存储器：

- 像寄存器一样快。
- 像硬盘一样大。
- 像普通内存一样便宜。

现实中做不到，所以计算机采用存储层次结构：

```text
寄存器 -> Cache -> 主存 -> 外存
```

越靠近 CPU：

- 速度越快。
- 容量越小。
- 单位成本越高。

越远离 CPU：

- 速度越慢。
- 容量越大。
- 单位成本越低。

常见存储器分类可以从几个角度看。

| 分类角度 | 常见类别 | 说明 |
| --- | --- | --- |
| 按位置 | 内存、外存 | 主存和 Cache 属于内存层次，磁盘/SSD 属于外存 |
| 按是否掉电丢失 | 易失性、非易失性 | RAM 通常掉电丢失，ROM/Flash 通常不丢失 |
| 按存取方式 | 随机访问、顺序访问、直接访问 | RAM 可按地址直接访问，磁带更偏顺序访问 |
| 按用途 | 主存、Cache、辅助存储器 | 分别服务于运行、加速和长期保存 |

## 工作原理或推导过程

### 1. 为什么分层结构有效

如果程序每次访问的地址都完全随机，Cache 可能没什么用。但真实程序通常有**局部性**。

局部性有两类：

- **时间局部性**：刚访问过的数据，过一会儿可能还会访问。
- **空间局部性**：访问某个地址后，附近地址很可能也会访问。

例子：

```c
for (int i = 0; i < n; i++) {
    sum += a[i];
}
```

这里：

- `sum` 会反复使用，体现时间局部性。
- 数组 `a[0], a[1], a[2]...` 连续访问，体现空间局部性。

Cache 正是利用局部性：不是只取一个字节，而是把一个主存块取进来。这样访问附近地址时很可能直接命中。

### 2. Cache 和虚拟存储器分别利用什么局部性

Cache 主要解决 CPU 和主存之间的速度差距：

```text
CPU 很快，主存相对慢 -> 用 Cache 缓冲最近常用的数据
```

虚拟存储器主要解决主存容量和程序地址空间之间的矛盾：

```text
主存有限，程序以为空间很大 -> 暂时不用的页放外存，需要时再调入
```

两者都依赖局部性，但作用层次不同：

| 层次 | 上层 | 下层 | 交换单位 |
| --- | --- | --- | --- |
| Cache-主存 | Cache | 主存 | 块 |
| 主存-外存 | 主存 | 外存 | 页 |

### 3. 平均访问时间怎么理解

存储层次的目标是让平均访问时间接近上层高速存储器。

Cache 常用公式：

```text
平均访问时间 = 命中时间 + 缺失率 × 缺失代价
```

直观理解：

- 命中时间是每次都要付出的基础成本。
- 只有缺失时才会额外访问下一级存储器。
- 缺失率越低，平均速度越接近 Cache。

## 典型例子

### 例 1：按速度排序

一般速度从快到慢：

```text
寄存器 > Cache > 主存 > SSD/磁盘
```

容量通常反过来：

```text
寄存器 < Cache < 主存 < SSD/磁盘
```

### 例 2：判断局部性

代码：

```c
for (int i = 0; i < 1000; i++) {
    a[i] = a[i] + 1;
}
```

分析：

- 顺序访问 `a[i]`，数组元素地址连续，空间局部性好。
- 循环变量 `i` 和数组基地址反复使用，时间局部性好。

所以这类程序通常比较适合 Cache。

### 例 3：为什么外存数据不能直接被 CPU 运算

CPU 的算术逻辑部件通常直接处理寄存器中的数据，数据要先从外存调入主存，再从主存进入 Cache/寄存器。外存负责长期保存，不直接参与 CPU 的每条指令执行。

## 常见考法

- 按速度、容量或成本排序不同存储层次。
- 解释为什么计算机需要层次化存储体系。
- 判断一段程序体现时间局部性还是空间局部性。
- 区分 Cache、主存、外存的作用。
- 用命中率或缺失率计算平均访问时间。

## 易错点

- **Cache 不是主存的一部分**，它位于 CPU 和主存之间，用来加速访问。
- **外存容量大但不能被 CPU 像寄存器一样直接运算**，外存数据通常必须先调入主存。
- **局部性是层次结构有效的根本原因**，不是因为 Cache “碰巧快”。
- **速度快不等于容量大**，存储层次中这两个指标通常相反。
- **虚拟存储器不是简单把硬盘当内存**，它需要页表、地址转换和缺页处理配合。

## 小结

- 存储层次是在速度、容量和成本之间折中。
- 越靠近 CPU 越快、越小、越贵。
- 时间局部性和空间局部性让 Cache 与虚拟存储器有效。
- Cache 解决速度差距，虚拟存储器解决地址空间和主存容量管理问题。
- 后续 Cache 和虚实地址转换题，本质都建立在存储层次思想上。
