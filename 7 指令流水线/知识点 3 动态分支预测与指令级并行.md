# 动态分支预测与指令级并行

> 来源依据：第 7 章 PPT“动态分支预测流水线”“BHT”“指令级并行技术”“超流水线”“多发射/超标量”。

## 一句话理解

动态分支预测是在分支结果出来前先猜下一条指令该取哪里，指令级并行是在条件允许时让一个周期内推进不止一条指令。

## 基本概念

### 1. 为什么需要分支预测

流水线最怕“断流”。分支指令会让 CPU 暂时不知道下一条指令该从哪里取：

```text
如果不跳转：取 PC+4
如果跳转：取分支目标地址
```

等分支结果完全确定再取指，会浪费周期。分支预测的思想是：先根据历史经验猜一个方向，让流水线继续工作；如果猜对，就几乎没有停顿；如果猜错，再冲刷流水线并改取正确地址。

### 2. 动态分支预测

动态分支预测会根据程序运行过程中的历史行为更新预测结果。它不是固定认为“所有分支都跳转”或“所有分支都不跳转”，而是观察某条分支过去的表现。

常见部件：

| 部件 | 作用 |
| --- | --- |
| BHT | Branch History Table，保存分支历史状态，预测跳转或不跳转 |
| BTB | Branch Target Buffer，保存分支目标地址，方便预测跳转时直接取目标 |
| 预测错误检测逻辑 | 分支实际结果出来后，判断是否猜错 |
| Flush/重定向逻辑 | 猜错时清除错误路径并修正 PC |

### 3. 2 位饱和计数器

常见 BHT 项使用 2 位状态。直觉是：不要因为一次偶然变化就立刻反向预测。

一种常见划分是：

| 状态 | 预测 |
| --- | --- |
| `00` | 强不跳转 |
| `01` | 弱不跳转 |
| `10` | 弱跳转 |
| `11` | 强跳转 |

实际跳转时状态向“跳转”方向移动；实际不跳转时向“不跳转”方向移动。因为有强弱两级，循环分支这种“多次跳转，最后一次不跳转”的情况不会被一次不跳转彻底带偏。

### 4. 指令级并行

指令级并行（ILP）指在不改变程序结果的前提下，让多条指令在时间上重叠甚至同周期执行。

五段流水线本身已经是一种指令级并行：同一周期内，多条指令处于不同阶段。更进一步的技术包括：

| 技术 | 思路 | 提升什么 |
| --- | --- | --- |
| 超流水线 | 把流水段切得更细，缩短每段关键路径 | 提高主频 |
| 多发射 | 一个周期发射多条指令 | 提高每周期处理指令数 |
| 超标量 | 硬件动态支持多发射和多功能部件并行 | 提高 IPC |
| 静态多发射 | 编译器提前安排哪些指令可并行 | 简化硬件 |
| 动态多发射 | 硬件运行时判断并处理冲突 | 更灵活但硬件复杂 |

基本流水线理想情况下 `CPI` 接近 1；超标量如果一个周期能完成多条指令，理想 `CPI` 可能小于 1。

## 工作原理或推导过程

### 1. 分支预测如何工作

以 `beq` 为例，预测流程可理解为：

1. IF 阶段用当前 PC 查询 BHT/BTB。
2. 如果预测不跳转，下一条取 `PC+4`。
3. 如果预测跳转，并且 BTB 中有目标地址，下一条取目标地址。
4. 分支在后续阶段真正计算出是否跳转和目标地址。
5. 若预测正确，流水线继续。
6. 若预测错误，清除错误路径指令，PC 改为正确地址，并更新 BHT 状态。

预测不是保证正确，而是用较低成本换取大多数情况下不断流。正确性由“预测错误后冲刷和重定向”保证。

### 2. 分支预测对 CPI 的影响

分支预测的代价通常体现在预测错误上。常见估算：

```text
平均分支惩罚 = 分支指令比例 * 预测错误率 * 每次错误惩罚周期
实际 CPI = 理想 CPI + 平均分支惩罚 + 其他停顿
```

例如，理想 CPI 为 1，分支占 20%，预测错误率 10%，每次错误损失 3 个周期：

```text
平均分支惩罚 = 0.20 * 0.10 * 3 = 0.06
实际 CPI = 1 + 0.06 = 1.06
```

这说明预测准确率越高、错误惩罚越小，流水线越接近理想状态。

### 3. 为什么循环分支适合动态预测

循环通常长这样：

```text
循环体
计数未结束则跳回循环开头
```

如果循环执行 100 次，分支可能前 99 次都跳转，最后 1 次不跳转。动态预测能根据前几次行为逐渐认为“这条分支大概率跳转”，于是循环主体中的大多数分支都能预测正确。

2 位预测比 1 位预测更稳定，因为一次退出循环的不跳转不会立刻让预测器完全改变态度。

### 4. 超流水线为什么能提高主频

超流水线把原来的阶段继续细分。例如原 EX 阶段太慢，可以拆成 EX1、EX2。这样每个阶段延迟变小，时钟周期可能缩短，主频提高。

但代价是：

- 流水线更深，分支预测错误要冲刷更多阶段。
- 数据相关距离变长，转发和停顿处理更复杂。
- 流水寄存器更多，额外开销增加。

所以超流水线不是越深越好。

### 5. 超标量为什么 CPI 可以小于 1

普通五段流水线理想情况下每周期完成 1 条指令：

```text
IPC ≈ 1
CPI ≈ 1
```

若超标量处理器每周期最多发射 2 条指令，且两条指令没有冲突、资源足够，理想情况下：

```text
IPC ≈ 2
CPI ≈ 1 / 2 = 0.5
```

但这只是理想情况。真实程序中会受到数据相关、控制相关、结构资源、缓存缺失等限制。

## 典型例子

### 例 1：预测错误惩罚

某流水线：

- 理想 CPI 为 `1`。
- 分支指令占 `25%`。
- 动态预测错误率为 `8%`。
- 每次预测错误损失 `2` 个周期。

则：

```text
平均分支惩罚 = 0.25 * 0.08 * 2 = 0.04
实际 CPI = 1 + 0.04 = 1.04
```

### 例 2：2 位预测器状态变化

初始状态为 `10`，预测跳转。实际执行序列为：

```text
跳转, 跳转, 不跳转
```

状态变化可理解为：

```text
10 -> 11 -> 11 -> 10
```

最后一次不跳转只把状态从强跳转拉到弱跳转，下次仍可能预测跳转。这就是 2 位预测器抗偶然变化的原因。

### 例 3：超标量 CPI

某处理器理想情况下每周期完成 4 条指令：

```text
IPC = 4
CPI = 1 / IPC = 0.25
```

如果由于相关和分支影响，实际平均每周期完成 2 条：

```text
实际 IPC = 2
实际 CPI = 0.5
```

所以 CPI 小于 1 并不神秘，它只是说明平均一个周期能完成不止一条指令。

## 常见考法

- 解释动态分支预测和静态分支预测的区别。
- 说明 BHT 和 BTB 的作用。
- 根据 2 位饱和计数器判断预测方向和状态变化。
- 计算预测错误带来的平均 CPI 增量。
- 区分超流水线、多发射、超标量。
- 判断 CPI 是否可能小于 1，以及需要什么条件。
- 分析分支预测错误后为什么要冲刷流水线。

## 易错点

- 分支预测不是保证分支一定正确，而是先猜，猜错再修正。
- BHT 主要保存方向历史，BTB 主要保存目标地址，不要混。
- 预测跳转但没有正确目标地址，仍然无法顺利取目标指令。
- 2 位预测器不会因为一次相反结果立刻从强状态跳到强相反状态。
- 超流水线是把阶段切细提高主频，不是一个周期发射多条指令。
- 超标量/多发射才是增加同周期指令数量。
- CPI 小于 1 通常意味着 IPC 大于 1，不适用于普通单发射五段流水线的理想模型。
- 指令级并行受相关和资源限制，不是所有相邻指令都能并行执行。

## 小结

- 动态分支预测通过历史信息减少控制冒险造成的停顿。
- BHT 预测方向，BTB 提供目标地址，预测错误时必须冲刷并重定向。
- 分支预测对 CPI 的影响常按“分支比例 * 错误率 * 惩罚周期”估算。
- 超流水线靠切细阶段提高主频；超标量靠多发射提高 IPC。
- 基本流水线理想 CPI 接近 1，超标量理想 CPI 可以小于 1。
