# 五段指令流水线

> 来源依据：`知识点汇总.txt` 第七章第 1 点；第 7 章 PPT“MIPS 单周期与流水线”“5 段指令流水线数据与信号传递”“指令在流水线中的执行过程”；教材第 7 章。

## 一句话理解

五段流水线把一条指令拆成取指、译码、执行、访存、写回五步，让多条指令像生产线一样重叠工作，从而提高连续执行指令的吞吐率。

## 基本概念

### 1. 为什么要流水线

单周期处理器中，一条指令要在一个很长的周期里完成所有动作。即使 `add` 不需要访存，也要等到这个长周期结束。

流水线的想法很直观：如果一条指令可以拆成 5 个阶段，那么当第 1 条指令进入第 2 阶段时，第 2 条指令就可以进入第 1 阶段。这样 CPU 内部多个部件同时忙起来。

类似洗衣服：

```text
洗 -> 烘 -> 叠
```

如果每件衣服都洗完、烘完、叠完再处理下一件，效率低。流水线是第 1 件在烘时，第 2 件已经开始洗。单件衣服并没有少经历步骤，但整体完成速度提高了。

### 2. MIPS 经典五段

| 阶段 | 英文 | 主要工作 | 直觉 |
| --- | --- | --- | --- |
| IF | Instruction Fetch | 根据 PC 从指令存储器取指，并计算 `PC+4` | 拿任务单 |
| ID | Instruction Decode | 译码，读寄存器，生成后续控制信号 | 看懂任务单，取原料 |
| EX | Execution | ALU 运算，计算有效地址，比较分支条件 | 加工或算地址 |
| MEM | Memory Access | 访问数据存储器 | 去仓库读/写数据 |
| WB | Write Back | 把结果写回寄存器 | 把成品放回货架 |

不是所有指令都真正使用每个阶段的功能。例如 `add` 不需要数据存储器，但它仍然会经过 MEM 这个流水段，只是这一段不做有效访存操作。这样做是为了让所有指令按统一节拍在流水线中前进。

### 3. 流水寄存器

五段之间需要加流水寄存器：

```text
IF/ID -> ID/EX -> EX/MEM -> MEM/WB
```

它们的作用是把前一阶段的结果锁存住，交给下一阶段使用。例如：

- IF/ID 保存取到的指令和 `PC+4`。
- ID/EX 保存读出的寄存器值、立即数和控制信号。
- EX/MEM 保存 ALU 结果、写存储器的数据、控制信号。
- MEM/WB 保存访存结果或 ALU 结果，等待写回。

流水寄存器很重要，因为同一时刻不同指令在不同阶段执行。如果不把每条指令自己的数据和控制信号带着走，后面的阶段就不知道自己该为哪条指令服务。

## 工作原理或推导过程

### 1. 单条指令并没有变快

一条指令在五段流水线中仍然要经历：

```text
IF -> ID -> EX -> MEM -> WB
```

如果每段 1 个时钟周期，那么一条指令从进入到完成仍然要 5 个周期。这叫**延迟**。

流水线提升的是**吞吐率**：流水线填满后，理想情况下每个周期都能完成一条指令。

```text
延迟：一条指令从开始到结束多久
吞吐率：单位时间完成多少条指令
```

### 2. 理想情况下为什么是 k+n-1

设流水线有 `k` 段，要执行 `n` 条指令。

- 第 1 条指令完成需要 `k` 个周期。
- 第 2 条指令比第 1 条晚 1 个周期进入，所以晚 1 个周期完成。
- 第 n 条指令比第 1 条晚 `n-1` 个周期进入，所以总周期数为：

```text
总周期数 = k + n - 1
```

五段流水线就是：

```text
总周期数 = 5 + n - 1 = n + 4
```

这多出来的 `4` 个周期就是流水线的填充和排空开销。指令越多，这个开销摊得越薄，平均 CPI 越接近 1。

### 3. 流水线时钟周期怎么定

单周期处理器的时钟周期要覆盖整条最慢指令。流水线中，一个周期只完成一个阶段，所以时钟周期按最慢流水段确定：

```text
流水线时钟周期 = 最慢阶段延迟 + 流水寄存器延迟
```

为什么不是所有阶段延迟相加？因为五个阶段在不同周期分别执行，时钟只需要保证一个阶段的数据能在本周期内稳定送到下一个流水寄存器。

但如果五段延迟严重不均衡，流水线效果会下降。慢阶段像生产线里最慢的工位，所有人都必须按它的速度等。

### 4. 理想流水线的条件

课件中强调“理想流水线”并不容易满足，它大致要求：

| 条件 | 如果不满足会怎样 |
| --- | --- |
| 所有指令阶段数相同 | 有些指令提前结束或需要额外处理 |
| 各段延迟接近 | 时钟周期被最慢段拖慢 |
| 没有资源冲突 | 多条指令争用同一硬件，必须停顿 |
| 没有数据相关影响 | 后一条指令等前一条结果 |
| 没有控制转移影响 | 分支跳转导致取错指令 |

所以 `k+n-1` 是理想公式。真实题目中如果出现停顿、气泡、分支冲刷，要额外加周期。

## 典型例子

### 例 1：5 段流水线执行 6 条指令

无阻塞时：

```text
周期数 = k + n - 1 = 5 + 6 - 1 = 10
```

时空图可以这样看：

| 周期 | I1 | I2 | I3 | I4 | I5 | I6 |
| --- | --- | --- | --- | --- | --- | --- |
| 1 | IF |  |  |  |  |  |
| 2 | ID | IF |  |  |  |  |
| 3 | EX | ID | IF |  |  |  |
| 4 | MEM | EX | ID | IF |  |  |
| 5 | WB | MEM | EX | ID | IF |  |
| 6 |  | WB | MEM | EX | ID | IF |
| 7 |  |  | WB | MEM | EX | ID |
| 8 |  |  |  | WB | MEM | EX |
| 9 |  |  |  |  | WB | MEM |
| 10 |  |  |  |  |  | WB |

第 5 个周期开始，每个周期都有一条指令完成；第 10 个周期最后一条完成。

### 例 2：为什么 CPI 接近 1

5 段流水线执行 100 条无阻塞指令：

```text
周期数 = 5 + 100 - 1 = 104
平均 CPI = 104 / 100 = 1.04
```

执行 10000 条时：

```text
周期数 = 10004
平均 CPI = 10004 / 10000 = 1.0004
```

指令越多，填充和排空的额外 4 个周期越不明显，所以理想 CPI 接近 1。

## 常见考法

- 写出五段流水线的阶段名称和作用。
- 判断某条 MIPS 指令在各阶段做什么。
- 给流水线段数 `k` 和指令条数 `n`，求无阻塞周期数。
- 根据阶段延迟求流水线时钟周期。
- 比较单周期、多周期、流水线的延迟和吞吐率。
- 判断控制信号为什么要随流水寄存器向后传递。

## 易错点

- 流水线提高的是连续指令吞吐率，不是让单条指令只用 1 个周期完成。
- 理想五段流水线执行 `n` 条指令是 `n+4` 个周期，不是 `5n`。
- `CPI` 接近 1 不代表一条指令只经过一个阶段。
- 时钟周期由最慢流水段和流水寄存器开销决定。
- MEM 阶段不是每条指令都真正访存，WB 阶段也不是每条指令都写回。
- 控制信号在 ID 阶段生成后，要跟随对应指令进入后续流水寄存器。

## 小结

- 五段流水线是 IF、ID、EX、MEM、WB。
- 流水线的核心价值是让多条指令重叠执行，提高吞吐率。
- 理想 `k` 段流水线执行 `n` 条指令需要 `k+n-1` 个周期。
- 五段流水线理想 CPI 接近 1，但真实流水线会受冲突、相关和分支影响。
