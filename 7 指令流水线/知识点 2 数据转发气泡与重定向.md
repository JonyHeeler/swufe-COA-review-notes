# 数据转发、气泡与重定向

> 来源依据：第 7 章 PPT“数据相关处理机制”“插入气泡消除 Load-Use 相关”“重定向流水线”“分支相关处理”。

## 一句话理解

转发是“结果还没写回但已经算出来，先借给后面的指令用”；气泡是“结果还没来得及产生，只能等一拍”；重定向是“发现取错路后，把 PC 改回正确方向并清掉错取指令”。

## 基本概念

### 1. 数据转发

数据转发也叫旁路。它绕过寄存器写回过程，把后面流水段已经得到的结果直接送到前面需要它的 ALU 输入。

常见转发来源：

- EX/MEM 流水寄存器中的 ALU 结果。
- MEM/WB 流水寄存器中的写回数据。

常见转发目标：

- EX 阶段 ALU 的输入 A。
- EX 阶段 ALU 的输入 B。

转发不改变程序语义，只是改变数据到达后续指令的路径。

### 2. 气泡

气泡是流水线中的空操作，也可以理解为硬件插入的 `nop`。它不完成有效指令，只占用一个周期，让后面的指令晚一点进入关键阶段。

硬件插入气泡时常做两件事：

- 暂停 PC 和 IF/ID，让当前取指、译码阶段不要前进。
- 清空 ID/EX 的控制信号，让 EX 阶段进入一个空操作。

### 3. 重定向

重定向用于控制冒险。分支或跳转方向确定后，如果之前取错了指令，就要：

1. 把 PC 改为正确地址。
2. 清除已经进入流水线的错误路径指令。

清除错误指令通常称为 flush，表现为把对应流水寄存器清零，使错误指令变成空操作。

## 工作原理或推导过程

### 1. 为什么 ALU 相关可以转发

看这个例子：

```mips
add $s1, $s2, $s3
sub $s4, $s1, $s5
```

`sub` 需要 `$s1`，而 `$s1` 是上一条 `add` 产生的。

按五段流水线：

- `add` 在 EX 阶段结束时已经算出 `$s1` 的新值。
- `sub` 在下一周期进入 EX 阶段时才真正需要 `$s1` 作为 ALU 输入。

虽然 `add` 还没有到 WB 阶段写回寄存器，但新值已经在 EX/MEM 或 MEM/WB 附近存在。因此可以直接转发给 `sub`。

直觉是：结果已经从加工机器出来了，只是还没放回仓库。下一道工序可以直接从机器出口拿。

### 2. 为什么 load-use 需要气泡

```mips
lw  $s1, 0($s0)
add $s2, $s1, $s3
```

`lw` 和普通 ALU 指令不同。它的最终数据不是 EX 阶段算出来的，而是在 MEM 阶段访问数据存储器后才出来。

若 `add` 紧跟在 `lw` 后面：

- `add` 太早进入 EX，已经需要 `$s1`。
- `lw` 的数据还在 MEM 阶段末尾才会出现。

所以数据不是“没写回”，而是“还没产生”。这种情况下转发也来不及，只能插入一个气泡：

```mips
lw  $s1, 0($s0)
nop
add $s2, $s1, $s3
```

插入一个周期后，`add` 的 EX 阶段推迟，`lw` 的数据就可以从 MEM/WB 转发过来。

### 3. 判断是否需要气泡的步骤

建议按下面顺序判断：

1. 找写寄存器的指令：它写哪个目标寄存器？
2. 找后续读寄存器的指令：它是否读同一个寄存器？
3. 判断结果产生时间：ALU 结果在 EX 末尾，load 数据在 MEM 末尾。
4. 判断使用时间：后续 ALU 运算通常在 EX 阶段需要操作数。
5. 若转发来得及，不插气泡；若来不及，插气泡。

一句话技巧：

```text
ALU 后紧跟使用：通常转发可解
lw 后紧跟使用：通常需要 1 个气泡
```

### 4. 转发控制信号怎么理解

流水线常在 ALU 输入前增加多路选择器，由转发控制信号选择数据来源：

| 选择 | ALU 输入来源 |
| --- | --- |
| 正常 | ID/EX 中读出的寄存器值 |
| EX/MEM 转发 | 前一条 ALU 指令刚算出的结果 |
| MEM/WB 转发 | 更早指令准备写回的结果 |

考试不一定要求写完整控制逻辑，但要理解它为什么比较“目标寄存器”和“源寄存器”：

```text
如果 EX/MEM.WriteReg == ID/EX.rs
并且 EX/MEM.RegWrite = 1
则 ALU 输入 A 可能要从 EX/MEM 转发
```

`rt` 输入同理。

### 5. 分支重定向为什么会损失周期

以 `beq` 为例，分支是否跳转要等比较结果出来。如果分支在 EX 阶段才确定，那么在它确定前，IF 阶段已经按 `PC+4` 取了后续指令。

如果后来发现应该跳转，之前取入的指令就是错误路径，需要清除：

```text
beq 确定跳转 -> PC 改为分支目标地址 -> 清除误取指令
```

分支越早确定，误取越少，损失越小。课件中提到 MIPS 可在 ID 段执行分支以减少损失；为了简化某些机制，也可在 EX 段执行分支。题目采用哪种，要以题干为准。

### 6. 分支延迟槽和重定向的区别

分支延迟槽的思想是：规定分支指令后一条指令无论是否跳转都执行。这样可以把本来可能浪费的一个周期放入有用指令。

重定向则是硬件发现方向后修改 PC，并清除错误路径指令。

两者目的相同：减少控制冒险损失。但延迟槽更依赖编译器安排有用指令，现代处理器更多使用动态分支预测。

## 典型例子

### 例 1：ALU 转发

```mips
add $1, $2, $3
sub $4, $1, $5
```

`add` 写 `$1`，`sub` 读 `$1`。`add` 的结果在 EX 后已经可用，`sub` 的 EX 阶段可以从 EX/MEM 或 MEM/WB 转发，因此通常不需要气泡。

### 例 2：load-use 插入气泡

```mips
lw  $2, 4($5)
and $1, $2, $3
```

`lw` 的 `$2` 要到 MEM 后才出来，`and` 紧跟着在 EX 要用 `$2`，所以需要：

```mips
lw  $2, 4($5)
nop
and $1, $2, $3
```

若 `lw` 和 `and` 中间本来就有一条无关指令，可能不再需要额外气泡。

### 例 3：分支冲刷

```mips
beq $1, $2, target
add $3, $4, $5
sub $6, $7, $8
target:
or  $9, $10, $11
```

若 `beq` 实际跳转，但流水线在分支确定前已经取入 `add` 和 `sub`，这些误取指令要被清除，PC 改为 `target`。损失几个周期取决于分支在哪个阶段确定。

## 常见考法

- 判断某组 RAW 相关是否能用转发解决。
- 判断 load-use 是否要插入气泡。
- 在时空图中画出 `nop` 或气泡位置。
- 分析硬件插气泡时 PC、IF/ID、ID/EX 应如何控制。
- 判断分支跳转后哪些流水寄存器要清零。
- 比较转发、停顿、编译器调度、分支延迟槽的作用。

## 易错点

- 转发不是把结果提前写回寄存器，而是直接送到需要的输入端。
- 气泡不是错误指令，它是空操作，用来保证正确性。
- load-use 的问题是数据产生太晚，不是单纯“没写回”。
- 有转发也不是所有数据冒险都自动消失。
- 插入气泡时要暂停前面的取指和译码，否则会丢失当前指令。
- 分支冲刷清掉的是错误路径指令，不是已经正确执行完的指令。
- 分支损失与分支在哪个阶段确定有关，不能脱离题目默认。

## 小结

- 判断数据冒险要看“产生时间”和“使用时间”。
- ALU 结果通常可通过转发解决紧邻相关。
- `lw` 后紧跟使用结果通常需要 1 个气泡。
- 气泡通过停顿和清空控制信号让流水线安全等待。
- 重定向用于控制冒险：改 PC，并清除误取指令。
