# 流水线时空图与性能计算

> 来源依据：第 7 章 PPT“单周期时空图”“多周期时空图”“指令流水线时空图”“理想流水线”。

## 一句话理解

流水线时空图把“哪条指令在第几个周期处于哪个阶段”画出来，性能计算就是在这张图上数周期、乘时钟周期、算吞吐率和加速比。

## 基本概念

### 1. 时空图怎么看

流水线时空图通常：

- 横轴表示时间或时钟周期。
- 纵轴表示指令。
- 表格中的 IF、ID、EX、MEM、WB 表示某条指令在某周期所处阶段。

它的价值是把“重叠执行”看清楚。没有流水线时，下一条指令要等上一条完成；有流水线时，多条指令在同一周期处于不同阶段。

### 2. 性能指标

| 指标 | 含义 | 公式直觉 |
| --- | --- | --- |
| 总周期数 | 完成所有指令花了多少个时钟周期 | 先数周期 |
| 总时间 | 实际花费的物理时间 | `总周期数 * 时钟周期` |
| 吞吐率 | 单位时间完成多少条指令 | `完成指令数 / 总时间` |
| 加速比 | 流水线比非流水线快多少 | `非流水时间 / 流水时间` |
| 平均 CPI | 平均每条指令几个周期 | `总周期数 / 指令条数` |

性能题的核心顺序是：

```text
先求周期数 -> 再求时钟周期 -> 再求总时间 -> 再算吞吐率或加速比
```

## 工作原理或推导过程

### 1. 无阻塞流水线周期数

`k` 段流水线执行 `n` 条指令：

```text
总周期数 = k + n - 1
```

如果有阻塞或气泡：

```text
总周期数 = k + n - 1 + 阻塞周期数
```

为什么加阻塞周期？因为气泡会占用流水线位置，却不完成有效指令。它像生产线上的空盒子，会把后面的成品完成时间向后推。

### 2. 时钟周期如何确定

设各段延迟为：

```text
t1, t2, ..., tk
```

流水线每一拍必须让最慢阶段也能完成，所以：

```text
T流水 = max(t1, t2, ..., tk) + T流水寄存器
```

非流水如果顺序完成全部阶段，单条指令时间常近似为：

```text
T非流水单条 = t1 + t2 + ... + tk
```

这解释了为什么流水线理想最大加速比接近段数 `k`：如果每段完全等长、没有额外寄存器延迟、没有冲突，那么一条指令的工作被均匀拆成 `k` 份，长期看每个周期完成一条。

### 3. 总时间

流水线总时间：

```text
T总 = (k + n - 1 + 停顿周期数) * T流水
```

若题目给主频 `f`，则：

```text
T流水 = 1 / f
T总 = 总周期数 / f
```

例如 `1GHz` 的周期是 `1ns`，`1.03GHz` 的周期约为 `1 / 1.03GHz`。

### 4. 吞吐率

吞吐率是完成指令数除以总时间：

```text
吞吐率 = n / T总
```

当 `n` 很大且无阻塞时，稳态吞吐率接近：

```text
1 / T流水
```

也就是每个周期完成一条指令。

### 5. 加速比

加速比定义为：

```text
加速比 = 非流水执行时间 / 流水执行时间
```

若非流水每条指令要走 `k` 段，每段时间相同为 `T`，则：

```text
非流水时间 = n * k * T
流水时间 = (k + n - 1) * T
加速比 = n*k / (k+n-1)
```

当 `n` 很大时：

```text
加速比 -> k
```

但真实情况中，流水寄存器延迟、阶段不均衡、冒险停顿都会降低加速比。

## 典型例子

### 例 1：5 段流水线执行 100 条指令

无阻塞：

```text
周期数 = 5 + 100 - 1 = 104
平均 CPI = 104 / 100 = 1.04
```

如果时钟周期为 `2ns`：

```text
总时间 = 104 * 2ns = 208ns
吞吐率 = 100 / 208ns ≈ 4.81 * 10^8 条/秒
```

### 例 2：4 段流水线，主频 1.03GHz

4 段流水线执行 100 条指令，无阻塞：

```text
周期数 = 4 + 100 - 1 = 103
```

主频为 `1.03GHz`：

```text
总时间 = 103 / 1.03GHz = 100ns
吞吐率 = 100 / 100ns = 1.0 * 10^9 条/秒
```

这里不要把 `103` 个周期误算成 `100` 个周期，前面的 `3` 个额外周期来自填充和排空。

### 例 3：阶段延迟不均衡

某 5 段流水线各段延迟为：

```text
IF=100ps, ID=120ps, EX=180ps, MEM=200ps, WB=80ps
```

流水寄存器延迟为 `20ps`，则：

```text
T流水 = max(100,120,180,200,80) + 20
      = 200 + 20
      = 220ps
```

不是：

```text
100 + 120 + 180 + 200 + 80 = 680ps
```

`680ps` 更接近非流水中一条指令顺序走完所有阶段的组合延迟。

## 常见考法

- 根据时空图数总周期数。
- 根据段数和指令数套用 `k+n-1`。
- 给出阻塞或气泡数量，求加入停顿后的周期数。
- 给各段延迟和流水寄存器延迟，求流水线时钟周期。
- 求总执行时间、吞吐率、平均 CPI。
- 比较流水线和非流水线的加速比。
- 判断为什么流水线实际加速比小于理想段数。

## 易错点

- `k+n-1` 只适用于无阻塞、单发射、每周期推进一段的理想流水线。
- 有气泡时要额外加阻塞周期。
- 吞吐率是“完成指令数 / 总时间”，不是“周期数 / 指令数”。
- 流水线时钟周期由最慢段决定，不是所有段相加。
- 流水寄存器延迟要加到时钟周期中。
- 加速比必须用时间比，不能只用 CPI 比。
- 指令数很少时，填充和排空开销明显，平均 CPI 不会接近 1。

## 小结

- 时空图是流水线题最可靠的分析工具。
- 无阻塞 `k` 段流水线执行 `n` 条指令需要 `k+n-1` 个周期。
- 总时间等于总周期数乘流水线时钟周期。
- 流水线吞吐率随指令数增大接近 `1/T流水`。
- 阶段不均衡、流水寄存器延迟和冒险停顿都会降低实际性能。
