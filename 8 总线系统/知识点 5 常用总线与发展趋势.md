# 常用总线与发展趋势

> 来源依据：第 8 章 PPT“常用总线”“总线标准化”“PCI 总线特点”“PCI-EXPRESS”“USB”“FSB 与 QPI”“CPU+PCH 方案”。

## 一句话理解

常用总线的发展主线是：从低速共享并行总线，逐步走向高速、点对点、串行、多通路和分层互连。

## 基本概念

这一节不建议把 ISA、EISA、PCI、AGP、PCIe、USB 的年份和细节硬背成流水账。更好的理解方法是抓住每一代总线想解决的瓶颈。

早期计算机中，多个设备共享一组并行信号线。这样结构简单、成本低，但问题很明显：

```text
设备越多，大家越抢同一条总线；
频率越高，并行线之间越容易串扰和偏斜；
高速设备和低速设备混在一起，快设备被慢设备拖住。
```

后来的总线技术主要朝几个方向演进：

- **局部化**：高速设备靠近 CPU 或主存，减少低速设备干扰。
- **分层化**：高速总线和低速总线之间用桥接器连接。
- **串行化**：用高速串行链路减少并行线的串扰和偏斜。
- **点对点化**：不再让所有设备共享同一组信号线，而是设备之间建立专用连接。
- **多通路化**：通过 x1、x4、x16 等多条串行通路并发提高带宽。

## 工作原理或推导过程

### 1. 总线标准化为什么重要

PPT 中强调，总线标准化使不同厂家的相同功能部件可以互换使用。标准规定了插槽形状、引脚功能、电平、时序、协议等细节。

直觉上，总线标准就像统一的插座和语言：

```text
插得上，电平能识别，时序能配合，命令能听懂。
```

没有标准化，每个厂家都设计自己的接口，设备很难兼容，系统扩展成本会很高。

### 2. ISA、EISA、VESA：早期并行扩展总线

**ISA 总线**是早期 PC 中常见的系统扩展总线。PPT 给出的典型参数：

```text
宽度：16 位
频率：8MHz
```

它适合早期低速设备，但带宽有限。

**EISA 总线**扩展到 32 位、16MHz，用于提升并行总线能力。它体现的是“加宽、提频”的传统改进思路。

**VESA/VL 局部总线**把高速图形等设备更靠近 CPU 和主存，减少传统低速扩展总线的瓶颈。PPT 中提到它的数据、地址总线宽度均为 32 位，最高传输率约 133MB/s。

这几类总线可以一起理解：

```text
早期思路：仍然以并行共享为主，通过加宽、提频、局部化来提高性能。
```

### 3. PCI：典型的共享并行局部总线

PCI 是考试中比较常见的总线。PPT 中给出的特点包括：

- 支持总线主控技术，智能设备可取得总线控制权。
- 支持猝发传输模式。
- 不受 CPU 速度和结构直接限制。
- 兼容 ISA/EISA/MCA 等传统系统。
- 数据宽度 32 位，时钟频率 33MHz。
- 最大数据传输率约 133MB/s。
- 同步时序、集中式仲裁。

PCI 带宽计算可以直接用总线带宽思想：

```text
32 位 = 4B
33MHz × 4B ≈ 132MB/s
通常写作约 133MB/s
```

PCI 的局限性也要理解：

```text
多个 PCI 设备共享总线带宽。
多个设备共享一组信号线，频率升高和插槽增多会带来信号问题。
CPU 或主桥需要与 PCI 总线时序配合，可能影响系统整体性能。
```

所以 PCI 是“共享并行总线”的代表，它比 ISA 强很多，但仍受共享和并行信号质量限制。

### 4. AGP：为图形显示单独开高速通道

AGP 是 Accelerated Graphics Port，高速图形接口。它的出现是因为图形显示对带宽需求增长很快，如果显卡继续和其他 PCI 设备共享总线，就容易形成瓶颈。

AGP 的直觉是：

```text
给显卡开一条更专用、更高速的局部通道。
```

PPT 中提到：

- AGP 1x 带宽约 266MB/s。
- AGP 2x 带宽约 533MB/s。
- AGP 4x 带宽约 1066MB/s。
- AGP 独立于 PCI 总线。
- AGP 显卡可以直接访问系统内存，缓解显存容量压力。

这里的 `1x`、`2x`、`4x` 可以和“单时钟传输次数”联系起来理解：在相同基本频率和宽度下，通过增加每周期传输次数提高带宽。

### 5. PCI-X 与 Compact PCI：继续改进 PCI

PCI-X 的目标是解决服务器中 PCI 带宽不足的问题。PPT 中提到它通过提高时钟速率和数据传输速率，使服务器外围设备获得更高吞吐量。

Compact PCI 则是在 PCI 电气和软件标准基础上，结合工业组装标准，面向工业计算机应用。

这两者不是本章计算重点，理解为：

```text
PCI 体系在不同场景下的增强和工业化变体。
```

### 6. PCI Express：从共享并行走向点对点高速串行

PCI Express 是总线发展中的关键转折。PPT 中强调：

```text
PCI Express 采用设备间点对点串行连接。
每个设备有自己的专用连接，不需要向整个共享总线请求带宽。
```

它解决了 PCI 的两个核心问题：

| PCI 的问题 | PCIe 的改进 |
| --- | --- |
| 多设备共享同一总线带宽 | 点对点连接，减少共享争用 |
| 并行线高频下串扰、偏斜明显 | 高速串行差分链路，抗干扰更好 |

PCIe 的 `x1`、`x4`、`x16` 表示并发通路数，不是传统意义上的 1 位、4 位、16 位并行总线。每条基本连接通常包含接收和发送方向，支持全双工通信。

复习时抓住这句话：

```text
PCIe 用多条高速串行通路，替代传统共享并行总线。
```

### 7. USB：面向外设的通用串行总线

USB 是 Universal Serial Bus，通用串行总线。PPT 中给出：

- USB1.0 带宽为 12Mbps。
- 支持热插拔、即插即用。
- 可接入多个外设。
- 使用四线电缆，其中两根传数据，另两根为下游设备供电。
- 采用级联星型拓扑。
- 基本组成包括主机 Host、集线器 Hub、功能设备。

USB 的意义是解决 PC 外设接口混乱、插槽有限、扩展不方便的问题。鼠标、键盘、打印机、扫描仪、移动存储等都可以通过统一接口接入。

PPT 还列出 USB 的数据传输类型：

| 类型 | 适用场景 | 理解方法 |
| --- | --- | --- |
| 控制传输 | 设备配置、命令控制 | 管理设备 |
| 等时传输 | 音频、视频等实时数据 | 保证时间连续性 |
| 中断传输 | 键盘、鼠标、手柄 | 数据量小，但要求响应快 |
| 数据块传输 | 打印机、扫描仪、相机等 | 数据量大，有带宽时传 |

### 8. FSB、QPI、DMI、PCH：系统互连结构的演进

PPT 展示了采用前端总线的南北桥结构、QPI 方案和 CPU+PCH 方案。

可以按结构变化来理解：

早期南北桥结构：

```text
CPU 通过 FSB 连接北桥 MCH。
北桥连接主存和高速图形接口。
南桥 ICH 连接磁盘、USB、音频、低速 I/O 等。
```

后来：

```text
内存控制器逐步集成到 CPU。
CPU 与高速 I/O 之间采用更高速的点对点互连，如 QPI。
传统北桥功能减少甚至消失。
PCH 承担平台控制中心角色，通过 DMI 与 CPU 连接。
```

这体现了系统总线结构的趋势：

- CPU 到主存路径越来越短。
- 高速设备靠近 CPU。
- 慢速外设集中到平台控制芯片。
- 点对点高速互连替代传统共享前端总线。

### 9. InfiniBand：面向极高速互连

PPT 中提到 InfiniBand 是统一 Future I/O 和 NGIO 的新型总线标准，定位是极高速传输总线。对本课程复习来说，通常只需知道它代表服务器、高性能系统中的高速互连方向，不必展开协议细节。

## 典型例子

### 例 1：PCI 最大带宽

PCI 数据宽度 32 位，频率 33MHz：

```text
32 位 = 4B
最大带宽 = 4B × 33MHz ≈ 132MB/s
通常近似为 133MB/s
```

### 例 2：为什么 PCIe 比 PCI 更适合高速设备

PCI 是多设备共享并行总线。设备越多，带宽越容易被分摊，高频下信号线还容易出现串扰和偏斜。

PCIe 使用点对点高速串行差分链路，每个设备有独立连接，并可通过 x4、x8、x16 等多通路并发扩展带宽。因此更适合显卡、固态硬盘、高速网卡等设备。

### 例 3：USB 鼠标为什么适合中断传输

鼠标每次传的数据量很小，但用户移动或点击后希望主机尽快响应。中断传输正适合这种“数据量小、无严格周期、对响应时间敏感”的外设。

### 例 4：为什么北桥会逐渐消失

内存控制器集成进 CPU 后，CPU 访问主存不再必须经过独立北桥。高速图形和 PCIe 也更靠近 CPU，于是传统北桥的核心功能被 CPU 吸收，剩余低速 I/O 控制集中到 PCH。

## 常见考法

- 根据宽度和频率计算 PCI 等总线带宽。
- 判断 PCI 的特点：同步时序、集中式仲裁、支持总线主控和猝发传输。
- 比较 PCI 和 PCIe：共享并行 vs 点对点高速串行。
- 解释 PCIe x1、x4、x16 中数字的含义。
- 判断 USB 的拓扑结构和基本组成：Host、Hub、功能设备。
- 区分 USB 控制、等时、中断、数据块传输类型。
- 理解 FSB、QPI、DMI、PCH 所体现的系统结构演进方向。

## 易错点

- **把 PCIe x16 当成 16 位并行总线**：x16 表示 16 条串行通路并发。
- **认为 PCIe 仍是传统共享总线**：PCIe 是点对点连接，和 PCI 的共享总线思想不同。
- **只记 USB 是串行总线，忘记它还能供电和热插拔**：USB 的优势是统一外设连接方式。
- **把 USB 中断传输理解为 CPU 中断机制本身**：这里是 USB 的传输类型，强调小数据量和快速响应。
- **认为总线发展只是频率变高**：真正趋势还包括分层、点对点、串行、多通路和控制器集成。
- **混淆 FSB 和 QPI**：FSB 是传统前端总线思路，QPI 更接近高速点对点互连。

## 小结

- 常用总线的演进主线是从共享并行到高速串行、点对点和多通路。
- PCI 是典型共享并行局部总线，32 位、33MHz 时约 133MB/s。
- AGP 为图形设备提供更专用的高速通道。
- PCIe 通过点对点串行链路和多通路并发提高带宽。
- USB 面向外设，支持热插拔、即插即用和级联星型拓扑。
- FSB 到 QPI、DMI、PCH 的变化体现了系统互连分层和集成化趋势。
