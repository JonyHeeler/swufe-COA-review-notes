# 总线性能指标与带宽计算

> 来源依据：`知识点汇总.txt` 第八章“总线相关指标的计算”；第 8 章 PPT“总线性能指标”“串行传送方式”“并串行传送”“总线定时”及相关例题。

## 一句话理解

总线带宽本质上是在问：这条“公共通道”每秒能搬走多少有效数据；计算时先看一次能搬多少，再看一秒能搬几次，最后扣掉地址、等待、编码等不搬有效数据的开销。

## 基本概念

理解总线性能，先不要急着背公式。可以把总线想成一条大家共用的传送带：

- **总线宽度**：传送带一次并排能放多少位数据。32 位总线一次能传 32 bit，也就是 4B；64 位总线一次能传 64 bit，也就是 8B。
- **总线时钟频率**：传送带每秒走多少拍。100MHz 表示每秒有 `100 × 10^6` 个时钟周期。
- **单时钟传输次数**：一个时钟周期里真正传数据的次数。普通 SDR 一周期传 1 次，DDR 一周期传 2 次，QDR 一周期传 4 次。
- **总线周期或总线事务时间**：完成一次完整访问要花的时间。一次访问通常不只是“传数据”，还可能包括申请总线、发地址、等待存储器、传数据、释放总线。
- **总线带宽**：单位时间内传输的数据量，常用 `B/s`、`MB/s`、`GB/s` 表示。
- **最大带宽**：只考虑理想的数据传输能力，不考虑申请、寻址、等待等额外开销。
- **有效传输率**：真正对题目有用的数据量除以整个事务花掉的时间，通常比最大带宽小。

最容易混淆的是“bit”和“Byte”。总线宽度常用 bit 表示，但带宽通常用 Byte/s 表示，所以只要从“位”变成“字节”，就要除以 8。

## 工作原理或推导过程

### 1. 最大带宽：先数一次搬多少，再数一秒搬几次

如果一条同步总线：

- 数据总线宽度为 `W bit`
- 时钟频率为 `f Hz`
- 每个时钟周期传输 `k` 次数据

那么一次传输的数据量是：

```text
W / 8 字节
```

一秒钟的数据传输次数是：

```text
f × k 次
```

所以最大带宽为：

```text
总线最大带宽 = W / 8 × f × k
```

这不是凭空来的公式，而是：

```text
带宽 = 每次传输的数据量 × 每秒传输次数
```

例如 32 位、400MHz、每周期传 1 次：

```text
每次传输 = 32 / 8 = 4B
每秒传输次数 = 400 × 10^6
最大带宽 = 4B × 400 × 10^6/s = 1.6GB/s
```

如果改成 64 位并采用 QDR：

```text
每次传输 = 64 / 8 = 8B
每秒传输次数 = 400 × 10^6 × 4
最大带宽 = 8B × 400 × 10^6/s × 4 = 12.8GB/s
```

提升倍数不要直接比较最后单位，先比较影响因素：

```text
宽度从 32 位到 64 位：×2
单周期传输次数从 1 到 4：×4
频率不变：×1
总提升 = 2 × 4 = 8 倍
```

### 2. 有效传输率：要把“不搬数据的时间”也算进去

考试题常常不会只问理想带宽，而是给出地址周期、等待周期、数据周期。这时要换一种想法：

```text
有效传输率 = 有效数据量 / 总耗时
```

关键是区分：

- **有效数据量**：题目真正要传的数据，例如一个存储字、一个数据块。
- **总耗时**：完成这次事务从开始到结束占用的全部时间，包括寻址、等待、恢复等。

如果总线频率为 `f`，则一个时钟周期为：

```text
T = 1 / f
```

如果一次事务共占 `C` 个周期：

```text
总耗时 = C × T = C / f
```

因此：

```text
有效传输率 = 有效数据量 / (C / f)
```

直觉上看，地址周期和等待周期虽然没有产生有效数据，但它们占着总线或拖慢了访问，所以必须计入分母。

### 3. 存储器读写题：不要只看总线，还要看存储器

有些题会同时给总线时钟和存储器周期。此时数据并不是“总线一空出来就能传”，因为存储器读出数据本身也要时间。

例如 PPT 例 8.4：

- 总线时钟频率 `1GHz`，所以一个总线周期是 `1ns`
- 数据总线宽度 `64 bit = 8B`
- 存储器存储周期为 2 个时钟周期，即 `2ns`
- CPU 读一个存储字

一次读操作可以拆成：

```text
寻址阶段：1ns
存储器读出并送上总线：2ns
CPU 从数据总线取走数据：1ns
总时间：4ns
```

所以：

```text
数据传输率 = 8B / 4ns = 2GB/s
```

这里的重点不是死记“4ns”，而是明白：读存储器必须先告诉存储器地址，存储器再准备数据，CPU 最后取走数据；这些阶段都消耗时间。

### 4. 突发传输：一次给地址，连续搬多份数据

普通传输如果每传一个字都要重新给地址，会浪费很多地址阶段时间。突发传输的思想是：

```text
只给一次首地址，后面连续传多个相邻数据
```

如果数据总线宽度为 `W bit`，连续传 `n` 个数据阶段，那么有效数据量为：

```text
有效数据量 = n × W / 8
```

如果总周期数为：

```text
地址周期 + 等待周期 + n 个数据周期 + 恢复周期
```

则：

```text
突发有效传输率 = n × W / 8 / 总时间
```

突发传输提高效率的原因不是让电线本身跑得更快，而是把多个数据共用一次地址阶段，减少了重复开销。数据块越长，地址阶段在总时间中的占比越小，效率越高。

### 5. 串行通信题：波特率看“线上所有位”，数据率只看“有效数据位”

异步串行通信中，一个字符通常不只包含数据位，还包括：

```text
起始位 + 数据位 + 校验位 + 停止位
```

这些位都要在线路上传输，所以：

```text
波特率 = 每秒传送的码元数
```

在课程题目中，若一个码元承载 1 bit，常可按 bit/s 计算。

若每个字符总共 `m` 位，每秒传 `N` 个字符：

```text
波特率 = m × N
```

如果每个字符只有 `d` 位是真正的数据位：

```text
有效数据传输率 = d × N
```

反过来，若已知波特率 `B`，每字符总位数为 `m`：

```text
字符速率 = B / m
```

典型直觉：起始位、校验位、停止位就像快递单、封条和分隔符，它们保证接收端能识别数据，但它们不是用户真正要的数据。

### 6. 高速串行总线：还要看编码效率和通路数

高速串行总线常常要对原始数据做编码，例如 8b/10b 编码。意思是：

```text
8 位有效数据会编码成 10 位在线路上传输
```

所以编码效率为：

```text
8 / 10 = 80%
```

如果有多条并发串行通路，例如 PCIe x4、x16，则多条通路可以同时传输。PPT 中给出的基本思路是：

```text
串行总线有效带宽 = 总线传输频率 × 编码效率 × 并发通路数
```

做题时要注意单位。如果题目给的是每条通路的原始速率，就先乘编码效率得到每条通路有效速率，再乘通路数。

## 典型例子

### 例 1：普通同步总线最大带宽

某同步总线宽度 32 位，频率 100MHz，每周期传 1 次数据。求最大带宽。

```text
32 位 = 4B
最大带宽 = 4B × 100MHz = 400MB/s
```

这里的 `400MB/s` 是理想值，默认每个周期都在传有效数据。

### 例 2：地址周期会降低有效传输率

某 32 位总线，频率 50MHz，突发读 8 个数据阶段，另有 1 个地址周期和 3 个等待周期。

```text
周期时间 = 1 / 50MHz = 20ns
有效数据量 = 8 × 32 / 8 = 32B
总周期数 = 1 + 3 + 8 = 12
总时间 = 12 × 20ns = 240ns
有效传输率 = 32B / 240ns ≈ 133.3MB/s
```

如果只用 `4B × 50MHz = 200MB/s`，算出来的是理想带宽，不是这次事务的有效传输率。

### 例 3：异步串行通信

异步串行传输一帧由 1 位起始位、7 位数据位、1 位奇偶校验位、1 位停止位组成，共 10 位。线路每秒传 120 个字符。

```text
波特率 = 10 位/字符 × 120 字符/秒 = 1200 波特
有效数据传输率 = 7 位/字符 × 120 字符/秒 = 840 bit/s
```

波特率大于有效数据率，因为线路上还传了起始位、校验位和停止位。

## 常见考法

- 给总线宽度、频率、DDR/QDR 倍数，求最大带宽。
- 给地址周期、等待周期、数据周期，求某次读写或突发传输的有效传输率。
- 比较改进前后的带宽提升倍数。
- 给异步串行帧格式和字符速率，求波特率与有效数据传输率。
- 给编码方式和并发通路数，求高速串行总线的有效带宽。
- 判断哪些措施能提高总线性能，例如增宽、提频、突发、减少等待周期、增加并发通路。

## 易错点

- **把 bit 当 Byte**：32 位总线一次传 4B，不是 32B。
- **只算数据周期，漏算地址和等待周期**：求有效传输率时，所有耗时都要进入分母。
- **把突发传输理解成频率变高**：突发传输主要是减少重复寻址开销。
- **把波特率等同于有效数据率**：有起始位、校验位、停止位时，两者通常不同。
- **忽略编码效率**：8b/10b 编码后，线路上传 10 位，只有 8 位是有效数据。
- **单位换算混乱**：教材题中 `MHz`、`MB/s` 通常按 `10^6` 估算；若题目特别说明 `MiB`，才按 `2^20`。

## 小结

- 最大带宽的核心公式是：`总线宽度 / 8 × 频率 × 单时钟传输次数`。
- 有效传输率的核心公式是：`有效数据量 / 总耗时`。
- 地址、等待、恢复、编码等开销不增加有效数据，但会降低有效传输率。
- 突发传输通过“一次地址，多次数据”提高效率。
- 串行通信题要分清波特率、字符速率和有效数据传输率。
