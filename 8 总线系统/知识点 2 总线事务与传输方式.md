# 总线事务与传输方式

> 来源依据：第 8 章 PPT“总线传输过程”“总线事务”“总线的信息传送”“并行传送”“串行传送”“高速串行总线编码效率”“并串行传送”。

## 一句话理解

总线事务描述一次完整通信从申请到结束的全过程；传输方式描述数据在这条通道上是并排走、排队走，还是多条串行通路一起走。

## 基本概念

总线不是“想传就传”。因为它是共享通道，多个主设备可能同时想用，所以一次传输必须有清楚的开始、过程和结束。

可以把一次总线通信想成去窗口办业务：

```text
先申请排队，轮到你后说明要办什么，再交接材料，办完离开窗口。
```

对应到总线中，就是一次**总线事务**。PPT 中的定义是：

```text
总线事务：总线上主从设备之间的一次信息交换过程。
```

一次典型事务包括：

- **请求阶段**：主设备申请使用总线。
- **寻址阶段**：主设备给出从设备地址或 I/O 端口地址，并发出相关命令。
- **传输阶段**：主设备和从设备进行数据交换。
- **结束阶段**：主设备撤销请求，释放总线。

注意，“总线周期”和“时钟周期”不是一回事。一个总线事务通常会占多个时钟周期，例如申请、地址、等待、数据、结束都可能各占时间。

## 工作原理或推导过程

### 1. 为什么总线事务要分阶段

一次读操作至少要解决三个问题：

```text
谁要用总线？访问哪个设备？数据什么时候有效？
```

所以事务必须分阶段：

1. **申请总线**：防止多个主设备同时驱动总线，造成冲突。
2. **发地址和命令**：让从设备知道“谁被访问”和“是读还是写”。
3. **传数据**：真正交换有效数据。
4. **释放总线**：让其他设备可以继续使用。

如果没有申请阶段，两个设备可能同时在数据线上输出不同电平；如果没有地址阶段，从设备不知道谁应该响应；如果没有结束阶段，总线控制器不知道什么时候能重新分配总线。

### 2. 常见总线事务类型

PPT 中列出的典型事务包括：

- **存储器读**：主设备给出主存地址，从主存读回数据。
- **存储器写**：主设备给出主存地址和数据，把数据写入主存。
- **I/O 读**：从 I/O 端口或设备寄存器读数据。
- **I/O 写**：向 I/O 端口或设备寄存器写数据。
- **中断响应**：CPU 响应中断，设备可能提供中断类型号等信息。
- **DMA 响应**：DMA 控制器取得总线控制权后进行数据传输。

这些事务的共同点是：都要通过地址、控制和数据等信息配合完成。

### 3. 突发传输：把重复寻址的开销摊薄

普通传输可能是：

```text
地址1 + 数据1
地址2 + 数据2
地址3 + 数据3
地址4 + 数据4
```

如果这些数据地址连续，每次都重新发地址很浪费。突发传输改成：

```text
首地址 + 数据1 + 数据2 + 数据3 + 数据4
```

也就是 PPT 中说的：

```text
一次寻址，多个数据传输阶段，传输过程不释放总线。
```

突发传输的本质是用一次地址阶段服务多个数据阶段。它特别适合 Cache 块填充、主存连续数据块读写、DMA 块传送等场景。

注意：突发传输不是“每根线传得更快”，而是“少做重复工作”。所以计算时仍然要按总线宽度、数据阶段数和总周期数来算有效传输率。

### 4. 并行传送：一次传多位，但高速和长距离有问题

并行传送的直觉是：

```text
多位数据各走一根线，同时到达。
```

例如 8 位并行总线需要 8 根数据线，一次可以传 8 bit。

优点：

- 一次传多位，低频或短距离时速度快。
- 接收端可以同时得到一个字节或一个字的多位。

缺点：

- 线多，成本高。
- 线与线之间会串扰。
- 高频时不同线的信号到达时间可能不一致，称为偏斜。
- 距离越长、频率越高，越难保证多位同时可靠到达。

因此早期很多总线强调并行宽度，而现代高速外部接口越来越多采用高速串行或多通路串行。

### 5. 串行传送：一位一位走，线少、距离长

串行传送的直觉是：

```text
只有一条或少数几条数据通路，数据按时间顺序一位一位传。
```

发送端通常要做并串转换，接收端再做串并转换：

```text
并行数据 -> 串行位流 -> 并行数据
```

串行传送的优点：

- 线少，成本低。
- 抗干扰能力较强。
- 适合长距离传输。
- 更容易做到很高频率。

串行传送的缺点：

- 单条通路一次只传 1 位，低频时带宽有限。
- 需要同步、编码、串并转换等机制。

串行通信按方向还可分为：

| 方式 | 含义 |
| --- | --- |
| 单工 | 只能单方向传输 |
| 半双工 | 双方都能传，但不能同时双向传 |
| 全双工 | 双方可以同时收发 |

### 6. 同步串行与异步串行

**同步串行传输**使用公共时钟或随路时钟来协调，信息帧可以包含多个字节。典型例子有 SPI、I2C。问题是距离长或频率高时，时钟和数据之间可能产生偏移。

**异步串行传输**不传公共时钟，而是双方事先约定波特率。接收端靠起始位识别一个字符的开始，再按约定节奏采样。典型例子是 RS-232。

异步串行帧常见格式：

```text
起始位 + 数据位 + 校验位 + 停止位
```

起始位告诉接收端“一个字符开始了”；停止位给接收端留出结束和准备下一帧的时间；校验位用于发现简单错误。

### 7. 波特率、字符速率和有效数据率

串行题最容易错在把所有速率混成一个。

- **波特率**：线路每秒传多少个码元。课程中如果一个码元表示 1 bit，可按 bit/s 理解。
- **字符速率**：每秒传多少个字符。
- **有效数据率**：每秒传多少个真正的数据位，不包括起始位、校验位、停止位等辅助位。

若每字符总位数为 `m`，其中数据位数为 `d`，每秒传 `N` 个字符：

```text
波特率 = m × N
有效数据率 = d × N
```

若已知波特率为 `B`：

```text
字符速率 = B / m
有效数据率 = B / m × d
```

直觉上，线路上所有位都占时间，所以波特率看“全帧”；用户真正关心的数据只占其中一部分，所以有效数据率通常更低。

### 8. 高速串行编码效率

高速串行总线不能只把 0 和 1 原样长时间连续发送，因为连续很多个 0 或 1 会让接收端难以恢复时钟，也会造成直流平衡问题。为此常用编码。

例如 **8b/10b 编码**：

```text
8 bit 有效数据 -> 10 bit 线路码
```

编码效率：

```text
8 / 10 = 80%
```

所以一条链路的原始线路速率不是全部有效数据速率。计算有效带宽时要乘编码效率。PPT 还提到部分高速串行总线采用 64b/66b、128b/130b，它们的编码开销更小。

### 9. 并串行传送：多条高速串行通路并发

单条串行线虽然可以很快，但一条通路带宽仍有限。现代高速总线常用“多条串行通路并发”：

```text
一条通路像一条车道，多条通路就是多车道同时跑。
```

PCIe x1、x4、x16 中的数字就是并发通路数。若每条通路有效速率相同，总有效带宽大致与通路数成正比。

PPT 给出的思路是：

```text
串行总线带宽 = 总线时钟频率 × 编码效率 × 并发通路数
```

这体现了现代总线的发展方向：不再单纯堆很多并行线，而是用更高频的串行通路，再用多通路并发提升总带宽。

### 10. 分时传送

分时传送是让同一组线在不同时间传不同信息，或让多个部件分时使用同一总线。

典型例子是地址/数据复用：

```text
某一时段传地址，另一时段传数据。
```

优点是减少线缆和引脚，降低成本；缺点是需要分阶段使用，可能增加总时间。

## 典型例子

### 例 1：突发读为什么更快

假设要读 4 个连续字。

普通传输：

```text
地址 + 数据
地址 + 数据
地址 + 数据
地址 + 数据
```

突发传输：

```text
首地址 + 数据 + 数据 + 数据 + 数据
```

如果地址阶段很耗时，突发传输能明显提高有效传输率，因为 4 个数据只付出 1 次寻址开销。

### 例 2：异步串行速率

一帧由 1 位起始位、7 位数据位、1 位奇偶校验位、1 位停止位组成，共 10 位。若线路每秒传 120 个字符：

```text
波特率 = 10 × 120 = 1200 波特
有效数据率 = 7 × 120 = 840 bit/s
```

这里不是 `1200 bit/s` 的有效数据率，因为每 10 位中只有 7 位是数据。

### 例 3：8b/10b 编码效率

某高速串行链路线路速率为 2.5Gb/s，使用 8b/10b 编码。

```text
有效数据率 = 2.5Gb/s × 80% = 2.0Gb/s
```

如果有 4 条并发通路：

```text
总有效数据率 = 2.0Gb/s × 4 = 8.0Gb/s
```

## 常见考法

- 按顺序写出一次总线事务的请求、寻址、传输、结束阶段。
- 判断存储器读、I/O 写、中断响应、DMA 响应属于哪类事务。
- 解释突发传输为什么能提高有效带宽。
- 比较并行传送和串行传送的优缺点。
- 根据异步串行帧格式计算波特率、字符速率、有效数据率。
- 根据编码效率和通路数计算高速串行总线有效带宽。
- 判断分时传送、地址/数据复用对成本和性能的影响。

## 易错点

- **把总线事务等同于数据阶段**：一次事务还包括请求、寻址和结束。
- **认为突发传输会释放总线再重新申请**：突发传输过程中通常不释放总线。
- **认为并行一定比串行快**：高速并行会受偏斜和串扰限制，现代高速串行可以非常快。
- **漏算异步帧的起始位和停止位**：它们不是有效数据，但占传输时间。
- **把编码后的线路速率当有效数据率**：8b/10b 编码要乘 80%。
- **把 PCIe x16 理解成 16 位并行总线**：它表示 16 条串行通路并发。

## 小结

- 总线事务是一次完整的信息交换过程，不只是传数据的一瞬间。
- 事务阶段的作用分别是申请使用权、定位从设备、交换数据和释放总线。
- 突发传输通过一次寻址传多个连续数据，减少重复开销。
- 并行传送一次多位，适合短距离低频；串行传送线少、抗干扰，适合高速长距离。
- 串行速率题必须分清波特率、字符速率和有效数据率。
- 高速串行总线要考虑编码效率和并发通路数。
