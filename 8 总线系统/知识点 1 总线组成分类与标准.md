# 总线组成、分类与标准

> 来源依据：第 8 章 PPT“总线概述”“总线分类”“总线组成”“总线标准与特性”。

## 一句话理解

总线就是计算机内部各部件共用的一套通信线路和通信规则，它解决的是 CPU、主存、I/O 设备之间怎样少连线、按规矩传信息的问题。

## 基本概念

如果每两个部件之间都单独连线，4 个部件就要很多连接，部件越多，线越乱，成本越高。总线结构的直觉很简单：

```text
大家不再两两单独修路，而是共用一条主干道。
```

这条“主干道”有两个特点：

- **共享**：多个部件都接在同一组线上。
- **分时**：同一时刻通常只能有一个主设备控制总线，其他设备等待。

所以总线带来两个结果：

- 好处是控制简单、成本低、扩展方便。
- 代价是大家抢同一条通道，总线可能成为性能瓶颈。

PPT 中把总线定义为：

```text
各功能部件间传递信息的公共通路。
```

注意，“公共通路”不只指物理导线，还包括接口、控制器、软件和通信协议。没有规则，大家同时说话就会冲突；有了协议，主设备什么时候发地址、从设备什么时候应答、数据什么时候有效，都能被约定清楚。

## 工作原理或推导过程

### 1. 总线由哪三类信号线组成

一件事要通过总线完成，至少要说清楚三个问题：

```text
访问谁？传什么？怎么控制？
```

对应三类总线：

| 信号线 | 解决的问题 | 典型内容 | 常见方向 |
| --- | --- | --- | --- |
| 地址总线 | 访问哪个位置 | 主存地址、I/O 端口地址 | 通常从主设备到从设备 |
| 数据总线 | 传输什么内容 | 指令、操作数、状态码、中断类型号 | 通常双向 |
| 控制总线 | 这次操作怎么进行 | 读/写、请求、许可、忙、应答、中断、时钟 | 视信号而定 |

例如 CPU 读主存：

```text
1. CPU 在地址总线上放出要读的主存地址。
2. CPU 在控制总线上发出“存储器读”信号。
3. 主存根据地址取出数据，把数据放到数据总线上。
4. CPU 从数据总线接收数据。
```

这里最重要的是分清“地址”和“数据”：

- 地址像门牌号，说明去哪里取。
- 数据像包裹内容，说明真正取到什么。
- 控制信号像交通灯和指令，说明现在是读、写、请求还是应答。

### 2. 主设备、从设备和总线主控

总线是共享的，因此必须有人发起一次传输。

- **主设备**：能主动发起总线事务的设备，例如 CPU、协处理器、DMA 控制器。
- **从设备**：被访问、被动响应的设备，例如主存、I/O 接口。
- **总线主控技术**：某些 I/O 设备内部带 DMA 控制器，可以暂时取得总线控制权，直接和主存交换数据。

直觉上，主设备像“提出办事申请的人”，从设备像“被要求提供数据的人”。CPU 不一定永远是主设备。DMA 传输时，DMA 控制器就可以成为总线主设备。

### 3. 总线复用：省线，但可能降速

总线复用是指同一组物理线在不同时间承担不同功能，例如地址线和数据线复用：

```text
前一个周期传地址，后一个周期传数据。
```

这样可以减少芯片引脚和连线数量，降低成本。但缺点也很直观：本来地址和数据可以分开走，现在要排队走，同一次事务可能需要更多周期，所以性能可能下降。

因此看到“地址/数据复用”时，不要自动认为它提高速度。它主要优化的是成本和引脚数量。

### 4. 总线分类：按不同角度给总线贴标签

PPT 给了多个分类角度。分类题不要死背一长串名字，要先问“分类依据是什么”。

| 分类依据 | 类型 | 理解方法 |
| --- | --- | --- |
| 传输方向 | 单向、双向 | 信息能不能双向流动 |
| 时序控制 | 同步、异步 | 靠统一时钟还是靠握手信号 |
| 信号类型 | 数据、地址、控制 | 线上传的是什么信息 |
| 传输位数 | 并行、串行 | 一次传多位还是逐位传 |
| 连接部件 | 片内、系统、I/O、外部、处理器总线 | 连接的是哪一层部件 |

按连接部件来理解：

- **片内总线**：CPU 芯片内部各功能单元之间的连接，例如寄存器、ALU、控制部件之间。
- **系统总线**：CPU、主存、I/O 部件之间的主要连接通道。
- **I/O 总线**：中低速 I/O 设备或 I/O 控制器之间的连接。
- **外部总线**：主机与外部设备之间的连接，例如 USB、RS-232。
- **处理器总线**：处理器与芯片组或其他高速部件之间的连接。

### 5. 总线标准：让不同厂家设备按同一套规则说话

总线标准规定了设备如何接入、信号怎样表示、时序如何配合。PPT 将它分成四类特性：

| 特性 | 规定什么 | 直觉理解 |
| --- | --- | --- |
| 物理特性 | 插头、插座、引脚数量、排列方式 | 插得上、连得对 |
| 电气特性 | 电平高低、有效电平、单端/差分、传输方向 | 信号能被正确识别 |
| 功能特性 | 每根信号线的用途 | 哪根线表示读，哪根线表示写 |
| 时间特性 | 各信号何时有效、保持多久 | 什么时候发地址，什么时候取数据 |

标准化的意义是：不同厂家生产的设备只要遵守同一标准，就能互换和协同工作。这也是 ISA、PCI、USB、PCIe 等总线能够推广的原因。

## 典型例子

### 例 1：CPU 读主存

```text
地址总线：CPU 发出要访问的主存地址
控制总线：CPU 发出“读”命令
数据总线：主存把该地址中的数据返回给 CPU
```

如果题目问“存储器读信号属于哪类总线”，答案是控制总线；如果问“返回的机器字走哪类总线”，答案是数据总线。

### 例 2：为什么总线需要标准

如果某设备认为低电平表示有效，而另一个设备认为高电平表示有效，它们即使物理上接在一起也无法正确通信。总线标准通过电气特性和时序特性把这些细节固定下来。

### 例 3：地址/数据复用

16 根复用线既传地址又传数据：

```text
第 1 个周期：线上放地址
第 2 个周期：线上放数据
```

这比单独准备 16 根地址线和 16 根数据线更省引脚，但一次访问至少要分阶段完成。

## 常见考法

- 判断某种信号属于地址总线、数据总线还是控制总线。
- 区分主设备和从设备，理解 DMA 为什么能成为总线主设备。
- 根据连接对象判断片内总线、系统总线、I/O 总线、外部总线。
- 说明总线共享结构的优点和缺点。
- 解释总线复用为什么能省线，以及为什么可能降低性能。
- 给出总线标准四类特性，说明标准化的意义。

## 易错点

- **把控制信号当数据**：读写信号、请求信号、应答信号、中断请求都属于控制信息。
- **认为 CPU 永远是主设备**：DMA 控制器也可以取得总线控制权。
- **认为总线共享一定更快**：共享结构成本低、扩展好，但同一时刻只能分时使用，容易形成瓶颈。
- **把地址线和数据线方向混淆**：地址通常由主设备发出，数据总线常是双向。
- **把复用理解成并行增加**：复用是同一组线分时承担多种功能，不是同时传更多数据。

## 小结

- 总线是连接多个功能部件的共享通信通道和规则集合。
- 地址总线说明“访问哪里”，数据总线说明“传什么”，控制总线说明“怎样传”。
- 主设备发起事务，从设备响应事务；DMA 控制器也可能成为主设备。
- 总线复用可以减少引脚和成本，但可能增加周期、降低有效速度。
- 总线标准通过物理、电气、功能、时序规范保证设备兼容。
