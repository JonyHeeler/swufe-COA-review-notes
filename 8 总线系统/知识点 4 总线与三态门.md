# 总线与三态门

> 来源依据：第 8 章 PPT“总线与三态门”。

## 一句话理解

三态门让设备在“不该说话”时彻底退出总线，只保留该说话的设备驱动线路，从而避免多个设备同时把同一根线拉成不同电平。

## 基本概念

总线是共享线路。共享线路最大的问题是：如果两个设备同时向同一根数据线输出信号，一个输出 1，另一个输出 0，会发生冲突。

普通数字电路输出通常只有两种状态：

```text
0：低电平
1：高电平
```

但接在总线上的输出电路还需要第三种状态：

```text
Z：高阻态
```

高阻态不是 0，也不是 1。它表示这个设备的输出端好像“断开”了一样，几乎不影响总线电平。

所以三态门的三个状态是：

| 状态 | 含义 | 直觉理解 |
| --- | --- | --- |
| 0 | 输出低电平 | 设备正在把总线拉低 |
| 1 | 输出高电平 | 设备正在把总线拉高 |
| Z | 高阻态 | 设备暂时退出，不驱动总线 |

三态门的控制信号常写作 `Enable` 或 `EN`：

```text
EN = 1：允许输出，Q = A
EN = 0：禁止输出，Q = Z
```

具体高有效还是低有效要看题目或电路符号，核心思想是：只有被选中的设备可以驱动总线，未被选中的设备必须处于高阻态。

## 工作原理或推导过程

### 1. 为什么总线必须配合三态门

假设有 4 个设备都接到同一条数据总线上。如果每个设备的输出端永远连着总线，那么只要两个设备输出不同值，总线电平就会冲突。

正确做法是：

```text
同一时刻，只允许一个设备输出到总线。
其他设备输出端进入高阻态。
```

这就是三态门在总线中的作用。

可以把三态门想成设备出口处的“闸门”：

- 闸门打开，设备可以把数据送到总线。
- 闸门关闭，设备和总线断开，不能影响总线。

### 2. 译码器如何选择哪个设备输出

PPT 图中出现了 `DeviceId`、译码器和多个设备使能信号。它们的逻辑是：

```text
DeviceId 指出当前选中哪个设备。
译码器把设备编号变成某一路使能信号。
只有对应设备的 EN 有效。
被选中设备通过三态门把数据送上 Data BUS。
其他设备处于高阻态。
```

例如 2 位设备编号可以选择 4 个设备：

| DeviceId | EN0 | EN1 | EN2 | EN3 | 被允许输出的设备 |
| --- | --- | --- | --- | --- | --- |
| 00 | 1 | 0 | 0 | 0 | D0 |
| 01 | 0 | 1 | 0 | 0 | D1 |
| 10 | 0 | 0 | 1 | 0 | D2 |
| 11 | 0 | 0 | 0 | 1 | D3 |

这样总线上永远只有一个设备输出，避免冲突。

### 3. 单向三态门和双向三态门

有些总线信号只需要单方向传，例如地址通常由主设备发向从设备。这时可以用单向三态缓冲器：

```text
A -> B
EN 有效时，A 的值传到 B。
EN 无效时，输出端为 Z。
```

数据总线常常需要双向传：

- CPU 写主存时，CPU 把数据送到总线，主存接收。
- CPU 读主存时，主存把数据送到总线，CPU 接收。

因此需要双向三态缓冲器。双向缓冲器除了使能信号，还要有方向控制信号：

```text
方向为 A -> B：A 端输出到 B 端。
方向为 B -> A：B 端输出到 A 端。
未使能：两端与总线隔离。
```

### 4. 三态门和总线仲裁的关系

总线仲裁决定“谁有资格使用总线”，三态门保证“只有有资格的人真的能输出到总线”。

两者配合起来才完整：

```text
仲裁器授权某主设备
控制逻辑打开该设备的三态门
其他设备三态门关闭，处于高阻态
被授权设备驱动总线完成传输
```

如果只有仲裁规则，但硬件上没有把其他设备隔离，总线仍可能冲突。

### 5. 输入设备是否需要高阻态

接收数据的设备可以同时“看”总线，因为读取电平通常不会主动改变总线电平。危险主要来自多个设备同时输出。

因此复习时要抓住：

```text
多个设备可以同时接收，但不能同时驱动同一条总线。
```

不过在实际接口电路中，输入、输出和方向控制常集成在总线收发器里，读写方向由控制信号统一决定。

## 典型例子

### 例 1：两个设备同时输出会怎样

设备 A 输出 1，设备 B 输出 0，如果两者输出端都直接连到同一根总线：

```text
A 想把线拉高
B 想把线拉低
```

结果可能是电平不确定、数据错误，甚至造成电路损坏。使用三态门后，未被选中的设备输出为 Z，不再和被选中设备争夺总线。

### 例 2：CPU 读主存时谁驱动数据总线

CPU 读主存：

```text
CPU 发地址和读控制信号。
主存被选中。
主存的数据输出三态门打开。
CPU 的数据输出三态门关闭，处于高阻态。
主存把数据送到数据总线，CPU 读取。
```

CPU 写主存时方向相反：

```text
CPU 的数据输出三态门打开。
主存接收数据。
主存的数据输出三态门关闭。
```

## 常见考法

- 判断三态门的三个输出状态：0、1、高阻态 Z。
- 解释为什么总线系统需要三态门。
- 根据使能信号判断某设备是否能把数据送上总线。
- 区分“接收总线数据”和“驱动总线数据”。
- 说明译码器如何选择某个设备输出到总线。
- 结合读写操作判断数据总线方向。

## 易错点

- **把高阻态当成 0**：高阻态不是低电平，而是近似断开，不驱动总线。
- **认为所有设备都能同时输出**：总线上同一时刻通常只能一个设备驱动同一组数据线。
- **只记三态门符号，不理解作用**：它的核心作用是让未选中设备退出共享线路。
- **混淆输入和输出**：多个设备可以监听总线，但多个设备不能同时输出到总线。
- **忽略方向控制**：双向数据总线既要控制是否使能，也要控制传输方向。

## 小结

- 三态门有 0、1、Z 三种输出状态。
- Z 表示高阻态，相当于设备输出端从总线上断开。
- 总线共享要求同一时刻只允许一个设备驱动总线。
- 译码器和使能信号用于选择哪个设备可以输出。
- 三态门是总线仲裁和总线共享能在硬件上成立的基础。
