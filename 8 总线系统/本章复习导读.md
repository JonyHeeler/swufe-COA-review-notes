# 第 8 章 总线系统复习导读

## 本章定位

第 8 章研究的是“计算机各部件怎样共用通信通道”。CPU、主存、I/O 设备不可能任意乱连，也不能同时往同一组线上输出信号，所以必须有总线结构、总线协议、总线仲裁和总线定时。

本章最核心的考试点是总线相关指标计算，但计算题背后的逻辑很简单：

```text
一次能传多少数据 × 每秒能传多少次 = 理想带宽
有效数据量 ÷ 完整事务耗时 = 实际有效传输率
```

复习时先把“公共通道”这个直觉建立起来，再去理解三态门、仲裁、突发传输、同步/异步和常用总线标准。

## 推荐复习顺序

1. 先读“总线组成、分类与标准”，建立总线是共享通信通道的整体图景。
2. 再读“总线与三态门”，理解硬件上怎样避免多个设备同时驱动总线。
3. 然后读“总线事务与传输方式”，掌握一次传输为什么要分请求、寻址、数据和结束阶段。
4. 接着重点读“总线性能指标与带宽计算”，把最大带宽、有效传输率、串行通信速率算熟。
5. 再读“总线仲裁、定时与系统结构”，解决谁先用、怎样同步、系统怎样分层连接。
6. 最后读“常用总线与发展趋势”，把 PCI、PCIe、USB 等标准放回总线发展主线中理解。

## 本章计算题抓手

遇到总线计算题，按这四步做：

1. **先找一次传多少**：总线宽度从 bit 换成 Byte，必要时乘数据阶段数。
2. **再找一秒多少次**：看频率、DDR/QDR、通路数、编码效率。
3. **然后找总耗时**：地址、等待、恢复、存储器周期都可能要算入。
4. **最后看题目问什么**：问最大带宽就用理想公式，问有效传输率就用有效数据量除以总时间。

最常见的两个公式：

```text
最大带宽 = 总线宽度 / 8 × 总线频率 × 单时钟传输次数
有效传输率 = 有效数据量 / 完整事务耗时
```

## 易错提醒

- 总线宽度是 bit，带宽常用 Byte/s，中间要除以 8。
- 地址周期、等待周期、恢复周期不传有效数据，但会降低有效传输率。
- 突发传输提高效率的原因是“一次地址，多次数据”，不是物理频率自动提高。
- 波特率统计线路上所有码元，有效数据率只统计真正的数据位。
- 高阻态 Z 不是 0，它表示设备输出端暂时退出总线。
- 链式查询硬件简单但优先级固定，独立请求速度快但控制线多。
- PCIe x16 表示 16 条串行通路并发，不是 16 位并行总线。
