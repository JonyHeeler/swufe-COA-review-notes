# CPU 组成与指令周期

> 来源依据：第 6 章 PPT“中央处理器概述”“CPU 中主要寄存器”“指令执行一般流程”“指令周期基本概念”。

## 一句话理解

CPU 的工作就是反复“取出下一条指令、看懂它、按它要求搬数据和算数据”，指令周期就是完成一条指令所花的全过程。

## 基本概念

### 1. CPU 到底负责什么

从程序员角度看，程序是一串指令；从硬件角度看，CPU 每次只面对一条二进制指令。CPU 要做的事情可以分成五类：

| 功能 | 通俗理解 | 对应部件 |
| --- | --- | --- |
| 数据加工 | 做加减、逻辑运算、比较 | 运算器、ALU |
| 程序控制 | 决定下一条执行哪条指令 | PC、控制器 |
| 操作控制 | 把机器指令翻译成控制信号 | 控制器 |
| 时序控制 | 决定哪个动作先做、哪个后做 | 时序发生器或状态机 |
| 异常控制 | 处理出错、外设请求、中断 | 中断异常逻辑 |

所以 CPU 不是单纯的“计算器”。它既要算，也要管流程、管时序、管异常。

### 2. CPU 内部主要部件

| 部件 | 作用 | 直觉解释 |
| --- | --- | --- |
| 运算器 | 完成算术和逻辑运算 | 真正“算”的地方 |
| 控制器 | 产生控制信号序列 | 指挥数据怎么走、什么时候走 |
| 寄存器组 | 保存临时数据和操作数 | CPU 内部最快的临时存储 |
| 数据通路 | 连接寄存器、ALU、存储器等 | 数据移动的道路系统 |

其中控制器非常关键。机器指令本身只是二进制编码，不能直接让硬件动起来。控制器要把它译码成一串控制信号，例如“PC 输出”“寄存器写使能”“ALU 做加法”“存储器读”。

### 3. 常见寄存器

| 寄存器 | 名称 | 保存什么 | 初学者理解 |
| --- | --- | --- | --- |
| PC | 程序计数器 | 下一条要取的指令地址 | 程序书签 |
| IR | 指令寄存器 | 当前正在译码或执行的指令 | 当前任务单 |
| MAR/AR | 存储器地址寄存器 | 要访问的主存地址 | 仓库货架号 |
| MDR/DR | 存储器数据寄存器 | 从主存读出的数据或将写入主存的数据 | 货物中转箱 |
| 通用寄存器 | 如 MIPS 的 `$0` 到 `$31` | 操作数、中间结果 | CPU 内部小变量 |
| PSW/状态寄存器 | 条件码、状态位等 | 结果是否为零、是否溢出、中断状态等 | 运算和系统状态标签 |

要注意：PC、通用寄存器通常和程序执行关系密切；MAR、MDR、IR 多数情况下是 CPU 内部使用，程序员通常不能直接访问。

## 工作原理或推导过程

### 1. 为什么必须先取指

CPU 不会“天然知道”下一步做什么。程序存放在主存中，PC 保存下一条指令的地址。执行一条指令前，CPU 必须先根据 PC 到主存取出指令。

典型取指过程可以写成：

```text
PC -> MAR
M[MAR] -> MDR
MDR -> IR
PC -> PC + 指令长度
```

MIPS 指令通常 32 位、按字节编址，因此顺序执行时常见更新为：

```text
PC = PC + 4
```

为什么是 `+4`？因为 32 位指令占 4 个字节，主存如果按字节编址，下一条指令地址比当前指令地址大 4。

### 2. 指令周期的直觉

一条指令从被取出到执行结束，完整时间叫**指令周期**。可以把它拆成：

```text
取指 -> 译码 -> 取操作数 -> 执行 -> 访存/写回 -> 更新 PC
```

不同指令并不一定每一步都需要。例如：

- `add` 不需要访问数据存储器。
- `lw` 需要访问数据存储器，并把读出的数据写回寄存器。
- `sw` 需要访问数据存储器，但不需要写回寄存器。
- `beq` 需要比较两个寄存器，并可能改变 PC。

所以，不同指令的数据通路和执行时间会不同。

### 3. 时钟周期、机器周期、指令周期的关系

这三个概念很容易混：

| 时间单位 | 含义 | 类比 |
| --- | --- | --- |
| 时钟周期 | CPU 最小同步节拍 | 一下节拍器 |
| 机器周期 / CPU 周期 | 完成一个相对完整的基本操作 | 一小段动作 |
| 指令周期 | 完成一条指令的全过程 | 做完一个任务 |

传统说法中，一个指令周期可以包含多个机器周期，一个机器周期可以包含多个时钟节拍：

```text
指令周期 = 若干机器周期
机器周期 = 若干时钟周期或节拍
```

现代处理器或 MIPS 多周期实现中，经常直接按时钟周期划分状态，不一定严格使用传统三级时序概念。但考试中仍然会考三者区别。

### 4. 定长和变长指令周期

CPU 时间控制有两种直觉：

| 控制方式 | 做法 | 优点 | 缺点 |
| --- | --- | --- | --- |
| 定长指令周期 | 所有指令都按同样长的周期完成 | 控制简单 | 简单指令浪费时间 |
| 变长指令周期 | 指令复杂就多用几个周期，简单就少用几个周期 | 更灵活，浪费少 | 控制更复杂 |

单周期 MIPS 可以看成定长指令周期的典型例子：所有指令都 1 个周期完成。多周期 MIPS 则是变长思想：`lw` 多走几个周期，`beq` 少走几个周期。

## 典型例子

### 例 1：`lw rt, imm(rs)` 的执行过程

以 `lw $s0, 32($s1)` 为例，它的含义是：

```text
$s0 = M[$s1 + 32]
```

CPU 的思考过程是：

1. 先取出这条 `lw` 指令，放入 IR。
2. 译码后知道它是 load 指令，需要读寄存器 `$s1`。
3. 用 ALU 计算有效地址：`$s1 + 32`。
4. 按这个地址访问数据存储器，读出数据。
5. 把读出的数据写回 `$s0`。

这说明 `lw` 为什么比普通 `add` 复杂：它不是只做加法，还要用加法结果当地址去存储器读数据。

### 例 2：PC 位数怎么判断

如果主存按字节编址，地址空间大小为 `2^32` 字节，那么 PC 至少需要 32 位，才能表示所有字节地址。

如果指令固定 32 位，顺序执行时 PC 常加 4，但这不代表 PC 只保存“第几条指令”。PC 保存的是地址，地址单位由编址方式决定。

## 常见考法

- 写出 CPU 的基本组成和各部件功能。
- 区分 PC、IR、MAR、MDR、PSW 的作用。
- 判断哪些寄存器对程序员可见。
- 说明取指周期的基本微操作。
- 区分指令周期、机器周期、时钟周期。
- 判断 MIPS 顺序执行时 PC 为什么加 4。
- 根据主存容量和编址方式判断 PC 或 MAR 位数。

## 易错点

- PC 通常保存“下一条要取的指令地址”，不一定是“当前刚执行完的指令地址”。
- MIPS 中 PC 加 4 的前提是 32 位定长指令且主存按字节编址。
- IR 保存的是指令本身，MAR 保存的是地址，MDR 保存的是数据，不要混。
- `sw` 的数据方向是寄存器到存储器，`lw` 的数据方向是存储器到寄存器。
- 指令周期不是固定等于一个时钟周期；只有单周期处理器才规定每条指令一个周期完成。
- 机器周期、节拍、时钟周期在不同教材和实现中叫法可能略有差异，做题时优先看题目定义。

## 小结

- CPU 通过“取指、译码、执行”的循环运行程序。
- PC 是下一条指令地址，IR 是当前指令，MAR/MDR 是访问主存时的地址和数据中转寄存器。
- 指令周期是一条指令的完整执行时间，时钟周期是 CPU 的基本节拍。
- 不同指令需要的数据通路不同，所以执行时间和控制信号也不同。
