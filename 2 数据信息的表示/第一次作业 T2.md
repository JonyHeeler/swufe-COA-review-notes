# 第一次作业 第二题

## 原题

> 有如下 C 语言程序段：
> 
> ```c
> short si = -32767;
> unsigned short usi = si;
> ```
> 
> 执行上述两条语句后，`usi` 的值为多少？

## 涉及知识点与公式

这道题主要涉及两个知识点。

第一，来自 **`核心知识点 1 数制转换与原码反码补码移码.md`** 的“真值与机器码”以及“补码”。

真值是人按数学意义理解的数，例如 `-32767`；机器码是计算机内部实际保存的二进制位。也就是说，同一串二进制位，按不同规则解释，可能得到不同的真值。

对于补码，负数的机器码可以理解为：

```text
负数 x 的 n 位补码 = 2^n + x
```

或者反过来：如果一个 n 位补码的最高位是 `1`，可以先按无符号数算出它的值 `U`，再减去 `2^n`，得到它作为有符号数时的真值。

```text
有符号真值 = U - 2^n
```

第二，涉及 C 语言中 **有符号整数转换为无符号整数** 的规则。

当一个有符号整数赋给同位数的无符号整数时，最终结果不是简单“去掉负号”，而是按照无符号类型的取值范围进行模运算。对于 16 位 `unsigned short`，取值范围是：

```text
0 ~ 65535
```

模数是：

```text
2^16 = 65536
```

所以：

```text
(unsigned short)(-32767) = -32767 mod 65536
                         = 65536 - 32767
                         = 32769
```

## 解题思路与答案

先明确题目中的类型：

```c
short si = -32767;
unsigned short usi = si;
```

在计算机组成原理这类题目中，`short` 通常按 **16 位** 处理，`unsigned short` 也按 **16 位** 处理。

`si = -32767` 是一个有符号 `short` 类型变量。16 位补码的范围是：

```text
-32768 ~ +32767
```

所以 `-32767` 可以被 16 位 `short` 正常表示。

接下来求 `-32767` 的 16 位补码。

```text
-32767 的 16 位补码 = 2^16 - 32767
                    = 65536 - 32767
                    = 32769
```

把 `32769` 转成十六进制：

```text
32769 = 8001H
```

写成 16 位二进制就是：

```text
1000 0000 0000 0001
```

因此，`si` 在机器内部保存的位模式是：

```text
si = 1000 0000 0000 0001B
```

然后执行：

```c
unsigned short usi = si;
```

这一步的关键是：`usi` 是无符号类型。赋值之后，位模式仍然可以理解为同一串 16 位二进制：

```text
1000 0000 0000 0001B
```

但是解释规则变了。现在它不再按有符号补码解释，而是按无符号整数解释。

无符号整数中，每一位都是数值位，所以：

```text
1000 0000 0000 0001B
= 2^15 + 1
= 32768 + 1
= 32769
```

所以最终答案是：

```text
usi = 32769
```

| **1. `short si = -1; unsigned short usi = si;`**                   | 16 位补码中，`-1` 的机器码是 `1111 1111 1111 1111B`，按无符号解释为 `65535`。                  | `65535`                          |
| **2. `short si = -2; unsigned short usi = si;`**                   | 按模运算：`-2 mod 65536 = 65534`。                                                | `65534`                          |
| **3. `short si = -32768; unsigned short usi = si;`**               | `-32768` 的 16 位补码是 `1000 0000 0000 0000B`，按无符号解释为 `32768`。                  | `32768`                          |
| **4. `short si = -32767;`，问 `si` 的机器码**                            | `-32767` 的 16 位补码为 `2^16 - 32767 = 32769 = 8001H`。                          | `8001H`，即 `1000 0000 0000 0001B` |
| **5. `unsigned short usi = 32769;`，再赋给 `short si`，问 `si` 的值**      | `32769` 的 16 位机器码是 `8001H`。若按 16 位补码解释，最高位为 1，真值为 `32769 - 65536 = -32767`。 | `-32767`                         |
| **6. `unsigned short usi = 65535;`，再赋给 `short si`，问常见机器中 `si` 的值** | `65535` 的机器码是全 1，按 16 位补码解释为 `-1`。                                          | 常见补码机器中为 `-1`                    |
| **7. 问 `-32767` 的 16 位补码对应十六进制是多少**                                | `-32767 mod 65536 = 32769 = 8001H`。                                         | `8001H`                          |
| **8. 问 `8001H` 按无符号数解释是多少**                                        | `8001H = 8×16^3 + 1 = 32768 + 1 = 32769`。                                   | `32769`                          |
| **9. 问 `8001H` 按 16 位补码解释是多少**                                     | 最高位为 1，先按无符号算为 `32769`，再减 `65536`：`32769 - 65536 = -32767`。                 | `-32767`                         |
| **10. 如果是 8 位：`signed char c = -1; unsigned char uc = c;`**        | 8 位无符号模数为 `2^8 = 256`，所以 `-1 mod 256 = 255`。                                | `255`                            |

## 同类题通用模板

遇到这类 **signed 转 unsigned** 的题，可以按下面步骤做。

第一步，确定位数。

例如：

```text
short / unsigned short：通常按 16 位
char / unsigned char：通常按 8 位
int / unsigned int：要看题目或机器设定
```

第二步，确定无符号类型的模数。

```text
n 位无符号整数的模数 = 2^n
```

如果是 16 位 `unsigned short`：

```text
模数 = 2^16 = 65536
```

第三步，把负数转换到无符号范围内。

```text
负数 x 转 n 位 unsigned = x mod 2^n
```

对于本题：

```text
-32767 mod 65536 = 32769
```

第四步，如果题目从机器码角度考，也可以先写补码，再按无符号解释。

```text
-32767 的 16 位补码：1000 0000 0000 0001B
按 unsigned short 解释：32769
```

## 最终答案

```text
usi 的值为 32769。
```
