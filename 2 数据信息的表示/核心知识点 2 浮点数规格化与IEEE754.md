# 浮点数规格化与 IEEE754

> 来源依据：`知识点汇总.txt` 第二章第 2 点；第 2 章 PPT“浮点数规格化”“IEEE754”；教材第 2 章。

## 一句话理解

浮点数就是二进制科学计数法：用“有效数字 × 2 的若干次方”来表示很大或很小的数。

## 基本概念

定点数的小数点位置固定，表示范围有限。浮点数的“小数点位置”可以随阶码移动，所以能表示更大的范围。

浮点数通常写成：

```text
N = (-1)^S × M × 2^E
```

其中：

| 字段 | 含义 | 直观作用 |
| --- | --- | --- |
| `S` | 符号位 | 决定正负 |
| `M` | 尾数 | 保存有效数字 |
| `E` | 阶码 | 决定小数点移动多少位 |

十进制科学计数法把 `12345` 写成 `1.2345×10^4`。二进制浮点数类似，只是底数从 10 换成 2。

IEEE754 单精度浮点数共 32 位：

| 字段 | 位数 | 含义 |
| --- | ---: | --- |
| 符号位 `S` | 1 | `0` 表示正，`1` 表示负 |
| 阶码字段 `Exponent` | 8 | 用偏置表示实际阶码 |
| 尾数字段 `Fraction` | 23 | 保存规格化尾数小数点后的部分 |

规格化数默认尾数长成：

```text
1.xxxxx × 2^E
```

因为规格化后二进制最高有效位一定是 `1`，IEEE754 不把这个 `1` 存进去，而是默认它存在。这叫**隐藏位**，相当于白赚 1 位精度。

## 工作原理或推导过程

### 1. 为什么要规格化

同一个数可以有多种写法：

```text
1000.01B = 1.00001B × 2^3
         = 10.0001B × 2^2
         = 0.100001B × 2^4
```

如果不规定统一形式，同一个数会有很多编码。规格化要求写成 `1.xxxxx × 2^E`，这样表示更统一，也能让尾数字段尽量保存有效数字。

### 2. 单精度转换步骤

十进制数转 IEEE754 单精度，一般按五步：

1. 判断正负，确定符号位 `S`。
2. 把绝对值转成二进制。
3. 写成 `1.xxxxx × 2^E` 的规格化形式。
4. 阶码字段写成 `E + 127` 的 8 位二进制。
5. 尾数字段写小数点后面的部分，不足 23 位补 0，超过 23 位按舍入规则处理。

为什么阶码要加 127？因为实际阶码可能是负数。加偏置后，保存的阶码字段变成非负二进制数，便于编码和比较。

### 3. 单精度字段含义

规格化数的实际值为：

```text
(-1)^S × (1.Fraction) × 2^(Exponent - 127)
```

特殊值要单独记：

| 阶码字段 | 尾数字段 | 含义 |
| --- | --- | --- |
| 全 0 | 全 0 | 正零或负零 |
| 全 0 | 非 0 | 非规格化数 |
| 1 到 254 | 任意 | 规格化数 |
| 全 1 | 全 0 | 正无穷或负无穷 |
| 全 1 | 非 0 | NaN |

## 典型例子

### 例 1：`-8.25` 转 IEEE754 单精度

先转二进制：

```text
8 = 1000B
0.25 = 0.01B
8.25 = 1000.01B
```

规格化：

```text
1000.01B = 1.00001B × 2^3
```

拆字段：

```text
S = 1
Exponent = 3 + 127 = 130 = 10000010B
Fraction = 00001000000000000000000
```

合起来：

```text
1 10000010 00001000000000000000000
```

按 4 位一组转十六进制：

```text
1100 0001 0000 0100 0000 0000 0000 0000
= C1040000H
```

### 例 2：为什么很多十进制小数不能精确表示

`0.5` 可以精确表示，因为：

```text
0.5 = 0.1B
```

但 `0.1` 十进制转换成二进制会出现循环小数，有限位尾数装不下，只能舍入。所以浮点数范围大，不等于所有小数都精确。

## 常见考法

- 给十进制数，求 IEEE754 单精度二进制或十六进制表示。
- 给单精度十六进制，拆字段求真值。
- 判断最大规格化正数、最小规格化正数、最小非规格化正数。
- 分析 `float` 与 `double` 的精度差异。
- 判断浮点数比较、舍入和溢出问题。

## 易错点

- 单精度偏置值是 `127`，不是 `128`。
- 规格化数尾数隐藏最高位 `1`，尾数字段只保存小数点后的部分。
- 阶码字段全 0 和全 1 有特殊含义，不能按普通规格化数解释。
- 最小规格化正数是 `1.0×2^-126`，最小非规格化正数是 `2^-149`。
- 浮点数能表示很大范围，但精度由尾数位数决定，不是无限精确。

## 小结

- 浮点数三要素是符号、阶码、尾数。
- 规格化让表示唯一，并利用隐藏位提高精度。
- IEEE754 单精度计算题按“转二进制、规格化、阶码加 127、写尾数”即可。
- 范围主要看阶码位数，精度主要看尾数位数。
