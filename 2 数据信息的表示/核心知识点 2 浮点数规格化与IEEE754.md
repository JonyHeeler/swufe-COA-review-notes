# 浮点数规格化与 IEEE754

> 来源依据：`知识点汇总.txt` 第二章第 2 点；第 2 章 PPT“浮点数规格化”“IEEE754”；教材第 2 章。

## 一句话理解

浮点数就是二进制科学计数法：用“有效数字 × 2 的若干次方”来表示很大或很小的数。

## 基本概念

定点数的小数点位置固定，表示范围有限。浮点数的“小数点位置”可以随阶码移动，所以能表示更大的范围。

浮点数通常写成：

```text
N = (-1)^S × M × 2^E
```

其中：

| 字段  | 含义  | 直观作用       |
| --- | --- | ---------- |
| `S` | 符号位 | 决定正负       |
| `M` | 尾数  | 保存有效数字     |
| `E` | 阶码  | 决定小数点移动多少位 |

十进制科学计数法把 `12345` 写成 `1.2345×10^4`。二进制浮点数类似，只是底数从 10 换成 2。

IEEE754 单精度浮点数共 32 位：

| 字段              | 位数  | 含义                                                                                                          |
| --------------- | ---:| ----------------------------------------------------------------------------------------------------------- |
| 符号位 `S`         | 1   | `0` 表示正，`1` 表示负                                                                                             |
| 阶码字段 `Exponent` | 8   | 对应科学计数法里“2的几次方”。但有个小套路：因为次方可能是负数（比如表示很小的数），为了不存负号，计算机规定存进阶码的值要加上 127（这叫偏置）。比如实际指数是 3，存进去就是 $3 + 127 = 130$。 |
| 尾数字段 `Fraction` | 23  | 对应科学计数法里保存有效数字的部分。                                                                                          |

为了让同一个数只有一种唯一的编码写法，计算机强制规定，二进制科学计数法的小数点前面必须是 `1`（也就是统一写成 $1.xxxxx \times 2^E$）。既然大家开头第一位全都是 `1`，为了节约宝贵的存储空间，IEEE754 规定：**这个 `1` 就不存进去了**，只存小数点后面的 $xxxxx$。这相当于我们凭空白赚了 1 位的精度，这种做法叫“隐藏位”。

## 工作原理或推导过程

### 1. 为什么要规格化

同一个数可以有多种写法：

```text
1000.01B = 1.00001B × 2^3
         = 10.0001B × 2^2
         = 0.100001B × 2^4
```

如果不规定统一形式，同一个数会有很多编码。规格化要求写成 `1.xxxxx × 2^E`，这样表示更统一，也能让尾数字段尽量保存有效数字。

### 2. 单精度字段含义

规格化数的实际值为：

```text
(-1)^S × (1.Fraction) × 2^(Exponent - 127)
```

特殊值要单独记：

| 阶码字段    | 尾数字段 | 含义      |
| ------- | ---- | ------- |
| 全 0     | 全 0  | 正零或负零   |
| 全 0     | 非 0  | 非规格化数   |
| 1 到 254 | 任意   | 规格化数    |
| 全 1     | 全 0  | 正无穷或负无穷 |
| 全 1     | 非 0  | NaN     |

## 典型例子

### 例 1：`-8.25` 转 IEEE754 单精度

判断十进制数的正负。正数为 `0`，负数为 `1`。因为是 `-8.25`，所以符号位 **$S = 1$**。

转二进制：

```text
8 = 1000B
0.25 = 0.01B
8.25 = 1000.01B
```

规格化。

- 移动小数点，让二进制数变成 $1.xxxxx \times 2^E$ 的形式。要求小数点前必须只能是一个 `1`。

- **例题应用**：`1000.01B` 的小数点需要向左移动 3 位，变成 $1.00001B \times 2^3$。此时我们记录下实际指数 **$E = 3$**。

```text
1000.01B = 1.00001B × 2^3
```

拆字段：

```text
S = 1
Exponent = 3 + 127 = 130 = 10000010B
Fraction = 00001000000000000000000
```

合起来：

```text
1 10000010 00001000000000000000000
```

按 4 位一组转十六进制：

```text
1100 0001 0000 0100 0000 0000 0000 0000
= C1040000H
```

### 例 2：为什么很多十进制小数不能精确表示

如果你经常使用 C++ 或 Python 编写算法代码，可能偶尔会遇到像 `0.1 + 0.2 == 0.3` 这样的条件判断最终返回 `False` 的诡异现象。要理解为什么有的数能精确表示，有的数却会变成“无限循环小数”，我们需要回归到进制的本质：**权重**。

计算机的世界里只有二进制，小数点后每一位的权重变成了 $2^{-1}$（0.5）、$2^{-2}$（0.25）、$2^{-3}$（0.125）、$2^{-4}$（0.0625）……**计算机要想表示一个小数，就必须只用 0.5、0.25、0.125 这些“二进制积木”来拼凑。**

- **为什么 `0.5` 能精确表示？**
  
  因为它刚好等于 $2^{-1}$。计算机只需要取一块“0.5”的积木就行了。所以转换成二进制就是 `0.1B`。

- **为什么 `0.1` 不能精确表示？**

我们在前面总结过十进制小数转二进制的方法：“乘基取整，顺序排列”。如果我们对 `0.1` 执行这个操作，你会看到一个无解的死循环：

- $0.1 \times 2 = 0.2$ （取整数部分 **0**，剩下 0.2）

- $0.2 \times 2 = 0.4$ （取整数部分 **0**，剩下 0.4）

- $0.4 \times 2 = 0.8$ （取整数部分 **0**，剩下 0.8）

- $0.8 \times 2 = 1.6$ （取整数部分 **1**，剩下 0.6）

- $0.6 \times 2 = 1.2$ （取整数部分 **1**，剩下 0.2）

- **注意！这里又剩下了 0.2，回到了第二步的起点！**

这就意味着接下来的计算将永远是 `0011 0011 0011...` 这样循环下去。所以十进制的 `0.1` 转换成二进制是：

`0.0 0011 0011 0011 0011...B`

# 

虽然数学上它是无限循环的，但在 IEEE 754 单精度浮点数标准中，**尾数字段（Fraction）只有 23 位**。

计算机的存储空间是有限的，装不下无限长的数字。当这 23 位填满后，计算机别无选择，只能像用剪刀一样把后面的数字“咔嚓”一刀切掉（进行舍入）。

这一刀切下去，你保存下来的数值就不再是绝对精确的 `0.1` 了，而是极其接近 `0.1` 的一个近似值。这就是为什么说浮点数范围大，但不等于所有小数都精确。

## 常见考法

- 给十进制数，求 IEEE754 单精度二进制或十六进制表示。
- 给单精度十六进制，拆字段求真值。
- 判断最大规格化正数、最小规格化正数、最小非规格化正数。
- 分析 `float` 与 `double` 的精度差异。
- 判断浮点数比较、舍入和溢出问题。

## 易错点

- 单精度偏置值是 `127`，不是 `128`。
- 规格化数尾数隐藏最高位 `1`，尾数字段只保存小数点后的部分。
- 阶码字段全 0 和全 1 有特殊含义，不能按普通规格化数解释。
- 最小规格化正数是 `1.0×2^-126`，最小非规格化正数是 `2^-149`。
- 浮点数能表示很大范围，但精度由尾数位数决定，不是无限精确。

## 小结

- 浮点数三要素是符号、阶码、尾数。
- 规格化让表示唯一，并利用隐藏位提高精度。
- IEEE754 单精度计算题按“转二进制、规格化、阶码加 127、写尾数”即可。
- 范围主要看阶码位数，精度主要看尾数位数。
