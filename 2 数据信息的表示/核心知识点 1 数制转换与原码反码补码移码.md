# 数制转换与原码、反码、补码、移码

> 来源依据：`知识点汇总.txt` 第二章第 1 点；第 2 章 PPT“进制转换”“原码/反码/补码/移码”；教材第 2 章。

## 一句话理解

这一知识点解决的是：一个数学上的数，怎样变成计算机内部固定长度的二进制编码。

## 基本概念

先分清两个词：

- **真值**：人按数学意义理解的数，例如 `+9`、`-9`、`0.75`。
- **机器码**：机器内部实际保存的二进制位，例如 `00001001`、`11110111`。

同一个机器码，按不同规则解释，可能得到不同真值。比如 8 位 `11111111`：

| 解释规则  | 真值   |
| ----- | ----:|
| 无符号整数 | 255  |
| 原码整数  | -127 |
| 反码整数  | -0   |
| 补码整数  | -1   |

下面是针对这个神奇现象的解释：

**1. 无符号整数 (Unsigned Integer)：结果 255**

- **规则**：所有 8 位全都是数值位，没有符号的概念。

- **计算**：`11111111` 代表 $128 + 64 + 32 + 16 + 8 + 4 + 2 + 1 = 255$。这是 8 位无符号数能表示的最大值。

**2. 原码整数 (Sign-Magnitude)：结果 -127**

- **规则**：最高位专门放符号（0 为正，1 为负），其余 7 位放绝对值。

- **计算**：
  
  - 最高位是 `1`，说明这是一个**负数**。
  
  - 剩下的数值位是 `1111111`，转换成十进制是 127。
  
  - 合起来就是 **-127**。

**3. 反码整数 (Ones' Complement)：结果 -0**

- **规则**：对于负数，符号位保持为 1 不变，数值位按位取反（0 变 1，1 变 0）才能得到绝对值。

- **计算**：
  
  - 最高位是 `1`，说明是**负数**。
  
  - 剩下的 7 个 `1` 取反后变成 `0000000`，绝对值是 0。
  
  - 合起来就是 **-0**。*(这也解释了为什么原码和反码都有 +0 和 -0，浪费了一个编码位置)*。

**4. 补码整数 (Two's Complement)：结果 -1**

- **规则**：负数的补码是在其反码的基础上加 1。

- **计算（方法一：逆向推导）**：
  
  - 最高位 `1`，是**负数**。
  
  - 先减 1 得到反码：`11111111 - 1 = 11111110`。
  
  - 再把数值位取反得到绝对值：`0000001`，也就是 1。
  
  - 加上负号，结果是 **-1**。

- **计算（方法二：模运算捷径）**：
  
  - 讲义中提到一个很实用的方法：如果最高位是 1，就先按无符号数算出 U，再减去 $2^n$。
  
  - 无符号值是 255。
  
  - $255 - 2^8 = 255 - 256 =$ **-1**。

几种常见编码可以这样理解：

| 编码  | 直观想法             | 正数规则          | 负数规则          |
| --- | ---------------- | ------------- | ------------- |
| 原码  | 最高位专门放符号，其余位放绝对值 | 符号位 0，数值位写绝对值 | 符号位 1，数值位写绝对值 |
| 反码  | 为补码做过渡           | 与原码相同         | 符号位不变，数值位取反   |
| 补码  | 让减法能用加法电路完成      | 与原码相同         | 反码加 1         |
| 移码  | 把有符号数整体平移成非负编码   | 真值加偏置         | 真值加偏置         |

## 工作原理或推导过程

### 1. 进制转换怎么想

整数部分用“除基取余，逆序排列”。原因是每次除以基数，余数就是当前最低位。

```text
十进制 13 转二进制：
13 ÷ 2 = 6 余 1
 6 ÷ 2 = 3 余 0
 3 ÷ 2 = 1 余 1
 1 ÷ 2 = 0 余 1

逆序得到：1101B
```

小数部分用“乘基取整，顺序排列”。原因是每次乘以基数，整数部分就是当前下一位小数位。

```text
0.625 转二进制：
0.625 × 2 = 1.25  取 1
0.25  × 2 = 0.5   取 0
0.5   × 2 = 1.0   取 1

顺序得到：0.101B
```

二进制和八进制、十六进制之间更简单：

```text
3 位二进制对应 1 位八进制
4 位二进制对应 1 位十六进制
```

### 2. 为什么原码和反码范围少一个

假设总共 `n` 位，其中最高位是符号位，剩下 `n-1` 位表示数值大小。

数值位最多表示：

```text
2^(n-1)-1
```

所以原码和反码的正负范围都是：

```text
-(2^(n-1)-1) ~ +(2^(n-1)-1)
```

它们少一个数，是因为有两个 0：

```text
原码：00000000 表示 +0，10000000 表示 -0
反码：00000000 表示 +0，11111111 表示 -0
```

两个编码都用来表示 0，就浪费了一个位置。

### 3. 为什么补码范围是 `-2^(n-1) ~ +(2^(n-1)-1)`

补码只有一个 0，所以比原码、反码多出来一个编码。这个多出来的编码被用来表示最小负数。

以 8 位补码为例：

```text
00000000 = 0
00000001 = 1
...
01111111 = 127
10000000 = -128
10000001 = -127
...
11111111 = -1
```

判断 8 位补码负数有一个很实用的方法：如果最高位是 1，就先按无符号数算出 `U`，再减去 `2^8`：

```text
11110111B = 247
247 - 256 = -9
```

### 4. 移码为什么常用于阶码

移码的核心是“整体平移”。如果要表示有正有负的阶码，可以把它们统一加上一个偏置，使保存下来的编码都变成非负数。

例如用 8 位 IEEE754 阶码字段时，偏置值是 127：

```text
实际阶码 E = 3
阶码字段 = 3 + 127 = 130 = 10000010B
```

这样做的好处是比较阶码大小更方便，也避免直接保存负号。

## 典型例子

### 例 1：8 位补码表示 `-9`

先写 `+9`：

```text
+9 = 00001001
```

对于一个**负数的原码**，求它的补码时建议使用扫描法：

> **符号位不变，从数值位最右边开始向左扫描；遇到第一个 1 及其右边的所有位保持不变，这个 1 左边的数值位全部取反。**

也可以简化记成：

> **从右往左找第一个 1：第一个 1 和它右边不变，左边取反。**

注意：这是对**数值位**操作，**符号位不参与扫描**。

按照这个方法，+9 的补码就是 11110111。

### 例 2：8 位补码 `10000000` 的真值

最高位是 1，说明按补码解释是负数。先按无符号算：

```text
10000000B = 128
128 - 256 = -128
```

所以它表示 `-128`。这也是为什么 8 位补码能表示 `-128`，但不能表示 `+128`。

## 常见考法

- 给真值，写原码、反码、补码。
- 给机器码，按指定编码求真值。
- 给定位数，求无符号、原码、反码、补码表示范围。
- 判断原码、反码、补码中 0 的表示个数。
- 结合 C 语言 signed/unsigned 做位模式解释题。

## 易错点

- 做任何编码题，第一步先确定位数。`-9` 的 8 位补码和 16 位补码写法不同。
- 原码、反码都有 `+0` 和 `-0`，补码只有一个 0。
- 补码最小负数没有对应的正数，例如 8 位补码有 `-128`，没有 `+128`。
- 负数补码求真值时，“取反加 1”得到的是绝对值，最后要加负号。
- IEEE754 阶码偏置是 `2^(e-1)-1`，不要和普通 n 位移码偏置机械混淆。

## 小结

- 原码最直观，反码是过渡，补码最适合硬件加减法，移码常用于阶码。
- 补码范围不对称，是因为补码只有一个 0，多出来的编码给了最小负数。
- 机器码题的关键是：位数、编码规则、真值解释三件事必须同时看。
