# C 语言中的数据表示转换

> 来源依据：`知识点汇总.txt` 第二章第 4 点；第 2 章 PPT“C 语言中的数据类型”“变量内存值”；教材 P131 例 4.4 的跨章节应用。

## 一句话理解

C 语言类型转换题的关键是：很多时候内存里的二进制位没有变，只是解释这串位的规则变了。

## 基本概念

在本课程常用的补码机器模型下，带符号整数用补码表示，无符号整数按普通二进制权值解释。

同样的位模式，解释方式不同，数值就不同：

```text
11111111
```

| 解释方式 | 数值 |
| --- | ---: |
| 8 位 unsigned | 255 |
| 8 位 signed 补码 | -1 |

常见转换规则：

| 转换场景 | 位模式怎样变 | 数值怎样解释 |
| --- | --- | --- |
| signed 转 unsigned | 通常位模式不变 | 按无符号数解释 |
| unsigned 转 signed | 通常位模式不变 | 按补码解释 |
| 无符号短整数转长整数 | 高位补 0 | 零扩展 |
| 有符号短整数转长整数 | 高位补符号位 | 符号扩展 |
| 整数转 float | 数值尽量保存 | 可能因精度不足而舍入 |

## 工作原理或推导过程

### 1. signed 和 unsigned 为什么会变出“大数”

以 8 位为例，`-1` 的补码是：

```text
11111111
```

如果按 signed 补码解释，它是 `-1`。如果按 unsigned 解释，它就是：

```text
128 + 64 + 32 + 16 + 8 + 4 + 2 + 1 = 255
```

所以不是内存里的内容突然变了，而是解释规则变了。

### 2. 符号扩展为什么要补符号位

有符号数从短位数变成长位数时，要保持数值不变。

例如 8 位补码 `-1`：

```text
11111111
```

扩展成 16 位时，如果高位补 0：

```text
00000000 11111111 = 255
```

数值就错了。正确做法是补符号位 `1`：

```text
11111111 11111111 = -1
```

这叫符号扩展。无符号数没有符号位，所以高位直接补 0。

### 3. float 为什么可能改变整数

`float` 单精度大约有 24 位二进制有效精度。小整数可以精确表示，但很大的整数不一定能逐个区分。

例如超过一定范围后，相邻可表示浮点数之间的间隔可能大于 1。此时整数转 `float` 时可能发生舍入，转回整数也可能不等于原值。

## 典型例子

### 例 1：16 位 `unsigned short` 转 `short`

```text
65535 的 16 位机器码 = 1111111111111111
按 unsigned short 解释 = 65535
按 short 补码解释 = -1
```

所以同一内存内容，类型不同，输出结果不同。

### 例 2：32 位输出格式变化

```c
int x = -1;
unsigned u = 2147483648u;
```

在 32 位补码机器中：

```text
x 的位模式 = FFFFFFFFH
按无符号方式解释 = 4294967295
按有符号补码方式解释 = -1

u 的位模式 = 80000000H
按无符号方式解释 = 2147483648
按有符号补码方式解释 = -2147483648
```

课程题常把 `%u`、`%d` 看成“按无符号或有符号方式解释同一串位”。严格写 C 程序时，格式控制符还要和实参类型匹配；复习本知识点时，先抓住位模式和解释规则的关系。

## 常见考法

- 判断 `short`、`unsigned short`、`int`、`unsigned int` 之间转换后的值。
- 分析符号扩展和零扩展。
- 判断 signed 和 unsigned 混合比较的结果。
- 判断 `float` 是否能精确保存某个整数或小数。
- 根据内存十六进制内容判断输出值。

## 易错点

- 不要只看变量“原来的数值”，要看它转换后的位数和解释规则。
- 有符号扩展补的是符号位，不是永远补 1。
- 无符号扩展永远高位补 0。
- signed 和 unsigned 混合运算时，signed 值可能先转换成 unsigned 再参与比较。
- `float` 范围大不代表精度无限，很多十进制小数和大整数不能精确表示。

## 小结

- C 类型转换题的主线是“位模式”和“解释规则”分开看。
- signed/unsigned 转换常用补码和无符号解释互换。
- 扩展题抓住零扩展和符号扩展。
- 浮点题抓住有效位数和舍入。
