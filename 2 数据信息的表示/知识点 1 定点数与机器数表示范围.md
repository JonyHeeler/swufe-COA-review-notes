# 定点数与机器数表示范围

> 来源依据：第 2 章 PPT“定点整数”“定点小数”“定点数机器码表示范围”。

## 一句话理解

定点数就是事先约定小数点固定在哪里，机器只保存二进制位，不额外保存小数点位置。

## 基本概念

“定点”的意思是小数点位置固定。小数点不一定真的存进机器码里，而是由编码规则约定。

常见两类：

| 类型 | 小数点位置 | 适合表示 |
| --- | --- | --- |
| 定点整数 | 默认在最低位之后 | 整数 |
| 定点小数 | 默认在符号位之后 | 绝对值小于或等于 1 的小数 |

例如 8 位定点整数 `00001010` 表示 `10`。如果同样的位串按定点小数解释，位权就完全不同。

机器数表示范围由三件事决定：

- 总位数是多少。
- 是否带符号。
- 使用无符号、原码、反码还是补码。

## 工作原理或推导过程

### 1. 定点整数范围

n 位无符号整数所有位都用来表示数值，所以范围是：

```text
0 ~ 2^n - 1
```

n 位补码整数有 1 位符号相关位，范围是：

```text
-2^(n-1) ~ 2^(n-1)-1
```

为什么不对称？因为补码只有一个 0，多出来的编码给了最小负数。

### 2. 定点小数范围

定点小数的小数点在符号位之后。以补码定点小数为例，后面 `n-1` 位的权值是：

```text
2^-1, 2^-2, 2^-3, ..., 2^-(n-1)
```

最大正数是这些小数位全为 1：

```text
0.111...1B = 1 - 2^-(n-1)
```

最小负数是：

```text
1.000...0B = -1
```

所以 n 位补码定点小数范围为：

```text
-1 ~ 1 - 2^-(n-1)
```

最小间隔由最低位权值决定，也就是：

```text
2^-(n-1)
```

这就是定点小数的精度。

## 典型例子

### 例 1：16 位补码整数范围

```text
n = 16
范围 = -2^15 ~ 2^15 - 1
     = -32768 ~ 32767
```

### 例 2：16 位无符号整数范围

```text
范围 = 0 ~ 2^16 - 1
     = 0 ~ 65535
```

### 例 3：8 位补码定点小数范围

共有 1 位符号位和 7 位小数位：

```text
范围 = -1 ~ 1 - 2^-7
     = -1 ~ 0.9921875
```

## 常见考法

- 给定位数，求无符号、原码、反码、补码表示范围。
- 判断定点整数和定点小数的小数点位置。
- 给机器码，按定点小数或定点整数求真值。
- 判断某个数是否超出表示范围。

## 易错点

- 定点小数的小数点是约定出来的，不一定真的存了一位“小数点”。
- 定点小数的位权从 `2^-1` 开始，不是从 `2^0` 开始。
- 补码定点小数能表示 `-1`，但不能表示 `+1`。
- 位数增加会同时影响范围和精度，不能只看范围。

## 小结

- 定点数的优点是表示和运算简单，限制是范围和精度固定。
- 定点整数看整数位权，定点小数看小数位权。
- 表示范围题先确认位数、小数点位置和编码方式。
