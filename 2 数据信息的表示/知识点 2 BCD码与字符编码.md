# BCD 码与字符编码

> 来源依据：第 2 章 PPT“BCD”“ASCII”“字符表示”。

## 一句话理解

BCD 和字符编码都在解决“人熟悉的符号怎样用二进制保存”，但 BCD 面向十进制数字，ASCII 面向字符。

## 基本概念

**BCD 码**是 Binary Coded Decimal，即“用二进制编码十进制数字”。最常见的是 8421 BCD，每 4 位表示一个十进制数字 `0~9`。

注意：BCD 不是把整个十进制数转成普通二进制，而是每一位十进制数字单独编码。

**ASCII 码**用二进制编码英文字符、数字字符、标点符号和控制字符。标准 ASCII 是 7 位编码，实际存储时常放在 1 个字节中。

## 工作原理或推导过程

### 1. BCD 为什么存在

普通二进制存储效率更高，但十进制输入输出和金融、商业计算中，经常希望保留十进制位的直观结构。BCD 每 4 位对应一个十进制数字，便于和人看到的十进制形式互相转换。

十进制 `59` 的 8421 BCD：

```text
5 -> 0101
9 -> 1001
59 -> 0101 1001
```

普通二进制的 `59` 是：

```text
111011B
```

所以 `0101 1001` 作为 BCD 表示十进制 `59`，但如果按普通二进制解释，它是 `89`。这就是编码规则不同造成的差异。

### 2. ASCII 字符和数字值不同

字符 `'0'` 不是整数 `0`。字符需要编码表来表示。

常见 ASCII 码：

| 字符 | ASCII 十六进制 |
| --- | --- |
| `'0'` | `30H` |
| `'1'` | `31H` |
| `'A'` | `41H` |
| `'a'` | `61H` |

数字字符连续排列，所以字符 `'5'` 的编码可以理解为：

```text
'0' + 5 = 30H + 5 = 35H
```

## 典型例子

假设内存中有一个字节：

```text
00110001
```

如果按 ASCII 解释，它是字符 `'1'`，因为 `00110001B = 31H`。如果按普通无符号整数解释，它是 `49`。这说明“编码”和“数值”必须分开看。

## 常见考法

- 区分 BCD 和普通二进制表示。
- 判断 8421 BCD 中哪些 4 位组合是非法的。
- 判断数字字符和整数值的编码差异。
- 根据 ASCII 码推算连续数字字符或大小写字母。

## 易错点

- 8421 BCD 中 `1010` 到 `1111` 不表示合法十进制数字。
- BCD 的 `0101 1001` 表示十进制数字串 `59`，不是普通二进制的 `59`。
- 字符 `'9'` 和整数 `9` 的内存编码不同。
- BCD 便于十进制处理，但存储效率低于纯二进制。

## 小结

- BCD 面向十进制数字，每 4 位表示一位十进制数码。
- ASCII 面向字符，数字字符也只是字符编码的一部分。
- 做题时先判断题目问的是“数值”还是“字符/编码”。
