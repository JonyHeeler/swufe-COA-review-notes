# 第一次作业 第三题

## 原题

> float 型数据通常用 IEEE 754 单精度浮点数格式表示。若编译器将 float 型变量 x 分配在一个 32 位浮点寄存器FR1中，且 x=-8.25，则 FR1 的内容是多少？

## 涉及知识点与公式

这道题主要涉及两个文件中的知识。

第一，来自 **`核心知识点 1 数制转换与原码反码补码移码.md`** 的“进制转换”：

整数部分转二进制常用 **除基取余，逆序排列**；小数部分转二进制常用 **乘基取整，顺序排列**。本题中的 `8.25` 要先转成二进制数，整数部分 `8` 转成 `1000B`，小数部分 `0.25` 转成 `0.01B`，所以：

```text
8.25 = 1000.01B
```

第二，来自 **`核心知识点 2 浮点数规格化与IEEE754.md`** 的“IEEE754 单精度浮点数格式”：

IEEE754 单精度浮点数共 `32` 位，由三部分组成：

| 字段              | 位数  | 含义                           |
| --------------- | ---:| ---------------------------- |
| 符号位 `S`         | 1   | `0` 表示正数，`1` 表示负数            |
| 阶码字段 `Exponent` | 8   | 保存实际指数加偏置值后的结果，单精度偏置值为 `127` |
| 尾数字段 `Fraction` | 23  | 保存规格化尾数 `1.xxxxx` 中小数点后面的部分  |

规格化数的实际值为：

```text
(-1)^S × (1.Fraction) × 2^(Exponent - 127)
```

注意，规格化形式要求写成：

```text
1.xxxxx × 2^E
```

其中小数点前面的这个 `1` 是隐藏位，不存入 `Fraction` 字段。也就是说，尾数字段只保存小数点后面的 `xxxxx`。

## 解题思路与答案

```text
8.25 = 1000.01B
```

因为原数是负数，所以数学真值是：

```text
-8.25 = -1000.01B
```

但是 IEEE754 不是直接把 `-1000.01B` 存进去，而是要把它拆成符号位、阶码字段和尾数字段。

接着进行规格化。`1000.01B` 的小数点原来在 `1000` 后面，要变成 `1.xxxxx` 的形式，需要把小数点向左移动 `3` 位：

```text
1000.01B = 1.00001B × 2^3
```

因此实际指数为：

```text
E = 3
```

单精度浮点数的阶码字段不是直接保存 `3`，而是保存：

```text
Exponent = E + 127 = 3 + 127 = 130
```

把 `130` 转成 8 位二进制：

```text
130 = 128 + 2 = 10000010B
```

所以阶码字段为：

```text
Exponent = 10000010
```

再看尾数字段。规格化尾数是：

```text
1.00001B
```

小数点前面的 `1` 是隐藏位，不存入尾数字段。因此只保存小数点后面的：

```text
00001
```

尾数字段一共要写满 `23` 位，所以后面补 `0`：

```text
Fraction = 00001000000000000000000
```

现在把三个字段合起来：

```text
S        Exponent   Fraction
1        10000010   00001000000000000000000
```

所以 FR1 中的 32 位内容为：

```text
11000001000001000000000000000000
```

为了更方便书写，可以按 4 位一组转成十六进制：

```text
1100 0001 0000 0100 0000 0000 0000 0000
 C    1    0    4    0    0    0    0
```

## 为什么不能按补码来做

这道题最容易错的地方，是把 `-8.25` 当成普通带符号整数去求补码。

补码主要用于表示整数，例如 8 位补码、16 位补码、32 位补码。可是题目明确说的是 `float` 型数据，并且通常用 IEEE754 单精度浮点数格式表示，所以 FR1 中保存的不是 `-8.25` 的补码，而是 IEEE754 浮点数位模式。

还有一个容易混淆的点是：题目问的是 **32 位浮点寄存器 FR1 的内容**，不是问内存中的字节存放顺序。因此答案通常写成完整的 32 位位串或十六进制字：

```text
C1040000H
```

## 拓展设问与变种考法

| 变种考法                                                 | 解题过程                                                                                                                                     | 答案                             |
| ---------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------ |
| **1. 只问 `+8.25` 的 IEEE754 单精度表示**                    | 二进制和规格化过程不变：`8.25 = 1000.01B = 1.00001B × 2^3`。区别是正数符号位 `S = 0`。阶码仍为 `10000010`，尾数仍为 `00001000000000000000000`。                          | `41040000H`                    |
| **2. 问 `-8.25` 的符号位**                                | IEEE754 中符号位单独表示正负，负数为 `1`，正数为 `0`。                                                                                                      | `S = 1`                        |
| **3. 问 `-8.25` 的实际指数 E**                             | `8.25 = 1000.01B`，规格化为 `1.00001B × 2^3`，所以实际指数为 `3`。                                                                                     | `E = 3`                        |
| **4. 问 `-8.25` 的阶码字段**                               | 单精度偏置值为 `127`，所以阶码字段等于 `3 + 127 = 130`，转成 8 位二进制为 `10000010`。                                                                            | `10000010B`                    |
| **5. 问 `-8.25` 的尾数字段**                               | 规格化尾数为 `1.00001B`，隐藏最高位 `1`，只保存小数点后的 `00001`，再补足 23 位。                                                                                   | `00001000000000000000000`      |
| **6. 给出二进制串 `11000001000001000000000000000000`，问真值** | 拆字段：`S=1`，`Exponent=10000010B=130`，实际指数 `E=130-127=3`，`Fraction=000010...`，尾数为 `1.00001B`，所以真值为 `-(1.00001B × 2^3) = -1000.01B = -8.25`。 | `-8.25`                        |
| **7. 给出十六进制 `C1040000H`，问真值**                        | 先转成二进制：`C1040000H = 11000001000001000000000000000000B`，再按 IEEE754 拆符号位、阶码和尾数。                                                            | `-8.25`                        |
| **8. 问为什么尾数不是 `100001...`**                          | 因为 IEEE754 规格化数有隐藏位。`1.00001B` 的小数点前 `1` 不存，只存小数点后的 `00001`。                                                                             | 尾数应为 `00001000000000000000000` |
| **9. 问为什么阶码不是 `00000011`**                           | `00000011` 是实际指数 `3` 的普通二进制，但 IEEE754 单精度阶码字段采用偏置表示，要加上 `127`。                                                                           | 阶码字段是 `10000010`               |
| **10. 改成问 `-0.25` 的 IEEE754 表示**                     | `0.25 = 0.01B = 1.0B × 2^-2`，所以 `S=1`，`Exponent=-2+127=125=01111101B`，`Fraction` 全 0。                                                    | `BE800000H`                    |
| **11. 改成问 `-1.0` 的 IEEE754 表示**                      | `1.0 = 1.0B × 2^0`，所以 `S=1`，`Exponent=0+127=127=01111111B`，`Fraction` 全 0。                                                               | `BF800000H`                    |
| **12. 问该数是否需要舍入**                                    | `8.25 = 1000.01B`，规格化后小数部分只有有限位 `00001`，23 位尾数字段完全能装下，后面补 0 即可。                                                                          | 不需要舍入，能精确表示                    |

## 同类题通用模板

遇到 IEEE754 单精度浮点数表示题，可以按下面步骤做。

第一步，判断符号位：

```text
正数：S = 0
负数：S = 1
```

第二步，把绝对值转成二进制。整数部分用“除基取余，逆序排列”，小数部分用“乘基取整，顺序排列”。如果是常见小数，也可以直接用分数判断，例如：

```text
0.5 = 2^(-1) = 0.1B
0.25 = 2^(-2) = 0.01B
0.125 = 2^(-3) = 0.001B
```

第三步，规格化成：

```text
1.xxxxx × 2^E
```

第四步，求阶码字段：

```text
Exponent = E + 127
```

并把结果写成 8 位二进制。

第五步，写尾数字段。把规格化尾数 `1.xxxxx` 中小数点前的 `1` 去掉，只保留小数点后的部分，再补足 23 位：

```text
Fraction = xxxxx 后面补 0 到 23 位
```

第六步，合成 32 位：

```text
S | Exponent | Fraction
```

最后可以按 4 位一组转换成十六进制。写答案时通常写：

```text
32 位二进制位串
十六进制表示
```

本题代入模板就是：

```text
-8.25
→ S = 1
→ 8.25 = 1000.01B
→ 1000.01B = 1.00001B × 2^3
→ Exponent = 3 + 127 = 130 = 10000010B
→ Fraction = 00001000000000000000000
→ FR1 = 1 10000010 00001000000000000000000
→ FR1 = 11000001000001000000000000000000B = C1040000H
```
