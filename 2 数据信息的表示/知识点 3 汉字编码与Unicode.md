# 汉字编码与 Unicode

> 来源依据：第 2 章 PPT“汉字表示法”“汉字机内码”“字形码”“Unicode”。

## 一句话理解

汉字编码解决的是三个环节：怎么输入汉字、怎么在计算机内部保存汉字、怎么把汉字显示出来。

## 基本概念

处理汉字时，经常会遇到三类编码：

| 编码 | 解决的问题 | 例子或说明 |
| --- | --- | --- |
| 输入码 | 人怎样把汉字输入进去 | 拼音码、五笔码 |
| 机内码 | 计算机内部怎样保存和处理汉字 | 文本处理中真正使用的字符编码 |
| 字形码 | 屏幕或打印机怎样画出汉字 | 点阵字形、轮廓字体 |

这三者对应的是不同阶段。输入码不是机内码，机内码也不是字形本身。

**Unicode** 的目标是给世界上大量字符分配统一码位。它更像“字符编号体系”。而 **UTF-8、UTF-16** 是把这些码位实际存成字节序列的具体编码方式。

## 工作原理或推导过程

一个汉字从输入到显示，大致经历：

```text
输入码 → 机内码 → 字形码 → 屏幕/打印输出
```

例如你用拼音输入一个汉字：

1. 输入法先根据拼音找到候选汉字。
2. 选中汉字后，程序保存的是这个字符对应的内部编码。
3. 显示时，系统根据字符编码去字体文件中找到对应字形。
4. 字体把这个汉字画到屏幕上。

为什么要区分这些编码？因为同一个汉字可以用不同输入法输入，也可以用不同字体显示，但它作为文本中的“同一个字符”应当保持一致。

## 典型例子

同一个汉字“中”，可以用拼音 `zhong` 输入，也可以用五笔输入。输入方法不同，但进入文本后，它应该对应同一个字符。显示时，如果换成宋体、黑体或楷体，字形风格会变化，但文本里的字符没有变。

这说明：

```text
输入方式 ≠ 内部字符编码 ≠ 显示字形
```

## 常见考法

- 区分输入码、机内码和字形码。
- 说明汉字从输入到显示的大致过程。
- 判断 Unicode 与 UTF-8、UTF-16 的关系。
- 理解为什么多语言文本需要统一字符集。

## 易错点

- Unicode 是字符集/码位体系，UTF-8 是一种具体编码方式。
- 输入法编码不等于内存中的字符编码。
- 字形码负责“长什么样”，机内码负责“是哪一个字符”。
- 字体变化不等于字符编码变化。

## 小结

- 汉字处理至少涉及输入、存储处理、显示输出三个环节。
- Unicode 的价值是让多语言字符有统一编号。
- 考试重点通常是概念区分和流程理解，不是背具体码表。
