# 第 2 章 数据信息的表示复习导读

## 本章定位

第 2 章回答一个很基础但很关键的问题：计算机里只有 `0` 和 `1`，它怎样表示整数、小数、字符，以及怎样发现数据出错？

本章是计算题高发章节。后面学加减乘除、存储器、C 语言机器级表示时，也会反复用到本章知识，尤其是补码、浮点数和校验码。

## 核心优先级

| 优先级 | 内容 | 复习要求 |
| --- | --- | --- |
| 核心 | 进制转换、原码、反码、补码、移码 | 会在“真值”和“机器码”之间互相转换，能解释表示范围 |
| 核心 | 浮点数规格化与 IEEE754 | 会拆分符号位、阶码、尾数，能做十进制和单精度格式转换 |
| 核心 | 海明校验与 CRC | 会计算校验位数、理解指错字和模 2 除法 |
| 核心 | C 语言数据表示转换 | 会分析 signed/unsigned、符号扩展、零扩展和 float 精度 |
| PPT 补充 | 定点数、BCD、字符编码、汉字编码、Unicode | 理解不同编码分别解决什么问题 |

## 学习顺序

1. 先学进制转换。它是所有机器码题的基本工具。
2. 再学原码、反码、补码、移码。重点不是背表，而是理解“同一串二进制按不同规则解释会得到不同数值”。
3. 然后学定点数和浮点数。定点数的小数点固定，浮点数的小数点可以靠阶码移动。
4. 再学校验码。海明码偏向定位并纠正 1 位错误，CRC 偏向高效检错。
5. 最后结合 C 语言题，把二进制位模式和程序输出联系起来。

## 本章最重要的思维方式

不要一看到二进制就急着算。先问三个问题：

- 这串二进制有多少位？
- 它按什么编码规则解释？
- 题目问的是机器码、真值、字符，还是校验位？

例如同样是 `11111111`：

| 解释方式 | 结果 |
| --- | --- |
| 8 位无符号整数 | 255 |
| 8 位补码整数 | -1 |
| 8 位原码整数 | -127 |
| 8 位反码整数 | -0 |

所以本章的核心不是“二进制长什么样”，而是“这串二进制按哪套规则解释”。

## 易错提醒

- 补码的负数范围比正数多一个最小负数，因为补码只有一个 0，空出的编码给了 `-2^(n-1)`。
- IEEE754 单精度阶码采用偏置表示，规格化数的实际阶码是 `阶码字段 - 127`。
- CRC 做模 2 除法时没有借位，减法就是异或。
- C 语言中同一串位按 signed 和 unsigned 解释，数值可能完全不同。
