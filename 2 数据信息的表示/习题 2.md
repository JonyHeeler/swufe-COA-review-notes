# 习题 2 答案

## 2.1 名词解释

#### 第一组：定点数与基本编码

这一组是计算机底层表示最基础的数值符号概念。

- **真值：** 现实中带有正负号的实际数值（例如 +5、-18）。它是我们人类习惯的数学表示。

- **机器码：** 真值在计算机内部的二进制表示形式，连同符号位一起被数字化（如用 0 表示正，1 表示负）。

- **原码：** 一种最直观的机器码表示法。最高位为符号位（0 正 1 负），其余有效数据位表示数值的绝对值。

- **反码：** 正数的反码与其原码相同；负数的反码是对其原码除符号位外的各位进行“按位取反”（0 变 1，1 变 0）。

- **补码：** 计算机中最常用的带符号整数表示法。正数的补码与原码相同；负数的补码是其反码在最低位加 1。它巧妙地将减法化为加法运算。在做题中常用扫描法快速求。

- **移码：** 通常用于表示浮点数的阶码。它是将一个真值加上一个固定的偏移量（偏置常数）得到的，这样能把所有的数转换为非负整数，非常方便计算机比对大小。

- **模：** 计量系统中的计数容量或循环周期（就像时钟的模是 12）。在计算机编码中，通常指某个字长的数据所能表示的状态总数。

- **定点数：** 小数点位置固定不变的数值表示法。通常分为定点整数（小数点固定在最右侧）和定点小数（小数点固定在符号位之后）。定点小数全是纯小数，整数部分默认为 0，所以它能表示的数值范围通常在 -1 到 1 之间（不包含 1）。

### 第二组：浮点数相关

浮点数是用来表示范围极广的实数的机制，类似科学记数法。

- **浮点数：** 小数点位置可以根据需要浮动的数值表示法。它由阶码（表示范围大小）和尾数（表示有效数字和正负）两部分组成。

- **溢出：** 当运算结果超出了计算机所分配的数据格式所能表示的最大范围时，就会发生溢出（分为向上溢出和向下溢出）。

- **精度：** 数据表示中有效数字的位数，决定了数值能被精确到什么程度。在浮点数中，精度主要由尾数的位数来决定。

- **浮点数规格化：** 为了充分利用尾数的有效位数并保证数据表示的唯一性，规定浮点数尾数的最高有效位必须为一个非零值（二进制中即为 1）的处理过程。

- **隐藏位：** 在 IEEE 754 标准中，规格化二进制浮点数的尾数最高位总是 1。为了节省一位存储空间以提高精度，这个 1 在存储时不予保存，计算时再默认加上，这就是隐藏位。

- **二进制浮点数 / 十进制浮点数：** 分别是以 2 为基数和以 10 为基数的浮点数表示系统。现代处理器为了避免二进制无法精确表示某些十进制小数（如 0.1），开始增加对十进制浮点数硬件支持。

### 第三组：十进制与字符编码

这部分基本上不考。

### 第四组：可靠性与检错纠错码

在数据传输和存储中，保证数据正确的防线。

- **码距：** 在一种编码系统中，任意两个合法字码之间，对应位置上二进制位不同的最小数量（汉明距离）。码距越大，检错和纠错的能力就越强。

- **校验码：** 在有效数据中人为加入一定的冗余位，使整体数据遵循某种特定的逻辑规则。当传输发生错误破坏了规则时，接收方就能以此发现甚至纠正错误。

- **多重奇偶校验：** 在简单的单维水平奇偶校验基础上，增加垂直奇偶校验等，形成多维度的交叉校验，不仅能提高检错率，还能实现一定的错误定位和纠错功能。

- **ECC 码 (Error Correction Code)：** 纠错码的统称，泛指那些不仅能发现错误，还能自动定位并纠正部分错误的编码技术（常见于服务器内存条）。

- **海明码 (Hamming Code)：** 一种著名的具有单比特纠错能力（或者双比特检错能力）的线性分组校验码，它通过巧妙地插入多个奇偶校验位来实现定位。

- **CRC 码 (Cyclic Redundancy Check)：** 循环冗余校验码。它利用特定的生成多项式进行模 2 除法产生校验序列。CRC 检错能力强，开销小，在数据通信（如以太网）、磁盘存储、甚至 GIF/TIFF 图像格式中无处不在 。

**第六题详细解析：**

1. 将十六进制数 `C640 0000H` 展开为二进制：`1100 0110 0100 0000 0000 0000 0000 0000`。

2. 按照 IEEE 754 单精度浮点数格式（1 位符号位 S + 8 位阶码 E + 23 位尾数 M）划分：
   
   - 符号位 $S = 1$，表示该数为负数。
   
   - 阶码 $E = 1000 1100_{(2)} = 128 + 8 + 4 = 140_{(10)}$。
   
   - 实际指数 $e = E - 127 = 140 - 127 = 13$。
   
   - 尾数部分 $M = 100 0000 \dots 0_{(2)}$，由于存在隐藏的最高有效位 `1`，所以真实的尾数值为 $1.1_{(2)} = 1 + 2^{-1} = 1.5_{(10)}$。

3. 综上，该数的值为 $-1.5 \times 2^{13}$。

**第七题详细解析：**

1. 要求最大正数，符号位 $S$ 必然为 `0`。

2. IEEE 754 规定阶码 $E = 255$ 用于表示特殊值（无穷大或 NaN），因此规格化数能取到的最大阶码为 $E = 254$。对应的实际指数 $e = 254 - 127 = 127$。

3. 尾数 $M$ 的 23 位全为 `1` 时取值最大，即 $1.111\dots1_{(2)}$。这个数可以看作是 $2 - 2^{-23}$。

4. 所以最大正数值为 $(2 - 2^{-23}) \times 2^{127} = 2^{128} - 2^{104}$。

**第八题详细解析：**

1. 要求“最小正数”，符号位 $S = 0$。

2. 题目强调了是“规格化”数。对于规格化数，阶码 $E$ 的取值范围是 1 到 254（$E=0$ 用于表示非规格化数和 0）。

3. 因此最小的规格化阶码 $E = 1$，对应的实际指数 $e = 1 - 127 = -126$。

4. 此时尾数 $M$ 的 23 位全取 `0` 时最小，加上隐藏位 1，真实的尾数值为 $1.0$。

5. 最终该数值为 $1.0 \times 2^{-126}$。（注：如果题目问的是“最小非规格化正数”，答案则是 $2^{-149}$）。

**第九题详细解析：**

1. 先看符号位：`CC90 0000H` 的最高位是 `C`（二进制 `1100`），符号位为 1；`B0C0 0000H` 的最高位是 `B`（二进制 `1011`），符号位也是 1。两者均为负数，所以**符号相同**。

2. 由于 IEEE 754 浮点数的设计使得它们可以直接像带符号的整数（原码形式）那样进行比对大小，我们去掉符号位直接比较绝对值部分（或者直接比较十六进制的后面部分）：`C900000H` 显然大于 `0C00000H`（因为前者的阶码更大）。

3. 这说明 x 的绝对值比 y 的绝对值大。在两个数都是负数的情况下，绝对值大的数反而小。因此 $x < y$。

**第十题详细解析：**

- **I 是真的：** $i = 785$ 转换成二进制只需要 10 位（因为 $2^9 < 785 < 2^{10}$）。float 型有 24 位有效数字（23 位尾数 + 1 位隐藏位），完全可以无损地保存下 785。因此强转为 float 再转回 int，值依然是 785。

- **II 是假的：** $f = 1.5678 \times 10^3 = 1567.8$。将它强制转换为 int 时，会发生截断，小数部分被丢弃，变成 $1567$。再转回 float 则变成 $1567.0$，这显然不等于原来的 $1567.8$。

- **III 是真的：** 把单精度的 float 转换为范围和精度都更大的 double，是一个无损扩展的过程。再将其强转回 float 时，数值原封不动，所以两者相等。

- **IV 是假的：** 这是一个典型的“大数吃小数”问题（精度丢失）。$d = 1.5 \times 10^{100}$ 极其庞大，而 $f = 1567.8$ 相对极小。double 类型的有效位（53 位）大约能保证 15~17 位的十进制精度。当执行 $(d + f)$ 时，$f$ 的值会因为超出了 $d$ 的最低有效位精度而完全被舍入掉，导致 $d + f = d$。随后再减去 $d$，结果是 $0$，$0$ 不等于 $f$。

**补充两点非常实用的小提示：**

1. **整型的循环特性：** 当无符号数或有符号数溢出时，它们在底层就像一个“表盘”一样会发生回绕。比如 `unsigned short` 达到了 65535 再加 1，就会变回 0。

2. **浮点数不能直接用 `==` 判断相等：** 因为存在精度限制，浮点数在底层通常是近似存储。在 C 语言中比较两个浮点数是否“相等”，通常是看它们的差值绝对值是否小于一个极小的阈值（例如 `1e-6`）。

## 2.3 简答题答案要点

1. 采用二进制是因为规则简单、物理器件容易表示两个稳定状态、逻辑电路实现方便。
2. 交叉奇偶校验提高能力的代价是增加校验位、编码解码更复杂、传输或存储开销增加。
3. 补码把减法转化为加法，0 的表示唯一，硬件实现简单，表示范围利用率高。
4. 浮点数范围主要由阶码位数决定，精度主要由尾数位数决定。
5. 输入码负责输入，机内码负责内部存储处理，字形码负责显示和打印。
6. 常通过编码范围或最高位组合区分 ASCII 和汉字机内码。
7. 十进制浮点数可减少商业和金融计算中的十进制舍入误差。
8. 符号位识别正负；范围看阶码，精度看尾数。
9. 两个 0 会增加比较和运算处理复杂度。
10. CRC 通过生成多项式模 2 除法检错，通常不能直接纠错。

## 2.5 补码求真值

- $[x]_补 = 0.10010$：符号位为0，正数。真值为 **+0.10010**

- $[x]_补 = 1.10010$：符号位为1，负数。反码为 1.10001，原码为 1.01110。真值为 **-0.01110**

- $[x]_补 = 1.11111$：符号位为1，负数。反码为 1.11110，原码为 1.00001。真值为 **-0.00001**

- $[x]_补 = 1.00000$：符号位为1，这是补码能表示的下界特殊值（小数部分全0）。真值为 **-1** （或写为 -1.00000）

- $[x]_补 = 0.10001$：符号位为0，正数。真值为 **+0.10001**

- $[x]_补 = 1.00001$：符号位为1，负数。反码为 1.00000，原码为 1.11111。真值为 **-0.11111**

## 2.6 C 语言输出

典型输出为：

```text
x=4294967295=-1
u=2147483648=-2147483648
```

在 32 位计算机中，整数是以 **32 位补码** 形式存放在内存中的。变量的类型（有符号/无符号）以及打印时的格式控制符（`%u` / `%d`），并**不会改变内存中的二进制机器码**，只决定了计算机如何**解释**这串机器码。

- **对于变量 `x`：**
  
  - `-1` 的 32 位补码在内存中是全 1，即十六进制的 `0xFFFFFFFF`。
  
  - 当用 `%u`（无符号十进制）打印时，`0xFFFFFFFF` 被当作一个纯二进制正数，其值为 $2^{32}-1$，即 **4294967295**。
  
  - 当用 `%d`（有符号十进制）打印时，它被解释为有符号的补码，即 **-1**。

- **对于变量 `u`：**
  
  - 无符号数 `2147483648` 正好是 $2^{31}$。存入 32 位内存时，最高位（第31位）为 1，其余 31 位全为 0，即十六进制的 `0x80000000`。
  
  - 当用 `%u` 打印时，最高位的 1 作为普通的数值位参与计算，正常输出 **2147483648**。
  
  - 当用 `%d` 打印时，最高位的 1 被强行解释为 **符号位**（表示负数）。在 32 位有符号补码中，`0x80000000` 规定代表能表示的最小负数，即 $-2^{31}$，因此输出 **-2147483648**。

## 2.7 表示范围

- 16 位无符号数：最小值为全 0（0），最大值为全 1（即 $2^{16}-1 = 65535$）。
- 16 位原码定点小数：原码是对称的。最大正数为 0.111...1（即 $1-2^{-15}$），最小负数为 1.111...1（即 $-(1-2^{-15})$）。，有两个 0。
- 16 位补码定点小数：补码的最大正数与原码相同，为 $+(1-2^{-15})$。但补码的负数比原码多表示一个最低点：当机器码为 1.000...0 时，它代表 -1。
- 16 位补码定点整数：最大正数为 0111...1（即 $2^{15}-1$），最小负数为 1000...0（即 $-2^{15}$）。即 -32768 ~ 32767。

## 2.8 补码的模

题目中的机器码长度为 8 位（从 $x_0$ 到 $x_7$）。在计算机中，“模”可以理解为该系统能表示的状态总数，或者是产生溢出的那个“进位”。对于 $n$ 位的二进制补码系统，其模为 $2^n$。这里 $n=8$，因此模为 $2^8 = 256$。

## 2.9 IEEE754 表示题方法

- **(1) $-6\frac{5}{8}$**
  
  - **转换二进制：** $-6\frac{5}{8} = -6.625$。整数部分 6 为 110，小数部分 0.625 为 101。合并后为 $-110.101_{(2)}$。
  
  - **规格化：** $-1.10101 \times 2^2$。
  
  - **求各字段：** 符号位 $S = 1$；实际指数 $e = 2$，阶码 $E = 127 + 2 = 129 = 10000001_{(2)}$；尾数 $M = 10101$ 后面补 0 到 23 位。
  
  - **组合与十六进制：** `1_10000001_10101000000000000000000` -> `1100 0000 1101 0100 0000 0000 0000 0000`。
  
  - **答案：** **C0D40000H**

- **(2) 3.1415927**
  
  - **转换二进制：** 整数部分 3 为 11。小数部分 0.1415927 不断乘 2 取整，得到 00100100001111110110101...
  
  - **规格化：** $1.100100100001111110110101 \times 2^1$。
  
  - **舍入：** 截取 23 位尾数，第 24 位为 1，进行“0舍1入”向上进位，尾数 $M$ 变为 `10010010000111111011011`。
  
  - **求各字段：** 符号位 $S = 0$；阶码 $E = 127 + 1 = 128 = 10000000_{(2)}$。
  
  - **组合与十六进制：** `0_10000000_10010010000111111011011` -> `0100 0000 0100 1001 0000 1111 1101 1011`。
  
  - **答案：** **40490FDBH**

- **(3) 64000**
  
  - **转换二进制：** $64000 = 32768 + 16384 + 8192 + 4096 + 2048 + 512$。转换为二进制是 $1111101000000000_{(2)}$。
  
  - **规格化：** $1.111101000000000 \times 2^{15}$。
  
  - **求各字段：** 符号位 $S = 0$；阶码 $E = 127 + 15 = 142 = 10001110_{(2)}$；尾数 $M = 111101$ 后面补 0。
  
  - **组合与十六进制：** `0_10001110_11110100000000000000000` -> `0100 0111 0111 1010 0000 0000 0000 0000`。
  
  - **答案：** **477A0000H**

## 2.10 单精度 `43940000H`

- **1. 展开为二进制：** `0100 0011 1001 0100 0000 0000 0000 0000`

- **2. 划分字段：**
  
  - 符号位 $S = 0$（正数）。
  
  - 阶码 $E = 10000111_{(2)} = 128 + 4 + 2 + 1 = 135_{(10)}$。
  
  - 尾数 $M = 001 0100 \dots 0$。

- **3. 计算真值：**
  
  - 实际指数 $e = 135 - 127 = 8$。
  
  - 加上隐藏位 1 后，真实的二进制数值为 $+1.00101 \times 2^8$。
  
  - 将小数点向右移 8 位：$100101000_{(2)}$。

- **4. 转十进制：** $100101000_{(2)} = 2^8 + 2^5 + 2^3 = 256 + 32 + 8 = 296$。

- **答案：** **296**

## 2.11 单精度范围

- **最大正数：** 符号位 0，阶码取规格化最大值 254，尾数全 1。
  
  - 值为 $+(2-2^{-23}) \times 2^{127}$，大约是 **$3.4 \times 10^{38}$**。

- **最小的负数：**

- 符号位 1，阶码取规格化最大值 254，尾数全 1。
  
  - 值为 $-(2-2^{-23}) \times 2^{127}$，大约是 **$-3.4 \times 10^{38}$**。

- **（补充）绝对值最小的非零正数：**
  
  - 如果是指最接近 0 的数字，在支持“非规格化数”的情况下，阶码为 0，尾数最后一位为 1。
  
  - 值为 $1.0 \times 2^{-149}$，大约是 **$1.4 \times 10^{-45}$**。

## 2.12 浮点数比较

**(1) 若 $m<n$，是否有 $N_1>N_2$？**

- **答案：不一定。**

- **解析：** 题目并没有说明尾数 $M_1$ 和 $M_2$ 已经经过规格化处理。由于尾数的值不受限制，$M_1$ 可能比 $M_2$ 大得多。例如，若 $m=1, M_1=10$（即 $N_1=20$），而 $n=2, M_2=0.1$（即 $N_2=0.4$），此时虽然 $m<n$，但却有 $N_1>N_2$。

**(2) 若 $M_1$ 和 $M_2$ 是规格化的数，上述结论是否正确？**

- **答案：不正确。如果是规格化正数，当 $m<n$ 时，必然有 $N_1<N_2$。**

## 2.13 创新浮点数的取值范围

**分析与前置条件：**

- 阶码 3 位（包含 1 位符号位），其补码表示范围是 -4 到 +3（即 `100` 到 `011`）。

- 尾数 7 位（包含 1 位符号位），规格化正数的形式为 `0.1xxxxx`，规格化负数的形式为 `1.0xxxxx`。

- **最大正数**
  
  - **机器码：** 阶码取最大正数 `011` (+3)；尾数取最大正数 `0.111111`。
  
  - **真值计算：** $+0.111111_{(2)} \times 2^3$
  
  - **十进制数：** $(1 - 2^{-6}) \times 8 = \frac{63}{64} \times 8 =$ **7.875**

- **最小正数（正数中最接近 0 的规格化数）**
  
  - **机器码：** 阶码取最小负数 `100` (-4)；尾数取最小正规格化数 `0.100000`。
  
  - **真值计算：** $+0.100000_{(2)} \times 2^{-4}$
  
  - **十进制数：** $0.5 \times 2^{-4} = 2^{-5} =$ **0.03125**

- **最大负数（负数中最接近 0 的规格化数）**
  
  - **机器码：** 阶码取最小负数 `100` (-4)；尾数取绝对值最小的负规格化补码 `1.011111`（其反码是 `1.100000`，原码是 `1.100001`，即真值尾数为 $-0.100001_{(2)}$）。
  
  - **真值计算：** $-0.100001_{(2)} \times 2^{-4}$
  
  - **十进制数：** $-(0.5 + 2^{-6}) \times 2^{-4} = -\frac{33}{64} \times 2^{-4} = -\frac{33}{1024} \approx$ **-0.0322265625**

- **最小负数（绝对值最大的负数）**
  
  - **机器码：** 阶码取最大正数 `011` (+3)；尾数取绝对值最大的负数补码 `1.000000`（原码也是 `1.000000`，代表真值尾数 -1）。
  
  - **真值计算：** $-1 \times 2^3$
  
  - **十进制数：** **-8**

## 2.14 创新浮点数的表示

**(1) 57 / 128**

- **步骤 1（转二进制）：** $\frac{57}{128} = 57 \times 2^{-7}$。由于 $57 = 32 + 16 + 8 + 1 = 111001_{(2)}$，所以小数真值为 $+0.0111001_{(2)}$。

- **步骤 2（规格化）：** 将小数点右移一位，使其变为 $+0.111001_{(2)} \times 2^{-1}$。

- **步骤 3（求机器码）：**
  
  - 阶码真值为 -1，4 位补码表示为：`1111`。
  
  - 尾数真值为 $+0.111001_{(2)}$，正数的补码和原码相同，凑齐 10 位（含符号位）并在后面补 0：`0.111001000`。

- **最终答案：** 阶码为 **1111**，尾数为 **0.111001000**。

**(2) -69 / 128**

- **步骤 1（转二进制）：** $\frac{-69}{128} = -69 \times 2^{-7}$。由于 $69 = 64 + 4 + 1 = 1000101_{(2)}$，所以小数真值为 $-0.1000101_{(2)}$。

- **步骤 2（规格化）：** 该数值本身已经是标准的二进制规格化小数形式，所以不需要移位，即 $-0.1000101_{(2)} \times 2^0$。

- **步骤 3（求机器码）：**
  
  - 阶码真值为 0，4 位补码表示为：`0000`。
  
  - 尾数真值为 $-0.1000101_{(2)}$，其原码为 `1.100010100`。求其补码（符号位不变，数值位取反后末位加1）：反码为 `1.011101011`，加 1 后补码为 `1.011101100`。（注意：补码规格化负数的特征是符号位与最高数值位必定相反，这里是 `1.0`，符合特征）。

- **最终答案：** 阶码为 **0000**，尾数为 **1.011101100**。

## 2.15 奇偶校验

- **奇校验码与偶校验码：**
  
  有效信息 $01011011$ 中共含有 5 个 `1`。
  
  - **奇校验码：** 要求整体（有效信息 + 校验位）含有奇数个 `1`。因为有效信息已经是奇数个 `1`，所以附加的校验位应为 `0`。假设将校验位附加在最低位，则奇校验码为 **$010110110$**。
  
  - **偶校验码：** 要求整体含有偶数个 `1`。为了凑成偶数个 `1`，附加的校验位应为 `1`。则偶校验码为 **$010110111$**。

- **如何发现错误：**
  
  接收方收到的有效信息变为 $01011010$（最低位从 1 变成了 0），此时数据中含有 4 个 `1`。
  
  - **若采用奇校验：** 接收方会将收到的数据与校验位 `0` 放在一起检查，总共有 4 个 `1`（偶数个）。这与奇校验“必/须有奇数个 `1`”的规则冲突，从而**发现错误**。
  
  - /。/**若采用偶校验：** 接收方会将收到的数据与校验位 `1` 放在一起检查，总共有 5 个 `1`（奇数个）。这与偶校验“必须有偶数个 `1`”的规则冲突，从而**发现错误**。

## 2.17 海明校验

- **确定校验位数量：** 数据位 $k = 8$，根据公式 $2^r \ge k + r + 1$，即 $2^r \ge 9 + r$，满足条件的最小整数 $r = 4$。因此需要 4 个校验位，海明码总长 12 位。

- **计算校验位（采用偶校验分组）：**
  
  - $P_1$ 负责所有二进制下标第 1 位为 1 的位置：$P_1 = H_3 \oplus H_5 \oplus H_7 \oplus H_9 \oplus H_{11} = 0 \oplus 1 \oplus 0 \oplus 1 \oplus 1 = 1$
  
  - $P_2$ 负责所有二进制下标第 2 位为 1 的位置：$P_2 = H_3 \oplus H_6 \oplus H_7 \oplus H_{10} \oplus H_{11} = 0 \oplus 1 \oplus 0 \oplus 1 \oplus 1 = 1$
  
  - $P_3$ 负责所有二进制下标第 3 位为 1 的位置：$P_3 = H_5 \oplus H_6 \oplus H_7 \oplus H_{12} = 1 \oplus 1 \oplus 0 \oplus 0 = 0$
  
  - $P_4$ 负责所有二进制下标第 4 位为 1 的位置：$P_4 = H_9 \oplus H_{10} \oplus H_{11} \oplus H_{12} = 1 \oplus 1 \oplus 1 \oplus 0 = 1$

- **生成的海明码：** **$110011011110$**

如果接收方收到的有效信息变为 $01101111$（即 $D_8$ 从 0 变成了 1），对应的接收海明码变为 **$110011011111$**（第 12 位出错了）。

- 重新计算指错字：
  
  - $S_1 = 1 \oplus 0 \oplus 1 \oplus 0 \oplus 1 \oplus 1 = 0$
  
  - $S_2 = 1 \oplus 0 \oplus 1 \oplus 0 \oplus 1 \oplus 1 = 0$
  
  - $S_3 = 0 \oplus 1 \oplus 1 \oplus 0 \oplus 1 = 1$
  
  - $S_4 = 1 \oplus 1 \oplus 1 \oplus 1 \oplus 1 = 1$

- **定位错误：** 指错字 $S_4 S_3 S_2 S_1 = 1100_{(2)} = 12_{(10)}$，这准确地指出了第 12 位（$H_{12}$）发生了错误。

- **纠正错误：** 将第 12 位的数据取反（从 1 恢复为 0），即可得到正确的有效信息 $01101110$。

## 2.18  CRC

先在后面补 3 个 0：1001→1001000，然后用模 2 除法（相同为 0，相异为 1）余数为：R=011。所以 CRC 校验码为 1001 011。

收到的完整码字为：1101 011，用 1101 去模 2 除，得到余数 011​。余数不为 0，说明发生错误。

在 CRC 码中，每一种非零余数（伴随式）都唯一对应着某一位的单比特错误。**第 n 位出错时，接收码字的余数 = “第 n 位为1，其余为0”的错误图样除以 G 的余数​。** 由此可见，我们可以建立一张“出错位置对照表”（假设最右侧为第 0 位）：

- 第 0 位错 ($0000001$)：余数 $001$

- 第 1 位错 ($0000010$)：余数 $010$

- 第 2 位错 ($0000100$)：余数 $100$

- 第 3 位错 ($0001000$)：$1000 \div 1101 \rightarrow$ 余数 $101$

- 第 4 位错 ($0010000$)：$1010 \div 1101 \rightarrow$ 余数 $111$

- **第 5 位错 ($0100000$)：$1110 \div 1101 \rightarrow$ 余数 $011$**

- 第 6 位错 ($1000000$)：$0110 \div 1101 \rightarrow$ 余数 $110$

通过查表可知，当前计算出的余数 **$011$** 恰好对应第 5 位（从右向左数，下标从 0 开始算起；即从左向右数的第 2 位）发生了单比特错误。

- **错误定位**：接收到的码字 $1101011$ 中，左起第 2 位的数据（值为 1）出错了。

- **纠正错误**：将该出错位取反（1 变为 0），得到正确的码字：**$1001011$**。剥离后 3 位校验码，恢复出的正确原始数据为 **$1001$**。
