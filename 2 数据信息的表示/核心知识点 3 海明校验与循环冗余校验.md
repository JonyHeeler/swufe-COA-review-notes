# 海明校验与循环冗余校验

> 来源依据：`知识点汇总.txt` 第二章第 3 点；第 2 章 PPT“数据校验”；教材 P48 例 2.11、P50 例 2.14。

## 一句话理解

校验码就是给原始数据加一些“检查位”，让接收方能发现数据在传输或存储过程中有没有出错。

## 基本概念

计算机保存和传输数据时，某一位可能因为干扰从 `0` 变成 `1`，或从 `1` 变成 `0`。校验码的思路是：除了原始数据，再额外保存一些冗余信息，用这些冗余信息检查数据是否仍然合理。

几个概念先分清：

| 概念 | 作用 |
| --- | --- |
| 奇偶校验 | 用 1 位校验位检查 1 的个数是否满足奇校验或偶校验 |
| 码距 | 两个合法码字之间不同位的个数 |
| 海明码 | 用多个校验位生成指错字，常用于定位并纠正 1 位错误 |
| CRC | 用模 2 除法生成余数，主要用于高效检错 |

“检错”和“纠错”不是一回事。检错只告诉你“可能错了”，纠错还要能指出“哪一位错了”并改回来。

## 工作原理或推导过程

### 1. 海明码为什么满足 `2^r >= k + r + 1`

设原始数据有 `k` 位，增加 `r` 位校验位，总长度就是 `k+r` 位。

如果要纠正 1 位错误，接收方需要区分这些情况：

- 没有错误。
- 第 1 位错了。
- 第 2 位错了。
- ...
- 第 `k+r` 位错了。

一共需要表示：

```text
k + r + 1
```

种状态。

`r` 个校验位能产生 `2^r` 种不同的指错字，所以必须满足：

```text
2^r >= k + r + 1
```

这就是公式的来源，不是凭空背出来的。

### 2. 海明码校验位放在哪里

海明码通常把校验位放在编号为 `1、2、4、8...` 的位置，也就是 `2^0、2^1、2^2、2^3...`。

这样安排的好处是：每个位置编号都能写成二进制，校验位可以按编号中的某一位是否为 1 来分组检查。最后得到的指错字，正好能指出错误位置。

例如位置 `6` 的二进制是 `110`，说明它会被第 2 位和第 4 位校验位检查。

### 3. CRC 为什么用模 2 除法

CRC 把数据位串看作一个二进制多项式。发送方用生成多项式去“除”数据，得到的余数作为校验位。接收方收到后再除一次，如果余数为 0，就认为没有检测到错误。

CRC 的模 2 运算规则很特别：

```text
加法 = 异或
减法 = 异或
没有进位，也没有借位
```

所以做 CRC 除法时，本质上是在不断做异或。

CRC 编码步骤：

1. 设生成多项式位串长度为 `r+1`，则校验位长度为 `r`。
2. 在原始数据后补 `r` 个 0。
3. 用生成多项式做模 2 除法。
4. 把得到的 `r` 位余数接到原始数据后面。

## 典型例子

### 例 1：8 位数据需要几个海明校验位

已知 `k=8`，找最小的 `r`：

```text
r=3：2^3 = 8，k+r+1 = 8+3+1 = 12，不够
r=4：2^4 = 16，k+r+1 = 8+4+1 = 13，够
```

所以至少需要 4 位校验位。

### 例 2：CRC 校验位长度

若生成多项式位串是：

```text
10011
```

它长度为 5，说明阶数为 4，所以 CRC 校验位是 4 位。做除法前要先在数据后补 4 个 0。

## 常见考法

- 已知数据位数，求海明校验位数。
- 构造海明码，确定校验位位置和数据位位置。
- 根据指错字判断哪一位出错。
- 已知生成多项式，计算 CRC 余数。
- 判断某种校验方式能检错还是能纠错。

## 易错点

- 海明码公式里的 `+1` 表示“没有错误”这一种状态。
- 海明码能纠正 1 位错误，不代表能纠正任意多位错误。
- CRC 的生成多项式长度为 `r+1`，校验位是 `r` 位。
- CRC 主要用于检错，通常不直接用于定位并纠正错误。
- 模 2 除法不要做普通减法，看到减法就按异或处理。

## 小结

- 海明码的重点是校验位数量、校验位位置、指错字。
- CRC 的重点是补 0、模 2 除法、余数拼接。
- 做校验码题先判断目标是检错还是纠错，再选择对应方法。
