# 冯诺依曼结构与计算机硬件组成

> 来源依据：第 1 章教师 PPT“冯诺依曼思想”“计算机硬件系统”；教材第 1 章。

## 一句话理解

冯诺依曼结构的核心是：把程序也当作数据一样存进存储器，CPU 按地址一条一条取出来执行。

## 基本概念

冯·诺依曼结构（von Neumann architecture）是现代电子计算机设计的基石。它的基本思想彻底改变了早期计算机需要手动重新接线来改变程序的繁琐方式，其核心理念可以概括为以下四个主要方面：

### 1. 存储程序（Stored-Program Concept）

这是冯·诺依曼结构中**最核心**的思想。

- 计算机必须具有内部存储器。

- **程序（指令）和数据同等对待**，混合存放在同一个可读写的存储器中，并且没有区别对待（都以同样的格式存储）。

- 计算机在运行前，会将编写好的程序和处理所需的数据预先输入并存入主存储器中。

### 2. 二进制表示（Binary System）

- 计算机内部无论是指令（告诉计算机做什么）还是数据（被处理的信息），都统一采用二进制数（0和1）来表示。

- 采用二进制极大简化了计算机硬件的物理实现（如电路的开/关状态）。

### 3. 五大基本组成部件

冯·诺依曼提出计算机硬件系统必须由以下五个基本部分组成：

- **运算器（ALU）：** 负责执行算术运算（如加减乘除）和逻辑运算（如与、或、非）。

- **控制器（Control Unit）：** 计算机的指挥中心，负责从存储器中取出指令，对指令进行解码，并协调其他所有部件来执行该指令。*（运算器和控制器通常被集成在一起，称为中央处理器 CPU）*

- **存储器（Memory）：** 存放程序和数据的“仓库”。

- **输入设备（Input Device）：** 将外部信息（程序、数据等）转换成机器能识别的形式输入到计算机中（如键盘、鼠标）。

- **输出设备（Output Device）：** 将计算机处理的结果转换成人类或其他设备能识别的形式输出（如显示器、打印机）。

### 4. 顺序执行指令

- 程序的执行是自动化的。计算机按照指令在存储器中存放的顺序，由控制器自动、按顺序地一条一条取出并执行，直到程序结束（除非遇到跳转指令改变了执行顺序）。

## 工作原理或推导过程

一条程序运行时，大致按下面的流程推进：

1. 程序和数据先被装入主存。
2. 程序计数器 `PC` 保存“下一条要执行的指令地址”。
3. CPU 按 `PC` 给出的地址从主存取出指令。
4. 控制器对指令译码，判断它是加法、访存、跳转还是其他操作。
5. 运算器、存储器或 I/O 设备按控制信号完成具体动作。
6. `PC` 更新，通常指向下一条指令；如果遇到转移指令，则改为新的地址。

为什么指令和数据都能用二进制表示？因为在存储器里，它们本质上都是一串 `0` 和 `1`。这串二进制到底被当成“指令”还是“数据”，取决于 CPU 当前怎么解释它。

## 典型例子

执行高级语言中的：

```c
c = a + b;
```

机器并不会直接理解这个数学表达式。它可能会做类似下面的事情：

```text
从内存取出 a
从内存取出 b
把 a 和 b 送到运算器相加
把结果写回 c 对应的内存位置
```

这一连串动作都由机器指令驱动。也就是说，高级语言里看起来很自然的一句话，落到硬件上会变成多条取数、运算、存数的机器指令。

## 常见考法

- 判断冯诺依曼结构的基本思想，尤其是“存储程序”。
- 写出五大部件并说明各自作用。
- 区分 CPU、主存、外存和 I/O 设备的职责。
- 解释程序为什么能够自动连续执行。

## 易错点

- “指令和数据都用二进制表示”不等于二者含义相同。含义来自 CPU 的解释方式。
- CPU 能直接访问的是主存和寄存器，外存中的程序通常要先装入主存。
- 控制器不负责真正的数值计算，它主要负责发出控制信号。
- 现代计算机虽然结构更复杂，但学习 CPU、存储器、总线时仍然离不开冯诺依曼主线。

## 小结

- 冯诺依曼结构的关键词是二进制、存储程序、按地址访问、取指执行。
- 五大部件中，运算器负责算，控制器负责指挥，存储器负责保存程序和数据。
- 程序能自动执行，是因为 CPU 能根据 `PC` 不断取出下一条指令。
