# CPI 与 MIPS 相关指标计算

> 来源依据：`知识点汇总.txt` 第一章；第 1 章教师 PPT“计算机系统的性能评价”；教材第 1 章习题。

## 一句话理解

CPI、MIPS 和 CPU 时间都是在衡量同一件事：CPU 为了跑完一段程序，到底忙了多久。

## 基本概念

先把 CPU 想成一个按节拍干活的机器。时钟每跳一下，就给 CPU 一次推进工作的机会，这一下就叫一个**时钟周期**。主频越高，说明每秒跳得越多，每一拍越短。

但一条指令通常不是一拍就能完成的。比如取指令、译码、访问寄存器、运算、写回结果，都可能消耗时间。所以课程用 **CPI** 表示“一条指令平均要花多少个时钟周期”。

几个名词可以这样记：

| 名词 | 直观理解 | 常用单位或写法 |
| --- | --- | --- |
| 时钟周期 | CPU 的一拍有多长 | 秒、纳秒 |
| 主频 | CPU 每秒有多少拍 | Hz、MHz、GHz |
| CPI | 每条指令平均花多少拍 | Cycles Per Instruction |
| IPC | 每拍平均完成多少条指令 | Instructions Per Cycle |
| MIPS | 每秒完成多少百万条指令 | Million Instructions Per Second |
| CPU 时间 | 程序真正占用 CPU 的时间 | 秒 |

其中 CPI 和 IPC 是从两个方向看同一件事。若不考虑复杂流水线影响，可以粗略理解为：

```text
IPC ≈ 1 / CPI
```

## 工作原理或推导过程

性能计算的核心不是死背公式，而是想清楚三步：

1. 程序一共执行了多少条指令。
2. 每条指令平均要用多少个时钟周期。
3. 每个时钟周期有多长。

于是：

```text
CPU 时间 = 指令条数 × 平均 CPI × 时钟周期
```

因为：

```text
时钟周期 = 1 / 主频
```

所以也可以写成：

```text
CPU 时间 = 指令条数 × 平均 CPI / 主频
```

如果题目给出多类指令，就先求平均 CPI。平均 CPI 不是简单平均，而是按“各类指令出现比例”加权：

```text
平均 CPI = Σ(某类指令比例 × 该类指令 CPI)
```

MIPS 从公式推出来也很自然：

```text
MIPS = 每秒执行的指令数 / 10^6
     = 主频(Hz) / 平均 CPI / 10^6
```

如果主频已经用 MHz 表示，可以直接：

```text
MIPS = 主频(MHz) / 平均 CPI
```

## 典型例子

某机器主频为 `1.2GHz`，四类指令比例和 CPI 如下：

| 指令类型 | 比例 | CPI |
| --- | ---: | ---: |
| A | 50% | 2 |
| B | 20% | 3 |
| C | 10% | 4 |
| D | 20% | 5 |

先算平均 CPI：

```text
平均 CPI = 0.5×2 + 0.2×3 + 0.1×4 + 0.2×5
        = 1 + 0.6 + 0.4 + 1
        = 3
```

`1.2GHz = 1200MHz`，所以：

```text
MIPS = 1200 / 3 = 400
```

如果程序共执行 `6×10^8` 条指令，则：

```text
CPU 时间 = 6×10^8 × 3 / (1.2×10^9)
        = 1.5 秒
```

这个例子里，MIPS 说的是“机器吞吐指令的速度”，CPU 时间说的是“这个程序实际跑多久”。考试比较程序快慢时，优先看 CPU 时间。

## 常见考法

- 给主频、指令比例和 CPI，求平均 CPI、MIPS 或 CPU 时间。
- 给优化前后指令条数、CPI 或主频，比较程序运行时间。
- 给 CPU 时间和 I/O 时间，判断 CPU 提速后总运行时间怎么变。
- 比较两台机器运行同一程序或同一基准程序时谁更快。

## 易错点

- `GHz` 转 `MHz` 要乘 `1000`，转 `Hz` 要乘 `10^9`。
- MIPS 变大不一定表示某个程序更快，因为程序的指令条数可能也变了。
- 不同指令集之间不能只看 MIPS。一条复杂指令可能抵得上多条简单指令。
- 平均 CPI 要按比例加权，比例变化后必须重新算。
- I/O 等待时间不随 CPU 主频提升而自动缩短，总时间题要分开计算。

## 小结

- 性能计算的主线是 `指令条数 × CPI × 时钟周期`。
- 主频决定“一拍多长”，CPI 决定“一条指令几拍”，指令条数决定“一共多少条”。
- MIPS 可以辅助观察吞吐量，但比较程序快慢时，CPU 时间更可靠。
- 遇到“优化”“加速”“主频”“CPI”题，先写 CPU 时间公式，再代入数据。
