# 第 1 章 计算机系统概述复习导读

## 本章定位

第 1 章先不急着讲 CPU 里面的细节，而是先回答三个大问题：

- 一台计算机大体由哪些部分组成？
- 人写的程序怎样一步步变成机器能执行的指令？
- 判断一台计算机“快不快”时，到底应该看什么？

可以把本章看成整门课的地图。后面学存储器、指令系统、CPU、流水线时，很多概念都会回到这张地图上。

## 核心优先级

| 优先级 | 内容 | 复习要求 |
| --- | --- | --- |
| 核心 | CPI、MIPS、CPU 时间等性能指标计算 | 会从题目中找出指令条数、CPI、主频，并用 CPU 时间公式比较性能 |
| PPT 补充 | 冯诺依曼结构、硬件系统组成 | 能用自己的话说明“程序存储”和五大部件各做什么 |
| PPT 补充 | 软件系统与层次结构 | 能说清高级语言程序为什么不能被硬件直接执行 |
| PPT 补充 | 计算机发展与摩尔定律 | 理解发展趋势和主要器件变化，不把它当成复杂计算题 |

## 学习顺序

1. 先理解冯诺依曼结构：程序和数据都放在存储器中，CPU 一条一条取指令执行。
2. 再理解软件层次：人写的是高级语言，机器执行的是机器语言，中间需要编译、汇编、链接等步骤。
3. 最后学习性能指标：一段程序跑得快不快，本质上看它消耗了多少 CPU 时钟周期。

## 本章最重要的一条主线

性能题不要先背 MIPS，而要先抓住这条主线：

```text
程序运行时间 = 程序用了多少条指令 × 每条指令平均花多少拍 × 每一拍有多长
```

对应公式就是：

```text
CPU 时间 = 指令条数 × 平均 CPI × 时钟周期
```

如果题目给的是主频，就把“每一拍有多长”换成主频的倒数：

```text
CPU 时间 = 指令条数 × 平均 CPI / 主频
```

## 易错提醒

- 主频高只是说明 CPU 的“节拍更快”，不代表程序一定更快，还要看指令条数和 CPI。
- MIPS 表示“每秒执行多少百万条指令”，但不同机器的一条指令可能做的事不一样，所以它不是绝对性能指标。
- 编译优化可能减少指令条数，也可能改变指令类型比例，做题时要重新算平均 CPI。
- CPU 时间不等于总运行时间。程序等待磁盘、网络、键盘输入时，CPU 可能并没有真正执行该程序。
