# 计算机软件系统与层次结构

> 来源依据：第 1 章教师 PPT“计算机软件系统”“计算机的层次结构”。

## 一句话理解

计算机系统的层次结构说明：人写的是容易理解的程序，硬件执行的是二进制指令，中间需要一层层翻译和管理。

## 基本概念

硬件只认识机器指令，但人不可能每天直接写一长串 `0` 和 `1`。所以计算机系统被设计成多层结构：越上层越接近人的表达，越下层越接近硬件电路。

可以从低到高这样理解：

| 层次 | 主要内容 | 直观作用 |
| --- | --- | --- |
| 硬件层 | CPU、主存、I/O 设备 | 真正执行和保存信息 |
| 机器语言层 | 二进制机器指令 | 硬件能直接执行的语言 |
| 操作系统层 | 进程、内存、文件、设备管理 | 管理资源，让程序能稳定运行 |
| 语言处理层 | 编译器、汇编器、链接器、解释器 | 把人的程序翻译成机器能执行的形式 |
| 应用层 | 浏览器、办公软件、学习程序等 | 直接解决用户问题 |

常见软件角色要分清：

| 软件 | 作用 |
| --- | --- |
| 编译器 | 把高级语言源程序翻译成目标代码 |
| 汇编器 | 把汇编语言翻译成机器代码 |
| 链接器 | 把多个目标文件和库组合成可执行文件 |
| 解释器 | 一边解释一边执行程序 |
| 操作系统 | 管理 CPU、内存、文件和 I/O 设备 |

## 工作原理或推导过程

以一个 C 程序为例，它从“人能读懂”到“机器能执行”通常经历：

```text
C 源程序 → 编译 → 汇编/目标文件 → 链接 → 可执行文件 → 装入内存 → CPU 执行机器指令
```

为什么要有这么多步骤？因为每一步解决的问题不同：

- 编译把 `if`、`for`、函数调用等高级结构变成更接近机器的代码。
- 汇编把助记符形式的指令变成真正的二进制指令。
- 链接把你写的代码和库函数组合起来，例如把 `printf` 所在的库接进来。
- 操作系统负责把可执行文件装入内存，并安排 CPU 执行它。

这也解释了为什么程序性能不只由硬件决定。编译器生成多少条指令、操作系统怎样调度、内存访问是否高效，都会影响最后的运行时间。

## 典型例子

同一段 C 代码：

```c
sum = a + b;
```

在程序员眼中是一句加法；在编译器眼中，它要被转换成取数、加法、存结果等机器操作；在 CPU 眼中，它最终只是一串按格式编码好的二进制指令。

如果打开编译优化，编译器可能减少不必要的指令。此时 CPU 主频没有变，但程序可能运行更快，因为“要执行的指令条数”变少了。

## 常见考法

- 判断硬件能直接执行的是机器语言，不是高级语言或汇编语言。
- 区分编译器、汇编器、解释器和链接器。
- 说明计算机系统从低到高有哪些层次。
- 结合性能指标，分析编译优化为什么可能改变 CPU 时间。

## 易错点

- 汇编语言虽然接近机器，但仍然不是硬件能直接执行的二进制机器码。
- 操作系统不是硬件的一部分，但它是程序运行环境的重要组成部分。
- 编译和解释不是同一件事：编译通常先生成目标代码，解释通常边分析边执行。
- 可执行文件也不是“自动运行”的，通常需要操作系统装入内存并建立运行环境。

## 小结

- 软件层次的意义是把人的意图逐步转换成硬件能执行的机器指令。
- 硬件直接执行机器语言，高级语言需要经过语言处理系统转换。
- 性能分析不能只看 CPU，也要考虑编译器、操作系统和程序本身。
