# 习题 1 答案

## 1.1 名词解释

#### 第一组：计算机发展规律

- **摩尔定律 (Moore's Law):** 由英特尔创始人之一戈登·摩尔提出。其核心内容是：当价格不变时，集成电路上可容纳的元器件（晶体管）的数目，约每隔 18 到 24 个月便会增加一倍，性能也将提升一倍。它揭示了信息技术进步的速度。

#### 第二组：程序翻译与系统软件

这一组名词主要涉及如何将人类编写的代码转化为计算机能识别的程序。

- **汇编器 (Assembler):** 一种系统软件，负责将使用汇编语言编写的源程序翻译成机器语言（二进制代码）形式的目标程序。

- **编译器 (Compiler):** 一种系统软件，负责将高级语言（如 C、C++）编写的源程序一次性、完整地翻译成等价的机器语言目标程序。

- **解释器 (Interpreter):** 另一种处理高级语言的系统软件。与编译器不同，解释器不生成完整的目标程序，而是读取源程序的每一条语句，逐句进行翻译并立即交由计算机执行。

- **链接器 (Linker):** 当程序被编译或汇编成若干个目标模块后，链接器负责将这些独立的模块以及所需的标准库函数有机地拼接、连接在一起，生成最终可以在计算机上运行的可执行文件。

#### 第三组：计算机基本结构参数

这些参数反映了计算机硬件的基本物理和架构属性。

- **时钟周期 (Clock Cycle):** 计算机主频（时钟频率）的倒数。它是CPU内部工作的最基本、最小的时间单位（节拍），计算机的各项微操作都是在时钟脉冲的控制下进行的。

- **机器字长 (Machine Word Length):** 计算机 CPU 一次能够直接处理和运算的二进制数据的位数。它通常与CPU内部运算器（ALU）和通用寄存器的宽度一致，直接影响计算机的计算精度和速度。

- **主存容量 (Main Memory Capacity):** 计算机主存储器（内存）所能容纳的最大二进制信息量，通常以字节（Byte）为单位进行衡量（如 KB, MB, GB）。

#### 第四组：计算机性能评价指标

这些指标主要用于衡量 CPU 或计算机系统的运行速度和效率。

- **CPI (Cycles Per Instruction):** 执行一条指令所需的平均时钟周期数。CPI 越小，说明执行指令的速度越快。

- **IPC (Instructions Per Cycle):** 每个时钟周期内 CPU 能执行的平均指令条数。它是 CPI 的倒数，现代超标量处理器通常 IPC 大于 1。

- **MIPS (Million Instructions Per Second):** 每秒钟执行的百万条指令数。它是一个衡量计算机运算速度的常用指标（通常用于定点指令）。

- **MFLOPS (Million Floating-point Operations Per Second):** 每秒钟执行的百万次浮点运算数。它主要用于衡量计算机处理科学计算和浮点数运算的能力。

- **CPU时间 (CPU Time):** CPU 真正用来执行某个特定程序所花费的时间。它只计算 CPU 处于运行状态的时间，不包括等待 I/O 操作完成等所消耗的等待时间。

## 1.3 冯诺依曼结构

基本思想：程序和数据都用二进制表示并存放在存储器中，计算机按地址取指令，译码后自动执行。

硬件系统包括运算器、控制器、存储器、输入设备和输出设备。运算器完成算术逻辑运算，控制器产生控制信号，存储器保存程序和数据，输入输出设备负责外部交互。

## 1.4 计算机系统层次

可分为硬件层、机器语言层、操作系统层、语言处理层和应用层。硬件层提供物理执行能力，机器语言层直接面向指令，操作系统管理资源，高级语言和应用层让用户用更自然的方式表达任务。

## 1.5 MIPS 计算

平均 CPI：

```text
CPI1 = 0.4×2 + 0.2×3 + 0.15×4 + 0.25×5 = 3.25
CPI2 = 0.4×2 + 0.2×2 + 0.15×3 + 0.25×4 = 2.65
```

MIPS：

```text
MIPS1 = 600 / 3.25 ≈ 184.62
MIPS2 = 800 / 2.65 ≈ 301.89
```

计算机 2 的 MIPS 更高。

## 1.6 编译优化前后

#### （1）优化前后程序的 CPI 各为多少？

**优化前：**

程序的综合 CPI 是各类指令 CPI 与其在程序中所占比例的加权平均值。

$CPI_{\text{前}} = (0.4 \times 1) + (0.2 \times 2) + (0.15 \times 2) + (0.25 \times 2)$

$CPI_{\text{前}} = 0.4 + 0.4 + 0.3 + 0.5 = 1.6$

**优化后：**

为了方便理解，我们可以假设优化前程序的总指令数为 100 条。

- 优化前各类指令数量：A类 40 条，B类 20 条，C类 15 条，D类 25 条。

- 优化前总时钟周期数：$100 \times 1.6 = 160$ 个周期。

- 优化后，A类指令数量减少一半（变为 20 条），其他指令数量不变。

- 优化后的总指令数变为：$20 + 20 + 15 + 25 = 80$ 条。

- 优化后的总时钟周期数：原来的 160 个周期减去减少的那 20 条 A 类指令所占的周期（$20 \times 1 = 20$），即 $160 - 20 = 140$ 个周期。

$CPI_{\text{后}} = \frac{\text{总时钟周期数}}{\text{总指令数}} = \frac{140}{80} = 1.75$

#### （2）优化前后程序的 MIPS 各为多少？

MIPS 的计算公式为：$MIPS = \frac{f}{CPI \times 10^6}$ （其中 $f$ 为 CPU 主频）。已知主频为 **500MHz**。

**优化前：**

$MIPS_{\text{前}} = \frac{500 \times 10^6}{1.6 \times 10^6} = \frac{500}{1.6} = 312.5$

**优化后：**

$MIPS_{\text{后}} = \frac{500 \times 10^6}{1.75 \times 10^6} = \frac{500}{1.75} \approx 285.71$

#### （3）通过上面的计算结果，你能得出什么结论？

通过计算发现一个看似矛盾的现象：程序经过“优化”后，总指令数减少了，总时钟周期数也减少了（从 160 降至 140），这说明**程序的实际运行时间变短了，计算机的实际性能是提升的**。然而，反映在指标上，CPI 反而升高了（1.6 涨到 1.75），MIPS 反而下降了（312.5 降到 285.71）。
