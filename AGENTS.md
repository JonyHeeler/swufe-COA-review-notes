# 《计算机组成原理》复习资料构建协作说明

本文件用于指导 AI 与用户一起从 0 开始构建《计算机组成原理》课程复习资料。

## 基础资料

当前文件夹根目录中的主要资料如下：

- `计算机组成原理 (谭志虎) (z-library.sk, 1lib.sk, z-lib.sk)(1.pdf`：教材。
- `知识点汇总.txt`：核心知识点。
- 八份教师 PPT：对应课程八个章节。

所有整理、讲解、习题与答案资料，建议统一使用 Markdown 格式输出。

## 第一项工作：整理文件夹

课程一共八章，需要在当前根目录下建立八个章节文件夹，命名格式为：

```text
x 章节名称
```

建议八个文件夹分别为：

```text
1 计算机系统概述
2 数据信息的表示
3 运算方法与运算器
4 存储系统
5 指令系统
6 中央处理器
7 指令流水线
8 总线系统
```

每个章节文件夹内部应包含以下内容：

1. 教师 PPT。
2. `知识点汇总.txt` 中属于本章节的核心知识点解说。
   - 命名格式：`核心知识点 x 知识点名字.md`
3. 本章 PPT 中出现、但 `知识点汇总.txt` 中未出现的知识点。
   - 命名格式：`知识点 x 知识点名字.md`
4. 本章章后习题及其答案。

## 知识点整理原则

知识点优先级如下：

```text
核心知识点 > PPT 中出现但不在核心知识点中的知识点
```

如果某个内容既不在核心知识点中，也不在本章 PPT 中出现，则直接忽略，不纳入复习资料。

## 写作要求

所有知识点必须做到：

- 准确：概念、公式、流程、结论不能含糊或错误。
- 清晰：结构分明，重点突出，便于复习。
- 深入浅出：面向从未学过计算机组成原理的学生，先解释直观含义，再进入专业表述。
- 善于总结：每个知识点应尽量包含核心定义、理解方法、常见考法、易错点和必要例子。
- 尊重资料来源：优先依据教材、教师 PPT 和 `知识点汇总.txt`，不要凭空扩展无关内容。

## 推荐的单个知识点 Markdown 结构

每个知识点文件建议采用以下结构：

```markdown
# 知识点标题

## 一句话理解

用一句话说明这个知识点到底在解决什么问题。

## 基本概念

解释核心定义、相关术语和背景。**这一步是最重要的重点，需要你用通俗易懂的语言讲清楚相关概念，并进行拓展。比如，| 补码整数 | `-2^(n-1) ~ +(2^(n-1)-1)` |，为什么是这样？你需要解释原理。**

## 工作原理或推导过程

如果涉及结构、流程、公式或计算方法，应分步骤说明。**不仅仅要说明过程怎么做，你还要解释清楚为什么要这么做，怎么想，有没有什么技巧。**

## 典型例子

给出便于初学者理解的例子，必要时配合表格或计算过程。

## 常见考法

总结考试中可能如何考查该知识点。

## 易错点

指出容易混淆、容易算错或容易误解的地方。

## 小结

用简洁条目总结本知识点必须掌握的内容。
```

## 总体协作流程

建议按章节顺序推进：

1. 先整理章节文件夹和原始资料。
2. 再从 `知识点汇总.txt` 中提取本章核心知识点。
3. 逐一为核心知识点编写 Markdown 解说。
4. 再阅读本章教师 PPT，补充 PPT 中出现但核心知识点未覆盖的知识点。
5. 最后整理本章章后习题及答案。
6. 每完成一章后，检查命名、内容准确性、覆盖范围和 Markdown 格式。

最终目标是形成一套适合零基础学生系统复习的《计算机组成原理》学习资料。
